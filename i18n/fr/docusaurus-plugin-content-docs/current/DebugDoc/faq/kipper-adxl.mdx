## Installation des paquets dépendants

:::tip
Il est recommandé d'utiliser des outils SSH comme `MobaXterm_Personal`.
:::

:::tip
Le logiciel supérieur FLY n'a pas besoin de l'exécuter, car il comprend déjà les paquets dépendants de l'accéléromètre.
:::

    * Veuillez exécuter les commandes ci-dessous pour installer les paquets dépendants.
    * Notez que selon la performance du processeur, cela peut prendre beaucoup de temps, jusqu'à 10-20 minutes. Veuillez patienter jusqu'à ce que l'installation soit terminée. Dans certains cas, si le RAM de la carte mère est insuffisant, l'installation pourrait échouer.

        ```bash
        sudo apt install python3-numpy python3-matplotlib libatlas-base-dev
        ```

        ```bash
        ~/klippy-env/bin/pip install matplotlib numpy
        ```

## Test de l'accéléromètre

    * Vous devez d'abord réinitialiser avant d'exécuter.
    * Exécutez `ACCELEROMETER_QUERY` dans la console.
    * Pour plusieurs accéléromètres, vous pouvez spécifier le test pour un certain `ACCELEROMETER_QUERY CHIP=<ADXL>` où `<ADXL>` doit être modifié en fonction du nom de l'accéléromètre correspondant.
    * Si une erreur se produit, vérifiez les connexions et les configurations.
    * La sortie normale est illustrée ci-dessous
        <ImageView image={require('@site/docs/DebugDoc/faq/img/adxl_1.webp').default} size="80%" align="center" />
    * Ainsi, vous pouvez effectuer la mesure de compensation de résonance.


## Calibration de l'accéléromètre

    * Réinitialisez.
    * Entrez `SHAPER_CALIBRATE` dans la console, cela commencera automatiquement la calibration sur X,Y.
    * Après la calibration, exécutez `SAVE_CONFIG` pour sauvegarder les données.
    * Vous pouvez également utiliser `SHAPER_CALIBRATE AXIS=X` pour calibrer automatiquement un axe.
    * Le processus de calibration peut prendre un certain temps, veuillez patienter.

:::tip
Attention ! Si le test est trop violent pendant l'impression, appuyez immédiatement sur l'arrêt d'urgence ou envoyez `M112` pour arrêter.
:::

    * Un `accel_per_hz` trop élevé peut rendre les vibrations intenses, diminuez-le si nécessaire.

        ```cfg
        [resonance_tester]
        accel_per_hz: 50  # Valeur par défaut de 75
        ```
