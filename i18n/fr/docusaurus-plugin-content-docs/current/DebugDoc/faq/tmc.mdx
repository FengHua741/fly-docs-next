# Erreurs liées au TMC

## `Impossible de lire l'enregistrement IFCNT du bus série UART 'stepper_x' du tmc` ou `Impossible de lire l'enregistrement de bus série SPI 'stepper_x' du tmc`

    1. Détails de l'erreur

        <ImageView image={require('@site/docs/DebugDoc/faq/img/tmc/uart.webp').default} size="80%" align="center" />

    2. Cause de l'erreur

        * Impossible de communiquer avec le pilote

    3. Méthodes principales pour résoudre le problème

        * Vérifier si la configuration des broches UART du pilote correspondante est correcte
        * Les sauts UART sont-ils corrects ?
        * Le pilote est-il alimenté ?
        * Remplacer le pilote ou configurer la prise pour effectuer un diagnostic
        * Si c'est un autre fabricant de pilotes, vous devez déterminer vous-même si les broches UART du pilote correspondent à celles de FLY. Les broches UART du pilote FLY sont la quatrième broche en partant du haut vers le bas.
        <ImageView image={require('@site/docs/DebugDoc/faq/img/tmc/2209uart.webp').default} size="80%" align="center" />
        * Si les broches UART de votre pilote sont sur la cinquième broche, vous pouvez résoudre cela avec les sauts mentionnés ci-dessous
        <ImageView image={require('@site/docs/DebugDoc/faq/img/tmc/uart1.webp').default} size="80%" align="center" />

## `Le TMC XXXX signale une erreur : GSTAT : 00000001 reset=1(RAZ)`

    1. Détails de l'erreur

        <ImageView image={require('@site/docs/DebugDoc/faq/img/tmc/reset.webp').default} size="80%" align="center" />

    2. Cause de l'erreur

        * Réinitialisation du pilote


    3. Méthodes principales pour résoudre le problème

        * Les câbles d'alimentation ne sont pas bien connectés
        * Le fil du moteur est coupé
        * Vous pouvez connecter l'enveloppe métallique du moteur au GND de la carte mère ou de l'outil

## `Le TMC 'extrudeur' signale une erreur : DRV_STATUS : c00d0103 otpw=1(Avertissement de surchauffe !)`

    1. Détails de l'erreur

        <ImageView image={require('@site/docs/DebugDoc/faq/img/tmc/overtemperror.webp').default} size="80%" align="center" />

    2. Cause de l'erreur

        * Le pilote est surchauffé !
        * Ce n'est pas le moteur qui est surchauffé !

    3. Méthodes principales pour résoudre le problème

        * Réduire le courant de fonctionnement
        * Supprimer le courant de maintien
        * Vérifier si les résistances de mesure du pilote sont correctes
        * Améliorer la dissipation thermique du pilote, par exemple en ajoutant un ventilateur

## `Le TMC 'extrudeur' signale une erreur : DRV_STATUS : 601f1000 s2vsa=1(Courant court-circuité vers l'alimentation A!) csactual=31 ola=1(Charge ouverte A!) olb=1(Charge ouverte B!)`

    1. Détails de l'erreur

        * Aucune image disponible

    2. Cause de l'erreur

        * La vérification du courant du pilote est incorrecte


    3. Méthodes principales pour résoudre le problème
        
        * Vérifier si la résistance des deux bobines de l'entraînement du pas-à-pas est identique
        * Vérifier si les fils internes du moteur de pas-à-pas ou des composants sont lâches ou en court-circuit
        * Désactiver le mode silencieux

## `Le TMC 'extrudeur' signale une erreur : DRV_STATUS : 00190060 s2vsb=1(Courant court-circuité vers l'alimentation B!) ola=1(Charge ouverte A!) cs_actual-25`

    1. Détails de l'erreur
    
        <ImageView image={require('@site/docs/DebugDoc/faq/img/tmc/ab.webp').default} size="80%" align="center" />

    2. Cause de l'erreur

        * Connexion incorrecte ou le moteur lui-même a un problème


    3. Méthodes principales pour résoudre le problème
        
        * Vérifier si la résistance des deux bobines de l'entraînement du pas-à-pas est identique
        * Vérifier votre moteur de pas-à-pas et vos connexions
        * Vérifier l'ordre des fils, s'il s'agit d'un groupe à gauche et d'un groupe à droite
