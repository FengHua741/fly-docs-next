# Tutoriel d'utilisation des pilotes externes

## Remarques importantes

:::danger À savoir

* Ce tutoriel est basé sur Klipper pour le contrôle
* Le module utilise une connexion en anode commune

:::

## Sauter les réglages de subdivision du pilote

    * Aucun saut de pilote n'est nécessaire !!!

## Codeur intégré au pilote
    * Il est conseillé d'utiliser `3200` pour le codeur intégré au pilote
        <ImageView image={require('@site/docs/DebugDoc/faq/img/tmc/3200.webp').default} size="80%" align="center" />

## Configuration de référence Klipper

    * Il faut masquer la configuration du pilote correspondant, par exemple `[tmc5160 stepper_x] `ou`[tmc2209 stepper_x] `
    * Définir `microsteps:` à `16`
    * Ajouter `step_pulse_duration: 0.000004`
    * Activation du moteur : supprimer le `!` dans `enable_pin:!`
        <ImageView image={require('@site/docs/DebugDoc/faq/img/tmc/cfg.webp').default} size="50%" align="center" />

## Explication des impulsions du pilote

    :::info

    * Apparition d'un décalage lors de l'impression du modèle
    * Vibration du moteur, bruits inhabituels, divers problèmes étranges
    * Modifier `step_pulse_duration: 0.000004`
    * Augmenter cette valeur à `0.000009`, par exemple `0.00001` ou `0.00002` etc.

    :::
