# Étendre le contrôle de l'LED via des plugins

Document original

```cfg
https://github.com/julianschill/klipper-led_effect
```

## Installation

Veuillez utiliser des outils SSH tels que **MobaXterm_Personal** et vous connecter via **WIFI** à votre ordinateur principal. Assurez-vous également des points suivants :

1. **Assurez-vous que le firmware Klipper est installé sur votre ordinateur principal**
2. **Assurez-vous que l'utilisateur connecté est celui qui a installé Klipper**
3. **Assurez-vous que votre méthode de saisie est en anglais**
4. **Assurez-vous que votre ordinateur principal peut détecter correctement l'appareil**
5. **Assurez-vous que toutes les précautions ci-dessus sont respectées, sinon vous ne pourrez pas passer à l'étape suivante**

Ce module peut être installé dans une installation Klipper existante à l'aide du script d'installation.

```cfg
cd ~
git clone https://github.com/julianschill/klipper-led_effect.git
cd klipper-led_effect
./install-led_effect.sh
```

## Configuration de référence

```cfg
[neopixel TP_led]
pin: PB7
chain_count: 25
# Nombre de LED
# Nombre de LED
color_order: GRB
initial_RED: 0.4    
initial_GREEN: 0.8
initial_BLUE: 1
initial_WHITE: 0.0
#66CCFF 

[led_effect sb_nozzle_cooling]
autostart:              false
frame_rate:             24
leds:
    neopixel:TP_led (9,10)
layers:
        breathing  3 1 top (0.0, 0.0, 1.0, 0.1)

[led_effect rainbow]
leds:
    neopixel:TP_led
autostart:                          true
frame_rate:                         24
layers:
    gradient  0.3  1 add (0.3, 0.0, 0.0),(0.0, 0.3, 0.0),(0.0, 0.0, 0.3)
```
