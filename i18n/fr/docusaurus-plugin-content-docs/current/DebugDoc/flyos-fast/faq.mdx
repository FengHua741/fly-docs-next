# Questions fréquentes sur FlyOS-Fast

## Question : L'exécution de la commande affiche « command not found »
- **Cause** : Le système n'a pas cette commande.
- **Solution** : Le système est préinstallé avec de nombreux packages logiciels courants. Si vous avez besoin d'autres packages, vous pouvez nous les soumettre, et s'il y a lieu, ils seront inclus dans la prochaine version.

## Question : Pourquoi l'exécution de la commande `apt update` échoue-t-elle ?
- **Cause** : FlyOS-Fast est un système léger et rapide, il ne dispose pas du gestionnaire de paquets apt et ne peut donc pas exécuter la commande `apt update`.
- **Solution** : Le système est préinstallé avec de nombreux packages logiciels courants. Si vous avez besoin d'autres packages, vous pouvez nous les soumettre, et s'il y a lieu, ils seront inclus dans la prochaine version.

## Question : L'exécution de la commande de gravure affiche « -bash: ~/klippy-env/bin/python: Aucun fichier ou dossier de ce type »
- **Cause** : FlyOS-Fast n'utilise pas l'environnement virtuel venv.
- **Solution** : Remplacez `~/klippy-env/bin/python` par `python` pour l'exécution.

## Question : Pourquoi ne peux-tu pas te connecter en tant qu'utilisateur « fly » ?
- **Cause** : FlyOS-Fast n'a qu'un utilisateur root, toutes les opérations se font sous cet utilisateur.
- **Solution** : Le système n'a pas d'utilisateur « fly », connectez-vous directement en tant que « root ».

## Question : Pourquoi fly-tools sur le port 9999 ne peut-il pas être ouvert ?
- **Cause** : FlyOS-Fast n'est pas installé avec fly-tools.
- **Solution** : Fluidd et mainsail prennent déjà en charge la consultation des ID, les configurations d'écran sont disponibles dans [la configuration système](/docs/DebugDoc/flyos-fast/config).

## Question : Pourquoi l'écran affiche-t-il uniquement le LOGO sans interface KlipperScreen ?
- **Cause** : KlipperScreen n'est pas activé.
- **Solution** : Activez KlipperScreen dans [la configuration système](/docs/DebugDoc/flyos-fast/config) en définissant `klipper_screen` à `1`.

## Question : Pourquoi le système ne fournit-il pas de mise à jour web pour klipper et autres composants ?
- **Cause** : Les composants klipper, moonraker intégrés dans FlyOS-Fast sont des versions secondaires.
- **Solution** : Si nécessaire, vous pouvez mettre à jour manuellement via la ligne de commande, mais il est déconseillé de le faire. Attendez la mise à jour du système.

## Question : Pourquoi les scripts d'installation de composants tiers comme Happy Rabbit ne fonctionnent-ils pas ?
- **Cause** : L'utilisateur par défaut de FlyOS-Fast est « root », ces scripts d'installation tiers détectent toujours root et ne procèdent pas à l'installation.
- **Solution** : Il suffit de modifier légèrement pour contourner la détection root dans le script d'installation. Vous pouvez vous référer [ici](https://github.com/kluoyun/Happy-Hare/tree/flyos-dev), les autres scripts d'installation sont similaires.
