# Questions fréquentes sur FlyOS-Fast

## Question : L'exécution de la commande affiche « command not found »
- **Cause** : Le système n'a pas cette commande.
- **Solution** : Le système est préinstallé avec de nombreux paquets logiciels courants. Si vous avez besoin d'autres paquets, vous pouvez nous les soumettre et ils seront peut-être préinstallés dans la version suivante.

## Question : Pourquoi la commande `apt update` ne fonctionne-t-elle pas ?
- **Cause** : FlyOS-Fast est un système léger et rapide qui ne dispose pas de l'outil de gestion des paquets apt et ne peut donc pas exécuter la commande `apt update`.
- **Solution** : Le système est préinstallé avec de nombreux paquets logiciels courants. Si vous avez besoin d'autres paquets, vous pouvez nous les soumettre et ils seront peut-être préinstallés dans la version suivante.

## Question : L'exécution de la commande de gravure affiche « -bash: ~/klippy-env/bin/python: Aucun fichier ou dossier de ce type »
- **Cause** : FlyOS-Fast n'utilise pas l'environnement virtuel venv.
- **Solution** : Remplacez `~/klippy-env/bin/python` par `python` pour l'exécution.

## Question : Pourquoi ne peux-je pas me connecter en tant qu'utilisateur « fly » ?
- **Cause** : FlyOS-Fast n'a qu'un utilisateur root et toutes les opérations se font sous cet utilisateur.
- **Solution** : Le système n'a pas d'utilisateur « fly ». Connectez-vous directement en tant que « root ».

## Question : Pourquoi fly-tools sur le port 9999 ne s'ouvre pas ?
- **Cause** : FlyOS-Fast n'est pas installé avec fly-tools.
- **Solution** : Fluidd et mainsail supportent tous deux la recherche d'ID, et les configurations d'écran sont disponibles dans [Configuration du système](/docs/DebugDoc/flyos-fast/config).

## Question : Pourquoi l'écran ne montre que le LOGO et non l'interface KlipperScreen ?
- **Cause** : KlipperScreen n'est pas activé.
- **Solution** : Activez KlipperScreen dans [Configuration du système](/docs/DebugDoc/flyos-fast/config) en définissant `klipper_screen` à `1`.

## Question : Pourquoi le système ne fournit-il pas de mise à jour web pour klipper et autres composants ?
- **Cause** : Les composants intégrés tels que klipper et moonraker dans FlyOS-Fast sont des versions développées en interne.
- **Solution** : Si nécessaire, vous pouvez mettre à jour vous-même via la ligne de commande, mais il est déconseillé de le faire. Attendez plutôt une mise à jour du système.
