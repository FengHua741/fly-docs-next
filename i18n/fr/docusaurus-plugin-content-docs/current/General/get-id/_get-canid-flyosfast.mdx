## Remplir l'ID CAN

    1. Fermez `DEVICES` et remplissez l'ID
    :::danger Avertissement

    * La première boîte rouge précédente avec le symbole # doit être supprimée, assurez-vous également de supprimer les espaces, puis changer `serial:` en `canbus_uuid:`
    * Le système FLY inclut automatiquement `[MCU]`, mais si l'ID ci-dessous est `serial: /tmp/klipper_host_mcu`, il faut renommer `[MCU]` en `[mcu HOST]`, ou supprimer ces deux lignes
    * Vous pouvez copier l'ID suivant `canbus_uuid:`
    :::

    ```cfg
    [mcu]
    canbus_uuid:
    ```

    <table>
      <tr>
        <td>

          <ImageView image={require('@site/docs/General/get-id/img/wed/fluidd_cfg.webp').default} size="100%" align="center" />
        </td>
        <td>
          - Dans mainsail, trouvez `[MCU]`
          <ImageView image={require('@site/docs/General/get-id/img/wed/mainsail_cfg.webp').default} size="100%" align="center" />
        </td>
      </tr>
    </table>


    2. Remplissez l'ID dans la configuration
    :::tip Conseil

    * Après avoir rempli l'ID, cliquez sur `SAVE & RESTART` en haut à droite
    :::

    3. Si Klipper affiche un message `ADC out of range`, c'est un phénomène normal, connectez le lit chauffant et le thermocouple au circuit principal, configurez bien les broches des sondes de température pour la buse et le lit chauffant, puis enregistrez et redémarrez.

:::danger
Attention : Tous les ID mentionnés dans ce document sont des exemples, chaque carte mère a un ID différent, veuillez utiliser l'ID réellement obtenu.
:::
