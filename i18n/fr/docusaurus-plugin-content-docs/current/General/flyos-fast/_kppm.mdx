## Configuration

### Désactivation de l'extinction automatique en cas de coupure de courant
    :::danger À savoir
    - Il est nécessaire de désactiver l'extinction automatique en cas de coupure de courant, sinon la fonction de reprise après coupure ne pourra pas être utilisée.
    - La fonction de reprise après coupure possède une fonctionnalité de sauvegarde de progression avant extinction automatique.
    :::

- Via la barre d'URL du navigateur, entrez l'adresse IP du périphérique, par exemple : `http://192.168.6.179`
- Accédez à la page de configuration
    - Ouvrez l'adresse IP du périphérique dans le navigateur, par exemple : `http://192.168.1.2/`
    - Dans Fluidid, cliquez sur l'image de gauche pour `désélectionner` -> `filtrer et masquer les fichiers et dossiers cachés`.
    - Dans Mainsail, cliquez sur l'image de droite pour `sélectionner` -> `afficher les fichiers cachés`.

### Fichier de configuration plr.cfg
    :::danger À savoir
    * Remplacez les éléments suivants dans le fichier de configuration : (PINS)  
    * Par <code>{props.power_pine}</code>

    :::

    - Dans la page de configuration de l'imprimante, trouvez le fichier `plr.cfg`.
    - Effacez tout son contenu, puis collez la configuration ci-dessous :
    - Le contenu du fichier de configuration est le suivant :



    ```cfg
    [mcu host]   
    serial: /tmp/klipper_host_mcu
    
    [power_loss_resume]
    power_pin: (PINS)
    is_shutdown: True # Exécute-t-il une opération d'arrêt ? Défaut : activé
    paused_recover_z: -2.0 # Si l'arrêt survient lors d'une pause d'impression, la distance de déplacement Z lors de la reprise d'impression, défaut : sans déplacement
    start_gcode:
        # G-code exécuté avant la reprise
        # Tous les paramètres enregistrés avant la coupure peuvent être récupérés via {PLR}
        # Utilisez M118 {PLR} pour afficher tous les paramètres disponibles
        M118 Début de la reprise: {PLR.print_stats.filename}
        M118 Position d'interruption: X:[{PLR.POS_X}] Y:[{PLR.POS_Y}] Z:[{PLR.POS_Z}] E:[{PLR.POS_E}]
        M140 S{PLR.bed.target}      ; Réglage de la température du plateau chauffant
        M104 S{PLR.extruder.target-10}  ; Réglage de la température de l'extrudeuse
        M109 S{PLR.extruder.target-10}  ; Attente que l'extrudeuse atteigne la température définie
        G91                             ; Coordonnées relatives
        G1 Z2 F100                      ; Élévation de Z, préparation au zéro X,Y
        G90                             ; Coordonnées absolues
        G28 X Y                         ; Zéro XY
        M140 S{PLR.bed.target}          ; Réglage de la température du plateau chauffant
        M104 S{PLR.extruder.target}     ; Réglage de la température de l'extrudeuse
        M190 S{PLR.bed.target}          ; Attente que le plateau chauffant atteigne la température définie
        M109 S{PLR.extruder.target}     ; Attente que l'extrudeuse atteigne la température définie
        M83                             ; Extrusion relative
        # G1 E0.5 F400                  ; Extrusion légère
    layer_count: 2 # Nombre de couches à imprimer avant l'exécution de layer_change_gcode
    layer_change_gcode:                
        # G-code à exécuter après avoir imprimé {layer_count} couches
        M118 Reprise de la vitesse d'impression
        M106 S{PLR.fan_speed}             ; Activation du ventilateur d'extrusion
        M220 S{PLR.move_speed_percent}    ; Définition du pourcentage de vitesse demandée
        M221 S{PLR.extrude_speed_percent} ; Définition du pourcentage de vitesse d'extrusion demandé
    shutdown_gcode:
        # G-code à exécuter avant l'extinction
        M118 Faible tension du courant, extinction
        # M112 ; Arrêt d'urgence

    ```

    :::warning Attention

    - Les macros dans le fichier de configuration `start_gcode` ci-dessus doivent peut-être être modifiées en fonction des spécificités de votre machine.
    - Notez que si vous utilisez `[homing_override]`, il est déconseillé de modifier arbitrairement les positions de mise à zéro dans `[homing_override]`. Aucune responsabilité ne sera prise en cas d'échec de la reprise après coupure due à cela.

    :::

- Enregistrez le fichier de configuration ci-dessus.
- Ouvrez le fichier `printer.cfg` et ajoutez le contenu suivant au début du fichier :

    ```cfg

    [include plr.cfg]

    ```

- Cliquez sur le bouton en haut à droite pour enregistrer et redémarrer.
- À ce stade, la fonction de reprise après coupure de Klipper est configurée.

## Test

- Imprimez un fichier quelconque pendant qu'un autre fichier est en cours d'impression, puis cliquez sur le bouton d'arrêt d'urgence pour simuler une coupure de courant.
- Cliquez ensuite sur « Redémarrer le firmware », attendez que Klipper se connecte normalement.
- Si une fenêtre de notification apparaît sur le site web ou dans KlipperScreen, cela signifie que la fonction de reprise après coupure fonctionne correctement.
- Vous pouvez ensuite tester la situation réelle de coupure de courant.
