## Important Notice

:::tip À lire absolument

- Les paramètres de la section `partie` dans le fichier de configuration ne prendront effet qu'au premier démarrage du système après gravure.
- Ce tutoriel ne fournit que les options modifiables.

:::

:::warning Attention

- Essayez d'utiliser les pages web `mainsail` ou `fluidd` pour modifier les configurations.
- Ne pas ouvrir la traduction des pages web, sinon le système risque de ne pas démarrer après enregistrement.
- Il est déconseillé d'introduire des caractères chinois ou des symboles spéciaux dans le fichier de configuration.
- Après modification du fichier de configuration, un redémarrage du système est nécessaire pour que les changements prennent effet.

:::

:::danger

- Ne modifiez pas les parties du fichier de configuration suivantes :
  `######### PLEASE DO NOT EDIT THE FOLLOWING #########`.

:::

## Comment modifier le fichier de configuration

### Modification du fichier de configuration après gravure

- Les paramètres dont l'effet n'est valable qu'au premier démarrage doivent être modifiés après gravure en utilisant un éditeur de texte professionnel pour ouvrir le fichier de configuration `config.txt` sur le disque amovible `FlyOS-Conf`.

### Modification en ligne pendant le fonctionnement du système

- Si vous modifiez les paramètres standards après le démarrage du système via la page de configuration du site web, il suffit de redémarrer le système pour que les modifications prennent effet.
    - Ouvrez le navigateur et accédez à l'adresse IP de l'appareil, par exemple : `http://192.168.1.2/`
    - Dans fluidd, décochez comme indiqué à gauche `filtrer les fichiers et dossiers cachés`.
    - Dans mainsail, cochez comme indiqué à droite `afficher les fichiers cachés`.

## Paramètres de configuration

### Configuration réseau sans fil

- Les paramètres suivants prendront effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :---: | :--- | :--- |
| `WIFI_SSID` | `vide` | Nom quelconque de réseau WIFI | Nom du réseau WIFI, si rempli, le système tentera de se connecter automatiquement à ce réseau au démarrage |
| `WIFI_PASS` | `vide` | Laisser vide si pas de mot de passe | Mot de passe du réseau WIFI, si rempli, le système essaiera de se connecter avec ce mot de passe |

----

### Configuration par défaut de l'imprimante

:::tip
- Les paramètres suivants ne prendront effet qu'au premier démarrage.
:::

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :---: | :--- | :--- |
| `language` | `zh-CN` | <details><summary>Cliquez pour déployer le contenu</summary>`zh-CN`, `zh-HK`, `en`, `de`, `cz`, `es`, `hu`, `ja`, `nl`, `pt`, `se`, `uk`, `da`, `fr`, `it`, `ko`, `pl`, `ru`, `tr`</details> | Définir la langue par défaut pour fluidd et mainsail au premier démarrage |
| `printer_name` | `vide` | Chaîne quelconque | Nom par défaut de l'imprimante pour fluidd et mainsail au premier démarrage |

----

### Configuration du système

- Les paramètres suivants prendront effet à chaque démarrage du système.
- Ci-dessous sont les options modifiables.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `board` |  <code>{props.board}</code> |   <code>{props.board}</code> | Modèle de carte mère, ne pas modifier les valeurs autres que celles disponibles |
| `localtime` | `Asia/Shanghai` | [Tableau des fuseaux horaires mondiaux](/docs/DebugDoc/flyos-fast/timezone) | Définir le fuseau horaire par défaut du système, le système ajustera automatiquement l'heure après connexion à Internet |
| `canbus_bitrate` | `1000000` | `1000000`<br/> `500000`<br/> `250000` | Définir le taux de transmission par défaut pour tous les appareils can du système FAST, ne peut être modifié que ici, valeur par défaut 1M |

----

### Configuration de l'affichage

- Les paramètres suivants prendront effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `screen` | `hdmi` | <code>nano</code>,<code>{props.screen}</code> <br/> <code>{props.screen1}</code><code>{props.screen2}</code> | Définir l'écran, `none` pour désactiver tous les affichages |
| `rotate` | `90` | `0`, `90`, `180`, `270` | Définir l'angle de rotation pour FLY-TFT, non applicable pour HDMI |
| `klipper_screen` | `0` | `0`, `1` | Définir si KlipperScreen est activé<br/>1: Activé<br/>0: Désactivé<br/>Valide lorsque `printer=klipper` |
| `moon2uart` | `0` | `0`, `1` | Définir si moon2uart est activé<br/>1: Activé<br/>0: Désactivé<br/>Valide lorsque `printer=klipper` |

----

### Configuration liée à Klipper

- Les paramètres suivants prendront effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `klipper_webui` | `fluidd` | `fluidd`, `mainsail` | Définir l'interface utilisateur par défaut pour Klipper<br/>Ne modifie pas la fonctionnalité de basculement en un clic en ligne<br/>Valide lorsque `printer=klipper` |
| `canbus_bitrate` | `1000000` | `1000000`, `500000`, `250000` | Définir le taux de transmission par défaut pour tous les appareils can du système FAST, ne peut être modifié que ici, valeur par défaut 1M |
| `crowsnest` | `0` | `0`, `1` | Définir si Crowsnest est activé<br/>1: Activé<br/>0: Désactivé<br/>Valide pour les modes Klipper et RRF |

----

### Configuration du type d'imprimante

- Les paramètres suivants prendront effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `printer` | `klipper` | `klipper`, `rrf` | Configurer le service de démarrage du système<br/>`klipper`: services Klipper, Moonraker, etc.<br/>`rrf`: services RRF DSF, DWC, etc. |

----

### Arrêt du système lors de la coupure de courant

- Les paramètres suivants prendront effet à chaque démarrage du système.
- Pour utiliser la fonctionnalité de reprise après coupure de courant, il faut désactiver la fonction d'arrêt du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `shutdown_pin_state` | `0` | `0`, `1` | Définir l'état de niveau de la broche d'arrêt<br/> 1: Niveau bas répété<br/> 0: Niveau haut déclenché  |
| `shutdown_pin` | <code>{props.shutdown_pin}</code>| <code>{props.shutdown_pin}</code> | Définir la broche de déclenchement de l'arrêt, définir `none` pour désactiver  |

----

### Configuration liée à RRF

- Les paramètres suivants prendront effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `rrf2uart` | `0` | `0`, `1` | Inactif pour l'instant, mise à jour disponible dans une version ultérieure |

----

:::tip
- Les paramètres suivants ne prendront effet qu'au premier démarrage.

| `firmware` | `vide` | Chemin valide vers un fichier de firmware | Configurer le firmware à flasher automatiquement au démarrage, actuellement valide pour C8, Gemini-V3 et C5 |
