## Important Notice

:::::tip À lire obligatoirement

- Les paramètres de la section `partie` du fichier de configuration ne prendront effet qu'une seule fois lors du premier démarrage après le flashage.
- En raison de la complexité du processus de configuration, il est vivement recommandé de configurer la machine via le navigateur une fois connecté en réseau.
- Ce tutoriel ne fournit que les options modifiables.

:::::

:::::warning Attention

- Essayez d'utiliser les pages web `mainsail` ou `fluidd` pour modifier les configurations.
- Ne traduisez pas les pages web, sinon le système risque de ne pas démarrer après la sauvegarde.
- Évitez d'introduire des caractères chinois ou spéciaux dans le fichier de configuration.
- Une fois les modifications apportées au fichier de configuration, un redémarrage du système est nécessaire pour que les changements prennent effet.

:::::

:::::danger

- N'éditez pas les contenus situés sous la section `######### PLEASE DO NOT EDIT THE FOLLOWING #########` du fichier de configuration.

:::::

## Comment modifier le fichier de configuration

### Modification du fichier de configuration après le flashage

- Les paramètres qui ne prennent effet qu'au premier démarrage doivent être modifiés après le flashage en utilisant un éditeur de texte professionnel pour ouvrir le fichier de configuration `config.txt` sur le disque amovible `FlyOS-Conf`.

### Modification en ligne de configuration après le démarrage du système

- Si vous modifiez les paramètres normaux via la page de configuration web du système et redémarrez ensuite, les modifications prendront effet.
    - Ouvrez l'adresse IP du périphérique dans le navigateur, par exemple : `192.168.1.2/`.
    - Pour fluidd, décochez comme indiqué à gauche dans la capture d'écran ci-dessous -> `Masquer les fichiers et dossiers cachés`.
    - Pour mainsail, cochez comme indiqué à droite dans la capture d'écran ci-dessous -> `Afficher les fichiers cachés`.

    - Après avoir effectué les modifications et sauvegardé, un redémarrage du système fera prendre effet les changements.

## Paramètres de configuration

### Configuration sans fil

- Les paramètres suivants prennent effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :---: | :--- | :--- |
| `WIFI_SSID` | `vide` | Nom quelconque du réseau Wi-Fi | Nom du réseau Wi-Fi, si rempli, le système essaiera de se connecter automatiquement à ce réseau au démarrage. |
| `WIFI_PASS` | `vide` | Laisser vide si sans mot de passe | Mot de passe du réseau Wi-Fi, si rempli, le système utilisera ce mot de passe pour se connecter. |

----

### Configuration par défaut de l'imprimante

:::tip
- Les paramètres suivants ne prennent effet qu'au premier démarrage.
:::

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :---: | :--- | :--- |
| `language` | `zh-CN` | <details><summary>Cliquez pour déployer le contenu détaillé</summary>`zh-CN`, `zh-HK`, `en`, `de`, `cz`, `es`, `hu`, `ja`, `nl`, `pt`, `se`, `uk`, `da`, `fr`, `it`, `ko`, `pl`, `ru`, `tr`</details> | Définissez la langue par défaut de fluidd et mainsail lors du premier démarrage. |
| `printer_name` | `vide` | Chaîne de caractère arbitraire | Définissez le nom par défaut de l'imprimante pour fluidd et mainsail lors du premier démarrage. |

----

### Configuration du système

- Les paramètres suivants prennent effet à chaque démarrage du système.
- Voici les options modifiables.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `board` |  <code>{props.board}</code> |   <code>{props.board}</code> | Modèle de la carte mère, ne modifiez pas les valeurs possibles. |
| `localtime` | `Asia/Shanghai` | [Tableau des fuseaux horaires mondiaux](/docs/DebugDoc/flyos-fast/timezone) | Configurez le fuseau horaire par défaut du système, qui s'ajustera automatiquement après connexion au réseau. |
| `canbus_bitrate` | `1000000` | `1000000`<br/> `500000`<br/> `250000` | Configurez le taux de transmission par défaut pour tous les appareils CAN dans le système FAST, cette option ne peut être modifiée qu'ici. La valeur par défaut est de 1M. |

----

### Configuration de l'affichage

- Les paramètres suivants prennent effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `screen` | `hdmi` | <code>nano</code>,<code>{props.screen}</code> <code>{props.screen3}</code> <br/> <code>{props.screen1}</code><code>{props.screen2}</code>| Configurez l'écran, `none` pour désactiver toutes les sorties vidéo. |
| `rotate` | `90` | `0`, `90`, `180`, `270` | Configurez l'angle de rotation de FLY-TFT, cette option est inutile pour HDMI. |
| `klipper_screen` | `0` | `0`, `1` | Activez ou non KlipperScreen.<br/>1: Activer<br/>0: Désactiver<br/>Valide uniquement si `printer=klipper`. |
| `moon2uart` | `0` | `0`, `1` | Activez ou non moon2uart.<br/>1: Activer<br/>0: Désactiver<br/>Valide uniquement si `printer=klipper`. |

----

### Configuration liée à Klipper

- Les paramètres suivants prennent effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `klipper_webui` | `fluidd` | `fluidd`, `mainsail` | Configurez l'interface utilisateur frontale par défaut de Klipper.<br/>Ne modifie pas la fonctionnalité de basculement en un clic en ligne.<br/>Valide uniquement si `printer=klipper`. |
| `canbus_bitrate` | `1000000` | `1000000`, `500000`, `250000` | Configurez le taux de transmission par défaut pour tous les appareils CAN dans le système FAST, cette option ne peut être modifiée qu'ici. La valeur par défaut est de 1M. |
| `crowsnest` | `0` | `0`, `1` | Activez ou non Crowsnest.<br/>1: Activer<br/>0: Désactiver<br/>Valide pour les modes Klipper et RRF. |

----

### Configuration du type d'imprimante

- Les paramètres suivants prennent effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `printer` | `klipper` | `klipper`, `rrf` | Configurez le service à lancer lors du démarrage du système.<br/>`klipper`: services Klipper, Moonraker, etc.<br/>`rrf`: services RRF DSF, DWC, etc. |

----

### Arrêt automatique après coupure de courant

- Les paramètres suivants prennent effet à chaque démarrage du système.
- Pour activer la fonctionnalité de reprise après coupure de courant, la fonction d'arrêt automatique doit être désactivée.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `shutdown_pin_state` |<code>{props.state}</code>| `0`, `1` | Configurez l'état de niveau logique du bouton d'arrêt.<br/> 1: État bas répété<br/> 0: État haut déclenché. |
| `shutdown_pin` | <code>{props.shutdown_pin}</code>| <code>{props.shutdown_pin}</code> | Configurez la broche de déclenchement du bouton d'arrêt, définissez `none` pour désactiver. |

----

### Configuration liée à RRF

- Les paramètres suivants prennent effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `rrf2uart` | `0` | `0`, `1` | Non valide pour l'instant, mise à jour prévue dans une version ultérieure. |

- Les paramètres suivants ne prennent effet qu'au premier démarrage.

| `firmware` | `vide` | Chemin du fichier de firmware valide | Configurez le firmware à burner au démarrage, actuellement valable uniquement pour C8, Gemini-V3 et C5. |
