## Important Notice

::::astuce À lire obligatoirement

- Les paramètres dans la section `partie` du fichier de configuration ne prendront effet qu'au premier démarrage après le gravage.
- En raison de la complexité du processus de configuration, il est vivement recommandé de configurer l'appareil via un navigateur une fois connecté en réseau.
- Ce tutoriel ne fournit que les options modifiables.

::::

::::avertissement Attention

- Essayez d'utiliser les pages web `mainsail` ou `fluidd` pour modifier les paramètres.
- Ne traduisez pas les pages web, sinon, après la sauvegarde, le système risque de ne pas démarrer.
- Il est préférable de ne pas entrer de caractères chinois ou de caractères spéciaux dans le fichier de configuration.
- Après avoir modifié le fichier de configuration, un redémarrage du système est nécessaire pour que les modifications prennent effet.

::::

::::danger

- Les parties du fichier de configuration sous `######### PLEASE DO NOT EDIT THE FOLLOWING #########` ne doivent pas être modifiées.

::::

## Comment modifier le fichier de configuration

### Modification du fichier de configuration après le gravage

- Les paramètres qui ne prennent effet qu'au premier démarrage après le gravage doivent être modifiés en utilisant un éditeur de texte professionnel pour ouvrir le fichier de configuration `config.txt` sur le disque amovible `FlyOS-Conf`.

### Modification en ligne après le démarrage du système

- Si c'est un paramètre normal, après modification dans la page de configuration du système et un redémarrage, cela prendra effet.
    - Ouvrez l'adresse IP du périphérique dans le navigateur, par exemple : `192.168.1.2/`
    - Pour fluidd, décochez comme indiqué à gauche dans la capture d'écran ci-dessous -> `filtrer et masquer les fichiers et dossiers cachés`.
    - Pour mainsail, cochez comme indiqué à droite dans la capture d'écran ci-dessous -> `afficher les fichiers cachés`.

    - Après la modification et l'enregistrement, un redémarrage du système est nécessaire pour que les changements prennent effet.

## Paramètres de configuration

### Configuration sans fil

- Les paramètres suivants prendront effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :---: | :--- | :--- |
| `WIFI_SSID` | `vide` | N'importe quel nom de réseau WiFi | Nom du réseau WiFi, si renseigné, se connectera automatiquement lors du démarrage |
| `WIFI_PASS` | `vide` | Laisser vide si aucun mot de passe | Mot de passe du réseau WiFi, si renseigné, utilisera le mot de passe pour se connecter au WiFi |

----

### Configuration par défaut de l'imprimante

:::astuce
- Les paramètres suivants ne prendront effet qu'au premier démarrage.
:::

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :---: | :--- | :--- |
| `language` | `zh-CN` | <details><summary>Cliquez pour déployer le contenu</summary>`zh-CN`, `zh-HK`, `en`, `de`, `cz`, `es`, `hu`, `ja`, `nl`, `pt`, `se`, `uk`, `da`, `fr`, `it`, `ko`, `pl`, `ru`, `tr`</details> | Définit la langue par défaut pour fluidd et mainsail au premier démarrage |
| `printer_name` | `vide` | N'importe quelle chaîne de caractères | Définit le nom par défaut de l'imprimante pour fluidd et mainsail au premier démarrage |

----

### Configuration du système

- Les paramètres suivants prendront effet à chaque démarrage du système.
- Voici les options modifiables.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `board` |  <code>{props.board}</code> |   <code>{props.board}</code> | Modèle de la carte mère, ne modifiez pas les valeurs possibles |
| `localtime` | `Asia/Shanghai` | [Tableau des fuseaux horaires mondiaux](/docs/DebugDoc/flyos-fast/timezone) | Définit le fuseau horaire par défaut du système, le système ajustera automatiquement l'heure après connexion au réseau |
| `canbus_bitrate` | `1000000` | `1000000`<br/> `500000`<br/> `250000` | Définit le taux de transmission par défaut pour tous les appareils CAN dans le système FAST, ne peut être modifié qu'ici, valeur par défaut 1M |

----

### Configuration d'affichage

- Les paramètres suivants prendront effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `screen` | `hdmi` | <code>nano</code>,<code>{props.screen}</code> <code>{props.screen3}</code> <br/> <code>{props.screen1}</code><code>{props.screen2}</code><br/> <code>{props.screen4}</code><code>{props.screen5}</code>| Définit l'écran, `none` pour fermer tous les affichages |
| `rotate` | `90` | `0`, `90`, `180`, `270` | Définit l'angle de rotation pour FLY-TFT, inutile pour HDMI |
| `klipper_screen` | `0` | `0`, `1` | Définit si KlipperScreen doit être activé<br/>1: Activer<br/>0: Désactiver<br/>Valide lorsque `printer=klipper` |
| `moon2uart` | `0` | `0`, `1` | Définit si moon2uart doit être activé<br/>1: Activer<br/>0: Désactiver<br/>Valide lorsque `printer=klipper` |


----

### Configuration liée à Klipper

- Les paramètres suivants prendront effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `klipper_webui` | `fluidd` | `fluidd`, `mainsail` | Définit l'interface utilisateur frontale par défaut de Klipper<br/>Neffectue pas d'impact sur la fonction de basculement en un clic en ligne<br/>Valide lorsque `printer=klipper` |
| `canbus_bitrate` | `1000000` | `1000000`, `500000`, `250000` | Définit le taux de transmission par défaut pour tous les appareils CAN dans le système FAST, ne peut être modifié qu'ici, valeur par défaut 1M |
| `crowsnest` | `0` | `0`, `1` | Définit si Crowsnest doit être activé<br/>1: Activer<br/>0: Désactiver<br/>Valide pour les modes Klipper et RRF |

----

### Configuration du type d'imprimante

- Les paramètres suivants prendront effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `printer` | `klipper` | `klipper`, `rrf` | Configure le service de démarrage du système<br/>`klipper`: services klipper, moonraker, etc.<br/>`rrf`: services RRF DSF, DWC, etc. |

----

### Arrêt automatique du système

- Les paramètres suivants prendront effet à chaque démarrage du système.
- Pour utiliser la fonction de reprise après coupure de courant, désactivez la fonction d'arrêt automatique du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `shutdown_pin_state` |<code>{props.state}</code>| `0`, `1` | Définit l'état de niveau logique du bouton d'arrêt<br/> 1: Appui répété en bas niveau<br/> 0: Appui unique en haut niveau  |
| `shutdown_pin` | <code>{props.shutdown_pin}</code>| <code>{props.shutdown_pin}</code> | Définit le port de broche du bouton d'arrêt, définir `none` pour désactiver  |

----

### Configuration liée à RRF

- Les paramètres suivants prendront effet à chaque démarrage du système.

| Paramètre | Valeur par défaut | Valeurs possibles | Description |
| :---: | :-------: | :--- | :--- |
| `rrf2uart` | `0` | `0`, `1` | Inactif temporairement, mise à jour disponible dans les prochaines versions |
