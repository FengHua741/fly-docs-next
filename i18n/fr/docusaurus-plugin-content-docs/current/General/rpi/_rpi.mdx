### Remarques

    1. Il est interdit de brancher ou de débrancher tout appareil autre que l'USB lorsque celui-ci est alimenté.
    2. L'interface Type C peut être utilisée pour l'alimentation ou comme interface USB SLAVE pour graver une image.
    3. Pour garantir une alimentation normale du CM4 lors de la gravure d'une image via l'interface Type C, ne connectez pas d'autres appareils.
    4. Le module n'est doté d'aucune protection. Veuillez éviter les courts-circuits.
    5. La carte mère FLY ne prend pas en charge la fonction POE ! Veuillez ne pas utiliser d'alimentation POE.

:::danger À savoir
* Attention : la version EMMC ne peut pas utiliser la carte SD.
* Attention : la version sans EMMC doit utiliser une carte MICRO SD pour le démarrage et nécessite un lecteur de cartes pour graver l'image.
* Il est recommandé que la vitesse de transmission de la carte mémoire TF soit de niveau `class10` ou supérieur. Il est conseillé d'utiliser des cartes TF de marque SanDisk par exemple.
:::

## Téléchargement de rpiboot (programme de démarrage)

    * Téléchargez et exécutez le logiciel rpiboot avec des privilèges d'administrateur [cliquez ici pour télécharger](https://files.waveshare.net/wiki/w/upload/f/f3/Rpiboot_setup.zip)
    * Il sert à installer les pilotes et les outils de démarrage. Une fois installé, il y a un fichier rpiboot.exe dans le répertoire d'installation.

    <ImageView image={require('@site/docs/General/rpi/img/boot.webp').default} size="80%" align="left" />

## Entrer en mode de gravure

    * Connectez l'ordinateur et alimentez la carte mère. Dans le gestionnaire de périphériques de l'ordinateur, un appareil BCMxxx sera détecté.

        <ImageView image={require('@site/docs/General/rpi/img/bcm2711.webp').default} size="80%" align="left" />


    * Ensuite, exécutez rpiboot

        <ImageView image={require('@site/docs/General/rpi/img/rpiboot.webp').default} size="80%" align="left" />


    * Attendez la fin de l'exécution, un lecteur amovible apparaîtra dans mon ordinateur.

:::danger À savoir
Attention : si l'ordinateur ne détecte pas l'appareil BCMxxx, c'est probablement parce qu'une étape précédente est incorrecte. Il est nécessaire de détecter l'appareil BCMxxx pour que l'exécution de rpiboot soit efficace, sinon elle est inefficace.
:::


## Formater la carte EMMC ou SD

    * Utilisez le logiciel SDFormatter.exe [cliquez ici pour télécharger](https://www.waveshare.net/w/upload/d/d7/Panasonic_SDFormatter.zip) pour formater la carte SD.

        <ImageView image={require('@site/docs/General/rpi/img/sdformatter.webp').default} size="80%" align="left" />


    * Ou utilisez la gestion des disques intégrée à Windows pour supprimer les partitions. Après avoir supprimé les partitions, créez-en de nouvelles.
    * Faites un clic droit sur cet ordinateur -> Gérer -> Gestion des disques -> Trouvez le nouveau lecteur (choisissez mal et c'est fichu) -> Supprimez toutes les partitions -> Créez une partition (n'importe quel système de fichiers). Si le système a été gravé auparavant, il y aura deux lecteurs, ignorez-les et formatez ou supprimez simplement les partitions.

## Écriture de l'image

    1. Utilisez Raspberry Pi Imager [cliquez ici pour télécharger](https://www.raspberrypi.com/software/) pour graver l'image.
    2. Avant de graver, configurez le nom d'utilisateur et le mot de passe, ainsi que l'activation de SSH.

            <ImageView image={require('@site/docs/General/rpi/img/imager.webp').default} size="80%" align="left" />


    3. Une fois la gravure terminée, un lecteur amovible apparaîtra dans mon ordinateur.

## Fin de la gravure et redémarrage

    * Une fois la gravure terminée, déconnectez l'alimentation, déconnectez le câble de connexion à l'ordinateur et remettez le commutateur BOOT à sa position initiale. Alimentez à nouveau pour démarrer.
