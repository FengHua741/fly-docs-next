# Méthode de configuration

:::danger À savoir
Veuillez noter que vous ne pourrez pas utiliser l'interface d'accéléromètre mini pad pour connecter un accéléromètre externe après avoir utilisé cette méthode.
:::

    * Ouvrez le disque BOOT et trouvez `armbianEnv.txt` et ouvrez-le
    * Modifiez **uart2** en **uart3** dans `overlays=usbhost2 usbhost3 uart2` puis enregistrez

    <ImageView image={require('@site/docs/ProductDoc/SBC/fly-mini-pad/img/uart.webp').default} size="80%" align="center" />

## Méthode de câblage

    * Veuillez noter que le TX du mini-pad doit être connecté au RX de la carte mère avec le firmware série flashée
    * Veuillez noter que le RX du mini-pad doit être connecté au TX de la carte mère avec le firmware série flashée
    * Veuillez noter qu'une connexion de terre commune est nécessaire
    * Sinon, vous ne pourrez pas connecter la carte mère avec le firmware série flashée

    <ImageView image={require('@site/docs/ProductDoc/SBC/fly-mini-pad/img/uart1.webp').default} size="80%" align="center" />

## Configuration de référence Klipper

    ```cfg
    [mcu]
    serial: /dev/ttyS3
    baud：250000
    #   Le taux de transmission utilisé. La valeur par défaut est 250000
    restart_method:command
    ```
