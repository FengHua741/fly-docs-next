# Prise d'alimentation 5V pour ventilateur on-board

## Le panneau dispose d'une interface contrôlable de 5V, utilisée pour le refroidissement du contrôleur principal. La méthode de câblage est illustrée ci-dessous

    <ImageView image={require('@site/docs/ProductDoc/SBC/fly-pi/img/corefan.webp').default} size="80%" align="center" />

## Configuration de référence

    ```cfg
    [mcu host]                      # FlyOS a généré par défaut la MCU Linux pour vous ; définir permet son utilisation          
    serial: /tmp/klipper_host_mcu 

    [temperature_sensor FLY-π]      # Température de la station supérieure
    sensor_type: temperature_host


    [temperature_fan core_fan]      # Ventilateur de refroidissement de la station supérieure
    pin: host:gpiochip1/gpio103     # Broche du ventilateur de la station supérieure
    max_power: 1.0
    sensor_type: temperature_host   # Définir comme température du contrôleur principal de la station supérieure
    control:watermark               # Mode de contrôle
    target_temp: 48                 # Température de démarrage du ventilateur de refroidissement de la station supérieure
    min_temp: 0                     # Température minimale, en dessous de laquelle une erreur sera signalée
    max_temp: 90                    # Température maximale, au-delà de laquelle une erreur sera signalée
    off_below: 0.10
    kick_start_time: 0.50
    max_speed: 0.8                  # Vitesse maximale, correspondant à 80% de la puissance pleine
    min_speed: 0.3                  # Vitesse minimale, correspondant à 30% de la puissance pleine
    ```
