# CAN 固件编译

## 开始编译

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez les broches GPIO à configurer au démarrage du microcontrôleur, entrez :!PA15  Veuillez noter que le ! est en majuscule
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-mmu/mmu/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, apparaît **Enregistrer la configuration**, appuyez ensuite sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte suivant, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Écriture du firmware

    * Avant d'écrire le firmware, connectez le panneau d'outils à UTOC ou à une carte mère avec du firmware de pont, ci-dessous se trouve un schéma de câblage de référence
    * Vérifiez si les commutateurs sont configurés correctement
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-mmu/mmu/flash/img/can_mode.webp').default} size="90%" align="center" />
    * Connectez-vous à UTOC par référence
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en utilisant un multimètre en mode coupé, si elle est de `120Ω`, connectez une résistance de `120Ω`, si elle est d'environ `40Ω`, retirez la résistance
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-mmu/mmu/flash/img/120.webp').default} size="50%" align="center" />
