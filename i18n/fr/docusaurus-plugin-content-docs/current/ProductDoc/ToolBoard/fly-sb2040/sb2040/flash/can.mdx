    * Sélectionnez les broches GPIO à configurer au démarrage du microcontrôleur, entrez : gpio24
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur `Q` pour afficher **Save configuration**, puis appuyez sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration devrait maintenant être sauvegardée et vous êtes revenu à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale est la suivante, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Burnage du firmware

    * Avant de brûler le firmware, connectez la carte outil à UTOC ou à une carte mère flashée avec le firmware bridge en utilisant un câble XT30. Voici un schéma de câblage de référence
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/sb2040_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en utilisant un multimètre en circuit ouvert. Si elle est de `120Ω`, vous devez connecter une résistance de `120Ω`. Si elle est d'environ `40Ω`, vous devez retirer la résistance
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/120.webp').default} size="50%" align="center" />
