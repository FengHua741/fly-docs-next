# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionner les broches GPIO à configurer au démarrage du micro-contrôleur, entrer : gpio24
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyer sur la touche `Q`, apparaît **Save configuration**, puis appuyer sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration doit maintenant être enregistrée et vous êtes revenu à l'interface de commande
    * Entrer la commande suivante pour commencer la compilation, cela peut prendre un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte ci-dessous, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating bin file out/klipper.bin
        ```

    <LS />


## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte d'outil via une ligne de données XT30 à UTOC ou à une carte mère flashed avec le firmware bridge. Voici un schéma de raccordement de référence
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/sb2040_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` lorsqu'il n'y a pas d'alimentation (si elle est de `120Ω`, connectez une résistance de `120Ω`, si elle est d'environ `40Ω`, retirez la résistance)
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/120.webp').default} size="50%" align="center" />

    <CANFLASH />
