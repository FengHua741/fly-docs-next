# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez les **broches GPIO à configurer au démarrage du micro-contrôleur**, entrez : **!gpio5,!gpio19,!gpio24**
    :::avertissement
        Veuillez entrer `!gpio5,!gpio19,!gpio24` en mode saisie anglaise.
    :::
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, puis apparaîtra **Enregistrer la configuration**, appuyez ensuite sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration doit maintenant être sauvegardée et vous êtes revenu à l'interface de commande
    * Entrez `make -j4` pour commencer la compilation, cela prendra un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble à ce qui suit, la compilation a réussi :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

    <LS />


## Écriture du firmware

    * Avant d'écrire le firmware, connectez la carte outil à UTOC ou à une carte mère flashed avec le firmware de pont en utilisant un câble XT30. Voici un schéma de raccordement de référence
    * Vérifiez que les codes sont correctement positionnés
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/sb2040v3_utoc.webp').default} size="80%" align="center" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en coupant le courant avec un multimètre. Si elle est d'environ `120Ω`, activez le code de `120Ω`. Si elle est d'environ `40Ω`, désactivez le code
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/120.webp').default} size="100%" align="center" />
