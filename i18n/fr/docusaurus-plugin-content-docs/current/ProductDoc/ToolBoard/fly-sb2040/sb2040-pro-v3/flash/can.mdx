# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez les **broches GPIO à configurer au démarrage du microcontrôleur**, entrez : **!gpio5,!gpio19,!gpio24**
    :::avertissement
        Veuillez entrer `!gpio5,!gpio19,!gpio24` en mode saisie anglaise.
    :::
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur `Q`, puis sur `Y` pour sauvegarder la configuration
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration doit maintenant être sauvegardée et vous devez être revenu à l'interface de commande
    * Entrez la commande suivante pour démarrer la compilation, cela peut prendre un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale est similaire à ce qui suit, la compilation a réussi :

        ```bash
        Linking out/klipper.elf
        Creating bin file out/klipper.bin
        ```

    <LS />


## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte outil à UTOC ou à une carte mère avec le firmware bridge via un câble XT30. Voici un schéma de raccordement de référence
    * Assurez-vous que les dip-switches sont positionnés correctement
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/sb2040v3_utoc.webp').default} size="80%" align="center" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en mesure à vide avec un multimètre. Si elle est de `120Ω`, activez le dip-switch de `120Ω`. Si elle est d'environ `40Ω`, désactivez le dip-switch
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/120.webp').default} size="100%" align="center" />
