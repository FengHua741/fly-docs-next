# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez **GPIO pins to set at micro-controller startup**, entrez : **!gpio5**
    :::tip
        Veuillez entrer en mode de saisie en anglais : `!gpio5,!gpio19,!gpio24`
    :::
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur `Q`, apparaît **Save configuration**, appuyez alors sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de commande
    * Entrez `make -j4` pour commencer la compilation, cela peut prendre un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale est similaire au texte ci-dessous, la compilation a réussi :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Graver le firmware

    * Avant de graver le firmware, connectez la carte d'outil à UTOC ou à une carte mère flashée avec le firmware bridge via un câble XT30. Voici un schéma de raccordement de référence
    * Assurez-vous que les sélecteurs sont bien positionnés
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/sb2040v3_utoc.webp').default} size="80%" align="center" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en cas d'absence d'alimentation. Si elle est de `120Ω`, activez le sélecteur `120Ω`. Si elle est d'environ `40Ω`, désactivez le sélecteur.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/120.webp').default} size="100%" align="center" />
