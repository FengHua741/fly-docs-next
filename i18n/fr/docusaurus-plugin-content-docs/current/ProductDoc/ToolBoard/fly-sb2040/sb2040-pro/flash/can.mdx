## 开始编译

    <SSH />

    <CAN />

    * Sélectionnez les broches GPIO à configurer au démarrage du microcontrôleur, entrez : gpio24
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, apparaît **Save configuration**, appuyez ensuite sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration devrait maintenant être enregistrée et vous êtes sorti dans l'interface de ligne de commande
    * Tapez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble à ceci, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Écriture du firmware

    * Avant d'écrire le firmware, connectez le panneau d'outils à UTOC ou à une carte mère flashée avec le firmware de pont à l'aide d'un câble XT30, voir ci-dessous pour le schéma de câblage de référence
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/sb2040_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en utilisant un multimètre en position déconnectée, si elle est de `120Ω`, connectez une jumelle de `120Ω`, si elle est d'environ `40Ω`, retirez la jumelle
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro/flash/img/120.webp').default} size="50%" align="center" />
