# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <CAN />

    * Choisir les broches GPIO à configurer au démarrage du micro-contrôleur, entrer : gpio24
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyer sur la touche `Q`, apparaît **Enregistrer la configuration**, puis appuyer sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration doit maintenant être enregistrée et vous êtes revenu à l'interface de commande
    * Entrer la commande ci-dessous pour démarrer la compilation, cela prend un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale est similaire à ce qui suit, la compilation a réussi :

        ```bash
        Linking out/klipper.elf
        Creating bin file out/klipper.bin
        ```

    <LS />

## Graver le firmware

    * Avant de graver le firmware, connectez la carte d'outils à UTOC ou à une carte mère flashed avec le firmware bridge à l'aide d'un câble XT30. Voici un schéma de raccordement de référence :
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/sb2040_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L mesure environ `60Ω` en coupant l'alimentation. Si elle est de `120Ω`, connectez une résistance de `120Ω`. Si elle est d'environ `40Ω`, retirez la résistance.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro/flash/img/120.webp').default} size="50%" align="center" />
