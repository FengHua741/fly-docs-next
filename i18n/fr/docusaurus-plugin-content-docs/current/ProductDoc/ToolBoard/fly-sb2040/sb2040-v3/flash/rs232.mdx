# Compilation du firmware RS232

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <RS232 />

    * Sélectionnez **GPIO pins to set at micro-controller startup**, entrez : **!gpio5**
    :::warning
        Veuillez entrer `!gpio5` en mode d'entrée anglaise.
    :::
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/rs232.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, apparaît **Save configuration**, appuyez alors sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte ci-dessous, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating uf2 file out/klipper.uf2
        ```


## Entrer en mode de programmation

    :::tip

    À gauche se trouve le bouton `RESET`, à droite se trouve le bouton `BOOT`

    :::

    * Il y a deux méthodes pour entrer en DFU
    * Première méthode : Dans le cas où la carte mère est complètement débranchée, maintenez enfoncé le bouton BOOT puis branchez le câble de transfert de données d'un côté sur la carte mère et de l'autre côté sur l'ordinateur
    * Deuxième méthode : Avec le câble de transfert de données branché d'un côté sur la carte mère et de l'autre côté sur l'ordinateur, maintenez enfoncé le bouton BOOT, puis appuyez sur le bouton RST, relâchez ensuite le bouton RST et enfin relâchez le bouton BOOT
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/boot.webp').default} size="100%" align="center" />
