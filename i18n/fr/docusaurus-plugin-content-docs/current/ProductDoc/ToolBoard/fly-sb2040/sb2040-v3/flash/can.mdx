# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez **GPIO pins to set at micro-controller startup**, entrez : **!gpio5,!gpio19,!gpio24**
    :::warning
        Veuillez entrer `!gpio5,!gpio19,!gpio24` en mode de saisie anglaise.
    :::
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur `Q`, apparaîtra **Save configuration**, puis appuyez à nouveau sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela prendra un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte ci-dessous, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

    <LS />

## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte outil à UTOC ou à une carte mère avec un firmware bridge via une broche XT30. Voici un schéma de raccordement de référence
    * Assurez-vous que les interrupteurs sont positionnés correctement
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/sb2040v3_utoc.webp').default} size="80%" align="center" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en coupant l'alimentation. Si elle est de `120Ω`, activez l'interrupteur correspondant, si elle est d'environ `40Ω`, désactivez l'interrupteur
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/120.webp').default} size="100%" align="center" />
