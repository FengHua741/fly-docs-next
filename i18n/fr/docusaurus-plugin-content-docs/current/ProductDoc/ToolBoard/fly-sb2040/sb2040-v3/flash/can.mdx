# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez **GPIO pins to set at micro-controller startup**, entrez : **!gpio5**
    :::warning
        Veuillez entrer dans le mode de saisie en anglais : `!gpio5,!gpio19,!gpio24`
    :::
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur `Q`, apparaît **Save configuration**, puis appuyez à nouveau sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de commande
    * Entrez `make -j4` pour démarrer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale est la suivante, la compilation a réussi :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte d'outil à UTOC ou à une carte mère flashed avec le firmware de pont en utilisant un câble XT30, ci-dessous est un schéma de câblage de référence
    * Assurez-vous que les commutateurs sont positionnés correctement
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/sb2040v3_utoc.webp').default} size="80%" align="center" />
    * Vérifiez que la résistance entre CAN H et CAN L est d'environ `60Ω` en coupant le courant avec un multimètre, si c'est `120Ω`, activez le commutateur `120Ω`, si c'est environ `40Ω`, désactivez le commutateur
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/120.webp').default} size="100%" align="center" />
