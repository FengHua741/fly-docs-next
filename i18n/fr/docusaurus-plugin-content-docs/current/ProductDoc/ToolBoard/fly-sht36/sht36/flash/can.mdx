# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <CAN />

    * Appuyez sur `Q`, une fois que **Save configuration** apparaît, appuyez à nouveau sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de ligne de commande
    * Entrez la commande ci-dessous pour démarrer la compilation, cela prendra un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Une sortie similaire à celle-ci indique que la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Programmation du firmware

    * Avant de programmer le firmware, connectez la carte d'outil avec une ligne de données XT30 à UTOC ou au contrôleur principal flashé avec le firmware bridge. Voici un schéma de raccordement de référence
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36/img/utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en utilisant un multimètre en cas d'absence d'alimentation. Si elle est de `120Ω`, connectez un saut de `120Ω`. Si elle est d'environ `40Ω`, retirez le saut.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36/flash/img/120.webp').default} size="100%" align="center" />
