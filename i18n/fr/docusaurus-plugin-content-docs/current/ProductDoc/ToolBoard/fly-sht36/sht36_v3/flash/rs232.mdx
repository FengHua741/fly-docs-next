# Compilation du firmware RS232

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <RS232 />

    * Sélectionnez **GPIO pins à configurer au démarrage du micro-contrôleur**, entrez : **!gpio5**
    :::avertissement
        Veuillez entrer `!gpio5` en mode de saisie anglaise.
    :::
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/flash/img/rs232.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, puis apparaît **Enregistrer la configuration**, appuyez ensuite sur `Y`
    <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="40%" align="center" />
    * La configuration devrait maintenant être enregistrée et vous êtes revenu à l'interface de ligne de commande
    * Entrez la commande ci-dessous pour commencer la compilation, le processus peut prendre un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte suivant, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating uf2 file out/klipper.uf2
        ```


## Entrer en mode d'écriture

    :::astuce

    L'icône pointée par les flèches correspond au bouton **BOOT**

    :::

    * Le circuit imprimé doit être complètement déconnecté lorsqu'on appuie sur le bouton **BOOT**, puis connectez-le via une ligne de données utilisable avec un câble USB Type-C, et reliez le circuit imprimé à l'interface USB de votre ordinateur de contrôle. Après une bonne connexion, relâchez le bouton **BOOT**
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/flash/img/boot.webp').default} size="80%" align="center" />
