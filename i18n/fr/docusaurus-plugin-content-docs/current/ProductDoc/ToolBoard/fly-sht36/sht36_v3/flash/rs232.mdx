# Compilation du firmware RS232

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <RS232 />

    * Sélectionnez **GPIO pins à définir au démarrage du micro-contrôleur**, entrez : **!gpio5**
    :::warning
        Veuillez entrer `!gpio5` en mode d'entrée anglaise.
    :::
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/flash/img/rs232.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, apparaît **Enregistrer la configuration**, puis appuyez sur `Y`
    <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="40%" align="center" />
    * Vous devriez maintenant avoir enregistré la configuration et être revenu à l'interface de ligne de commande
    * Tapez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="65%" align="center" />
    * Si la sortie finale ressemble au contenu ci-dessous, la compilation a réussi
       <ImageView image={require('@site/docs/General/dfu-flash/img/make_uf2_finish.webp').default} size="65%" align="center" />




## Entrer en mode de flashage

    :::tip

    La flèche indique le bouton **BOOT**

    :::

    * Avec la carte outil complètement débranchée, maintenez le bouton **BOOT**, connectez-la à l'aide d'un câble de données TypeC avec fonctionnalité de transfert de données, au port USB de votre ordinateur hôte, une fois correctement connectée, relâchez le bouton **BOOT**
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/flash/img/boot.webp').default} size="80%" align="center" />
