# Utilisation d'Eddy

    :::info
    * Lors de l'installation d'Eddy, la position du bas doit être au moins à 1-2 mm au-dessus de la buse
    * Lors de l'étalonnage de la fréquence d'Eddy, le PCB de la bobine doit être éloigné du lit chauffant d'au moins 5 mm
    * Veuillez ne pas tester le lit chauffant à température ambiante et ne pas l'allumer pendant le test

    :::

## Configuration de référence et instructions d'utilisation

##### Ajoutez la configuration suivante dans le fichier de configuration `printer.cfg` pour le débogage d'Eddy

    ### Configuration de l'analyse du lit

        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 43
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

        * Veuillez noter qu'actuellement, il n'y a pas de module externe 1612 disponible, donc cette configuration n'est pas utilisable pour le moment
        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 42
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

    ### Configuration de la compensation de température

        ```cfg
        [temperature_probe fly_eddy_probe]
        sensor_type:Generic 3950
        sensor_pin:SHT36:gpio28
        ```

    ## Commandes de nivellement du lit
    
        * Ajoutez la macro suivante dans le fichier de configuration `printer.cfg`
            ```bash
            [gcode_macro BED_MESH_CALIBRATE]
             rename_existing: _BED_MESH_CALIBRATE
             gcode: 
                   _BED_MESH_CALIBRATE METHOD=rapid_scan {rawparams}
            ```
        * Pour les commandes de nivellement du lit en réseau, assurez-vous que Klipper peut utiliser la fonction d'exclusion d'objet, il est recommandé de l'ajouter au début de la macro
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan adaptive=1 
            ```

### Configuration de bed_mesh

    * Le paramètre `horizontal_move_z` dans `[bed_mesh]`, il est recommandé de le régler à 2 pour que Eddy soit le plus proche possible de la plateforme lors de l'analyse

    ```cfg
    [bed_mesh]
    horizontal_move_z: 2
    ```
## Macro anti-collision

    ### Optimisation de l'analyse du lit avec `[quad_gantry_level]`
        * Si vous avez configuré `[quad_gantry_level]`, vous pouvez ajouter la macro ci-dessous à la configuration, sinon ce n'est pas nécessaire
        * Cette macro réduit la hauteur de détection en fonction de la différence de hauteur de la gantry
            ```cfg
            [gcode_macro QUAD_GANTRY_LEVEL]
            rename_existing: _QUAD_GANTRY_LEVEL
            gcode:
                SAVE_GCODE_STATE NAME=STATE_QGL
                BED_MESH_CLEAR
                {% if not printer.quad_gantry_level.applied %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_QGL
            ```

    ### Optimisation de l'analyse du lit avec `[z_tilt]`

        * Si vous avez configuré `[z_tilt]`, vous pouvez ajouter la macro ci-dessous à la configuration, sinon ce n'est pas nécessaire
        * Cette macro réduit la hauteur de détection en fonction de la différence de hauteur de la gantry
            ```cfg
            [gcode_macro Z_TILT_ADJUST]
            rename_existing: _Z_TILT_ADJUST
            gcode:
                SAVE_GCODE_STATE NAME=STATE_Z_TILT
                BED_MESH_CLEAR
                {% if not printer.z_tilt.applied %}
                _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _Z_TILT_ADJUST horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_Z_TILT
            ```

    ### Macro d'optimisation de l'analyse du lit pour éviter les interférences du lit chauffant avec une grande probabilité

        * Si vous utilisez un lit chauffant AC de grande puissance (plus de 500W)
        * La macro ci-dessous éteint le lit chauffant pendant l'analyse et le rallume après son achèvement
            ```cfg
            [gcode_macro BED_MESH_CALIBRATE]
            rename_existing: _BED_MESH_CALIBRATE
            gcode:
                {% set TARGET_TEMP = printer.heater_bed.target %}
                M140 S0
                _BED_MESH_CALIBRATE {rawparams}
                M140 S{TARGET_TEMP}
            ```

   # Étapes de débogage : 
   ## Première étape : Modification de la fréquence

    ### Modification de la fréquence par défaut de ldc1612
    Cette méthode est uniquement utilisée pour sht36-v3/SB2040-v3 ou SB2040-Pro-v3
    Vous pouvez utiliser l'outil Mobaxterm pour exécuter les commandes
    
    Téléchargez l'outil requis <Button variant="contained" disableElevation href="/docs/ResDownload/auxiliary_software/">Mobaxterm</Button>
    
    [Pour une utilisation détaillée, veuillez vous référer à](/docs/ProductDoc/SBC/fly-pi-v2/ssh)


    Exécutez la première commande et appuyez sur Entrée

    ```cfg
sed -i 's/LDC1612_FREQ = 12000000/LDC1612_FREQ = 40000000/g' ~/klipper/klippy/extras/ldc1612.py
    ```

    Exécutez la deuxième commande et appuyez sur Entrée
  ```cfg
grep "LDC1612_FREQ = 40000000" ~/klipper/klippy/extras/ldc1612.py
  ```
    Si le remplacement a réussi, une ligne contenant `LDC1612_FREQ = 40000000` sera retournée. Si rien n'est retourné, cela signifie que le remplacement n'a pas réussi.
    Ou utilisez cette commande pour vérifier si le remplacement a réussi
    <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/1612.webp').default} size="110%" align="left" />
```cfg
sed -n '10,15p' ~/klipper/klippy/extras/ldc1612.py
```
   <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/ldc161240mhz.webp').default} size="30%" align="left" />

   Remarque : Après la modification, utilisez cette commande pour redémarrer le service Klipper et recalibrer la fréquence d'Eddy
   ```cfg
   systemctl restart klipper
   ```
   Commande de calibration de la fréquence d'Eddy
   ```cfg
   PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
   ```
