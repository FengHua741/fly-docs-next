# Utilisation d'Eddy

    :::info

    * Veuillez noter que la numérisation du lit doit utiliser une limite Z séparée, il n'est pas recommandé d'utiliser la numérisation du lit comme limite Z
    * Lors de l'installation d'Eddy, la position de la buse doit être au moins à 1-2 mm au-dessus de la buse
    * Lors de l'étalonnage de la fréquence d'Eddy, le PCB de la bobine doit être à au moins 5 mm du lit chauffant
    * Veuillez ne pas tester avec le lit chauffant à température ambiante, et ne pas allumer le lit chauffant pendant le test

    :::

## Configuration de référence et instructions d'utilisation
   
    ### Modification de la fréquence par défaut de ldc16121612
    Cette méthode est uniquement destinée à sht36-v3/SB2040-v3 ou SB2040-Pro-v3
    Utilisez l'outil Mobaxterm pour exécuter les commandes
    
    Téléchargez l'outil <Button variant="contained" disableElevation href="/docs/ResDownload/auxiliary_software/">Mobaxterm</Button> requis
    
    [Pour plus de détails, consultez](/docs/ProductDoc/SBC/fly-pi-v2/ssh)


    Exécutez la première commande et appuyez sur Entrée

    ```cfg
sed -i 's/LDC1612_FREQ = 12000000/LDC1612_FREQ = 40000000/g' ~/klipper/klippy/extras/ldc1612.py
    ```

    Exécutez la deuxième commande et appuyez sur Entrée
  ```cfg
grep "LDC1612_FREQ = 40000000" ~/klipper/klippy/extras/ldc1612.py
  ```
    Si le remplacement réussit, une ligne contenant `LDC1612_FREQ = 40000000` sera retournée, ce qui signifie que le remplacement a été effectué avec succès. Si aucune ligne n'est retournée, cela signifie que le remplacement n'a pas réussi.
    Ou utilisez cette commande pour vérifier si le remplacement a réussi
    <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/1612.webp').default} size="110%" align="left" />
```cfg
sed -n '10,15p' ~/klipper/klippy/extras/ldc1612.py
```
   <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/ldc161240mhz.webp').default} size="30%" align="left" />

   Conseil : Après la modification, veuillez utiliser cette commande pour redémarrer le service klipper et réexécuter l'étalonnage de la fréquence d'Eddy
   ```cfg
   systemctl restart klipper
   ```
   Commande d'étalonnage de la fréquence d'Eddy
   ```cfg
   PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
   ```


    ### Configuration de la numérisation du lit

        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 43
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

        * Veuillez noter qu'il n'y a actuellement aucun module externe 1612 disponible, donc cette configuration ne peut pas être utilisée pour le moment
        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 42
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

    ### Configuration de la compensation de température

        ```cfg
        [temperature_probe fly_eddy_probe]
        sensor_type:Generic 3950
        sensor_pin:SHT36:gpio28
        ```

### Configuration de bed_mesh

    * Le paramètre `horizontal_move_z` dans [bed_mesh], il est recommandé de le régler à 2, pour que Eddy soit le plus près possible du plateau lors de la numérisation

    ```cfg
    [bed_mesh]
    horizontal_move_z: 2
    ```
