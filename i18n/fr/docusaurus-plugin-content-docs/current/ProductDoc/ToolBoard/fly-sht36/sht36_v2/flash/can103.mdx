# Compilation du firmware CAN 103

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez **GPIO pins to set at micro-controller startup**, entrez : !PC13, veuillez noter que c'est en majuscules
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/103.webp').default} size="100%" align="center" />
    * Appuyez sur `Q`, apparaît **Save configuration**, appuyez ensuite sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale contient les lignes suivantes, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flasher le firmware

    * Avant de flasher le firmware, connectez la carte de la boîte à outils au UTOC ou à la carte mère avec une image de firmware bridge, voici un schéma de câblage de référence
    * Vérifiez que les commutateurs sont bien positionnés
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en coupant l'alimentation, si elle est de `120Ω`, branchez une résistance de `120Ω`, si elle est d'environ `40Ω`, retirez la résistance
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/120.webp').default} size="50%" align="center" />
