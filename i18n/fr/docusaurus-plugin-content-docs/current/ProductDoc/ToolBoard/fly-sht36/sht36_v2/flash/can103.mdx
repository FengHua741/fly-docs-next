# Compilation du firmware CAN version 103

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez **GPIO pins to set at micro-controller startup**, entrez : !PC13, veillez à ce que ce soit en majuscules
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/103.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, apparaît **Save configuration**, appuyez alors sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de commande
    * Entrez la commande ci-dessous pour commencer la compilation, cela peut prendre un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale est similaire au texte suivant, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

    <LS />

## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte d'outils à UTOC ou à une carte mère flashed avec le firmware bridge en utilisant un câble XT30, voir le schéma de raccordement ci-dessous
    * Vérifiez que les interrupteurs sont positionnés correctement
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en mesure avec un multimètre en cas d'arrêt de courant, si elle est de `120Ω`, connectez une résistance de `120Ω`, si elle est d'environ `40Ω`, retirez la résistance
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/120.webp').default} size="50%" align="center" />

    <CANFLASH />
