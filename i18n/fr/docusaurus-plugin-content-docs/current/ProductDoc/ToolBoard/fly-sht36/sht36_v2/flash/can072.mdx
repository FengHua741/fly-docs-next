# Compilation du firmware CAN version 072

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez **GPIO pins to set at micro-controller startup**, entrez : !PC13 (remarque : en majuscules)
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/072.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, une fenêtre **Save configuration** apparaît, appuyez ensuite sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de commande
    * Entrez la commande ci-dessous pour démarrer la compilation, le processus peut prendre un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte suivant, la compilation a réussi :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

    <LS />


## Écriture du firmware

    * Avant d'écrire le firmware, branchez la carte via une ligne XT30 sur UTOC ou une carte mère flashée avec le firmware bridge. Voici un schéma de raccordement de référence
    * Vérifiez que les interrupteurs sont bien positionnés correctement
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en utilisant un multimètre dans une situation débranchée. Si elle est de `120Ω`, connectez une résistance de `120Ω`. Si elle est d'environ `40Ω`, retirez la résistance.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/120.webp').default} size="50%" align="center" />
