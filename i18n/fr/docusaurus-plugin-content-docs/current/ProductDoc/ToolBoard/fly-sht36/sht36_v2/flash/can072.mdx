# Compilation du firmware CAN 072

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez **GPIO pins to set at micro-controller startup**, entrez : !PC13, veuillez noter que c'est en majuscules
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/072.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, une fenêtre **Save configuration** apparaît, appuyez ensuite sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si le message suivant s'affiche, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Création du fichier hex out/klipper.bin
        ```

## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte de l'outil à UTOC ou à une carte mère avec le firmware de pont utilisant un câble XT30. Voici un schéma de raccordement de référence
    * Vérifiez si les dip-switches sont positionnés correctement
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` lorsqu'il n'y a pas d'alimentation électrique. Si elle est de `120Ω`, vous devez connecter une jumelle de `120Ω`. Si elle est d'environ `40Ω`, vous devez retirer la jumelle.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/120.webp').default} size="50%" align="center" />
