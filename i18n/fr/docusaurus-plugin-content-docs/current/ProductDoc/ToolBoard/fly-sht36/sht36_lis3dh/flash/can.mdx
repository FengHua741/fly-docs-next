## Démarrer la compilation

    <SSH />

    <CAN />

    * Sélectionnez les broches GPIO à configurer au démarrage du micro-contrôleur, entrez : `!gpio20`. Veuillez noter que `!` est en majuscule.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur `Q`, apparaîtra **Enregistrer la configuration**, puis appuyez à nouveau sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de ligne de commande
    * Entrez la commande ci-dessous pour démarrer la compilation, cela prendra un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble à ceci, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Création du fichier binaire out/klipper.bin
        ```

## Programme du firmware

    * Avant de programmer le firmware, connectez la carte outil à UTOC ou à une carte mère flashée avec le firmware bridge en utilisant un câble XT30. Ci-dessous se trouve un schéma de câblage de référence
    * Vérifiez que les commutateurs sont correctement positionnés
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en coupant le courant avec un multimètre. Si elle est de `120Ω`, branchez une résistance de `120Ω`. Si elle est d'environ `40Ω`, retirez la résistance
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/120.webp').default} size="50%" align="center" />
