# CAN 固件编译

## 开始编译

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionner les broches GPIO à configurer au démarrage du micro-contrôleur, entrez : !gpio20 Veuillez noter que le "!" est en majuscule
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, une fenêtre **Save configuration** apparaît, appuyez ensuite sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte ci-dessous, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Brûler le firmware

    * Avant de brûler le firmware, connectez la carte de l'outil à l'UTOC ou à la carte mère flashée avec le firmware de pont en utilisant un câble XT30. Voici un schéma de câblage de référence
    * Vérifiez que les commutateurs sont positionnés correctement
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en coupant l'alimentation, si c'est `120Ω`, vous devez connecter une résistance de `120Ω`, si c'est `40Ω` environ, retirez la résistance
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/120.webp').default} size="50%" align="center" />
