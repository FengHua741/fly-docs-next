# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionner les broches GPIO à configurer au démarrage du microcontrôleur, entrer : !gpio20 (remarque : le point d'exclamation est en majuscule)
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyer sur `Q`, apparaît **Enregistrer la configuration**, puis appuyer sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration doit maintenant être sauvegardée et vous êtes revenu à l'interface de ligne de commande
    * Entrer `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte ci-dessous, la compilation a réussi :

        ```bash
        Linking out/klipper.elf
        Création du fichier hex out/klipper.bin
        ```

    <LS />


## Flashage du firmware

    * Avant de flasher le firmware, connectez le boîtier d'outils à UTOC ou à une carte mère flashed avec le firmware bridge à l'aide d'un câble XT30. Voici un schéma de raccordement de référence
    * Vérifiez que les interrupteurs sont positionnés correctement
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` lorsqu'il n'y a pas d'alimentation. Si elle est de `120Ω`, ajoutez une résistance de `120Ω`. Si elle est d'environ `40Ω`, retirez la résistance
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/120.webp').default} size="50%" align="center" />

    <CANFLASH />
