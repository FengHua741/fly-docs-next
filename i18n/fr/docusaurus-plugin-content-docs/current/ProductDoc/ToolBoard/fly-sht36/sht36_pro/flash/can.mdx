# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez les broches GPIO à configurer au démarrage du microcontrôleur, entrez : !gpio13 Veuillez noter que ! est en majuscule
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, apparaît **Save configuration**, appuyez ensuite sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte ci-dessous, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Création du fichier hex out/klipper.bin
        ```

## Flash du firmware

    * Avant de flasher le firmware, connectez le panneau d'outils à l'UTOC ou à la carte mère avec le firmware bridge installé à l'aide d'un câble XT30, voir ci-dessous pour le schéma de câblage
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/sht36pro_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en coupant le courant, si c'est `120Ω`, connectez une résistance de `120Ω`, si c'est `40Ω`, retirez la résistance
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/120.webp').default} size="80%" align="center" />
