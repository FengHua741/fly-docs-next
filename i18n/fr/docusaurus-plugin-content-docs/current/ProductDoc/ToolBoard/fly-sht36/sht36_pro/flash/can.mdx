# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez les broches GPIO à configurer au démarrage du micro-contrôleur, entrez : !gpio13 (remarque : c'est en majuscule)
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur `Q`, apparaît **Save configuration**, puis appuyez à nouveau sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de commande
    * Entrez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale est la suivante, alors la compilation a réussi :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```
    <LS />


## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte outil à UTOC ou à une carte mère avec le firmware bridge via une ligne XT30. Voici un schéma de raccordement de référence
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/sht36pro_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en utilisant un multimètre en cas de coupure de courant. Si elle est de `120Ω`, connectez une jumelle de `120Ω`. Si elle est d'environ `40Ω`, retirez la jumelle
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/120.webp').default} size="80%" align="center" />

    <CANFLASH />
