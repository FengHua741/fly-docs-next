# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionner les broches GPIO à configurer au démarrage du micro-contrôleur, entrer : `!gpio13`. Veuillez noter que `!` est en majuscule.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyer sur `Q`, apparaît **Enregistrer la configuration**, puis appuyer sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir enregistré la configuration et être revenu à l'interface de commande
    * Entrer la commande suivante pour commencer la compilation, cela peut prendre un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale contient les lignes ci-dessous, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating bin file out/klipper.bin
        ```
    <LS />


## Écriture du firmware

    * Avant d'écrire le firmware, connectez la carte outil à UTOC ou à une carte mère flashée avec le firmware pont via un câble XT30. Voici un schéma de raccordement de référence
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/sht36pro_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en utilisant un multimètre en position hors tension. Si elle est de `120Ω`, connectez une jumelle de `120Ω`. Si elle est d'environ `40Ω`, retirez la jumelle
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/120.webp').default} size="80%" align="center" />

    <CANFLASH />
