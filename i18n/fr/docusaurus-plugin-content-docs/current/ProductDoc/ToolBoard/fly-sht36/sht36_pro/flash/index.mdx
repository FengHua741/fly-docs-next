# Explications sur la programmation de la mise à jour du micrologiciel

    * SHT36 PRO peut utiliser le mode de connexion CAN pour se connecter à l'ordinateur hôte
    * CAN doit être connecté en utilisant `XT30(2+2)-F`
    * Ci-dessous est le style du câble de données `XT30(2+2)-F`
        <ImageView image={require('@site/docs/General/get-id/img/xt30.webp').default} size="80%" align="left" />

:::avertissement
* La programmation de la mise à jour du micrologiciel de la carte mère doit être effectuée via Katapult pour programmer Klipper
* Si vous êtes sûr que la carte mère est déjà programmée avec Katapult, il n'est pas nécessaire d'exécuter la dernière étape de programmation du micrologiciel Katapult, car il ne s'agit que d'une précaution au cas où la programmation du micrologiciel Katapult échouerait
* Vous pouvez appuyer rapidement deux fois sur le bouton de réinitialisation pour voir si une LED clignote sur la carte mère. Si c'est le cas, cela signifie que la carte mère est déjà dans le micrologiciel Katapult
:::

## Compilation du micrologiciel CAN

    * SHT36 PRO ne peut utiliser que le mode de connexion CAN pour se connecter à l'ordinateur hôte
    * <Button variant="contained" disableElevation href="./can">Programmation de la mise à jour du micrologiciel CAN</Button>
        * Avantages : Si vous avez installé le micrologiciel Katapult, vous pouvez mettre à jour le micrologiciel sans toucher la carte mère et cela prend en charge les connexions en bus
        * Inconvénients : Cela peut entraîner des erreurs comme des retards de positionnement si vous utilisez KlipperScreen ensemble
        * Conseils : Si vous devez utiliser CAN, assurez-vous que l'ordinateur hôte supporte CAN et qu'il faut un UTOC ou une carte mère avec un micrologiciel de pont installé
