# Compilation de la firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Choisir les broches GPIO à configurer au démarrage du microcontrôleur, entrer : gpio17
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyer sur la touche `Q`, apparaît **Save configuration**, puis appuyer sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration est maintenant sauvegardée et vous êtes revenu à l'interface de ligne de commande
    * Entrez la commande ci-dessous pour commencer la compilation, cela prend un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble à ceci, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating bin file out/klipper.bin
        ```

## Programme de la firmware

    * Avant de programmer la firmware, connectez le boîtier outil à UTOC ou à une carte mère flashée avec la firmware bridge à l'aide d'un câble XT30, voir ci-dessous le schéma de raccordement
    * Vérifiez que les commutateurs sont bien positionnés
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en coupant le courant (si elle est de `120Ω`, connectez une résistance de `120Ω`, si elle est d'environ `40Ω`, retirez la résistance)
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/120.webp').default} size="100%" align="center" />
