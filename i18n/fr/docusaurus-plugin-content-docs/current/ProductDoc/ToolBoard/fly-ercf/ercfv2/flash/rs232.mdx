# Compilation du firmware RS232

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <RS232 />

    * Sélectionner les broches GPIO à configurer au démarrage du microcontrôleur, entrer : gpio17
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyer sur la touche `Q`, apparaît **Enregistrer la configuration**, puis appuyer sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration devrait maintenant être enregistrée et vous êtes revenu à l'interface de ligne de commande
    * Entrer `make -j4` pour commencer la compilation, cela peut prendre un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale est similaire à ce qui suit, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Création du fichier uf2 out/klipper.uf2
        ```


## Entrer en mode de mise à jour

    :::tip

    Le cercle indique le bouton `BOOT`

    :::

    * Pour entrer en DFU, il faut que la carte mère soit complètement débranchée, maintenir enfoncée la touche BOOT et connecter ensuite le câble de données à la fois à la carte mère et à l'hôte
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/boot.webp').default} size="100%" align="center" />
