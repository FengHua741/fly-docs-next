# Compilation du firmware RS232

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <RS232 />

    * Choisir les broches GPIO à configurer au démarrage du microcontrôleur, entrée : gpio17
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyer sur la touche `Q`, puis apparaît **Sauvegarder la configuration**, appuyer ensuite sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration est maintenant sauvegardée et vous êtes revenu à l'interface de ligne de commande
    * Entrer la commande suivante pour commencer la compilation, cela peut prendre un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble à ceci, la compilation a réussi :

        ```bash
        Linking out/klipper.elf
        Creating uf2 file out/klipper.uf2
        ```


## Passer en mode de programmation

    :::tip

    Le cercle indique le bouton **BOOT**

    :::

    * Pour entrer en DFU, il faut que la carte mère soit complètement débranchée. Maintenir le bouton BOOT enfoncé puis connecter un câble de transmission de données entre la carte mère et l'ordinateur
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/boot.webp').default} size="100%" align="center" />
