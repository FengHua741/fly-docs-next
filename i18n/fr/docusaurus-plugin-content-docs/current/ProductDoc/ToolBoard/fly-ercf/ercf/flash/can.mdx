# CAN 固件编译

## 开始编译

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionner les broches GPIO à configurer au démarrage du microcontrôleur, entrez : gpio17
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercf/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, apparaît **Enregistrer la configuration**, appuyez alors sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration devrait maintenant être enregistrée et vous êtes revenu à l'interface de ligne de commande
    * Tapez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale contient le texte suivant, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Écrire le firmware

    * Avant d'écrire le firmware, connectez le PCB au UTOC ou à la carte mère flashée avec le firmware de pont en utilisant un câble XT30, voir ci-dessous pour un schéma de câblage
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` en utilisant un multimètre en mode débranché, si elle est de `120Ω`, connectez une liaison de `120Ω`, si elle est d'environ `40Ω`, retirez la liaison
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercf/flash/img/120.webp').default} size="50%" align="center" />
