# Compilation du firmware CAN

## Démarrer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Choisir les broches GPIO à configurer au démarrage du microcontrôleur, entrer : gpio17
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercf/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyer sur la touche `Q`, apparaît **Enregistrer la configuration**, appuyer ensuite sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration est maintenant enregistrée et vous êtes revenu à l'interface de commande
    * Entrer la commande suivante pour démarrer la compilation, cela prend un certain temps
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte ci-dessous, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating bin file out/klipper.bin
        ```

## Flasher le firmware

    * Avant de flasher le firmware, connectez le circuit de l'outil via une ligne de données XT30 à UTOC ou à la carte mère avec un firmware bridge, voici un schéma de raccordement de référence
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * Vérifiez que la résistance entre CAN H et CAN L est d'environ `60Ω` lorsqu'il n'y a pas d'alimentation, si c'est `120Ω` ajoutez une jumelle de `120Ω`, si c'est autour de `40Ω` retirez la jumelle
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercf/flash/img/120.webp').default} size="50%" align="center" />
