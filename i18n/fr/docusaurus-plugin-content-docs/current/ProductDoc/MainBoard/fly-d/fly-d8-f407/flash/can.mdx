## Vérification du mode de flashage

    * Assurez-vous que l'ordinateur principal est correctement connecté à Internet et que le câble de communication entre l'ordinateur principal et l'ordinateur secondaire a la fonctionnalité de transmission de données.
    * Connectez-vous via SSH à l'ordinateur principal, puis entrez `lsusb` et appuyez sur Entrée. Si l'information entourée dans les images ci-dessous apparaît, vous pouvez passer à l'étape suivante.
        * Si aucune information n'est retournée, il s'agit d'un problème système de l'ordinateur principal, nous ne pouvons rien faire. Vous devez changer de système fonctionnel ou remplacer l'ordinateur principal.
        * Si la commande `lsusb` n'est pas reconnue, exécutez cette commande pour l'installer :
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" /> 
    * L'ID affiché dans le cadre rouge peut être utilisé comme référence mais n'est pas **fixe**, l'indication `OpenMoko, Inc. stm32f407xx` avec le suffixe `XX` signifie qu'il est variable.
        * Certains ordinateurs principaux peuvent afficher des informations incomplètes ou ne pas afficher du tout en raison de problèmes système.
