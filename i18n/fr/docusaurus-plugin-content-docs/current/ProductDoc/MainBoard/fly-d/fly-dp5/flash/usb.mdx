    * Appuyez sur `Q`, apparaît **Save configuration**, puis appuyez à nouveau sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Vous devriez maintenant avoir sauvegardé la configuration et être revenu à l'interface de ligne de commande
    * Tapez `make -j4` pour commencer la compilation, cela prend un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte suivant, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Vérifier si le mode de programmation est activé

    * Assurez-vous que l'ordinateur hôte peut se connecter correctement à Internet et que le câble de connexion entre l'ordinateur hôte et l'hôte distant possède la fonctionnalité de transmission de données.
    * Connectez-vous via SSH à l'ordinateur hôte et tapez `lsusb` puis appuyez sur Entrée. Si vous voyez les informations encerclées dans l'image ci-dessous, vous pouvez passer à l'étape suivante.
        * Si aucune information n'est retournée, il s'agit d'un problème avec le système de l'ordinateur hôte. Nous ne pouvons rien faire à ce sujet, vous devez changer de système ou remplacer l'ordinateur hôte.
        * Si l'instruction `lsusb` n'est pas reconnue, vous pouvez installer cette commande avec
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" /> 
    * L'ID affiché dans la zone rouge peut servir de référence mais n'est pas **ID fixe** (il est variable). Le suffixe `stm32f072xx` après `OpenMoko, Inc.` permet de juger que `XX` est variable.
        * Certains ordinateurs hôtes peuvent afficher des informations incomplètes ou ne pas afficher du tout en raison de problèmes liés au système.
