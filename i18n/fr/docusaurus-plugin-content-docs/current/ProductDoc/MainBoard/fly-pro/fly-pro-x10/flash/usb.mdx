    * Appuyez sur `Q`, puis apparaît **Save configuration**, appuyez ensuite sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration doit maintenant être sauvegardée et vous êtes revenu à l'interface de ligne de commande
    * Entrez `make -j4` pour démarrer la compilation, cela peut prendre un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale est similaire au texte ci-dessous, la compilation a réussi :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Vérifier si le mode de programmation est activé

    * Assurez-vous que l'ordinateur hôte est connecté correctement à Internet et que le câble de connexion entre l'ordinateur hôte et l'hôte distant possède une fonctionnalité de transmission de données.
    * Connectez-vous via SSH à l'ordinateur hôte et entrez `lsusb` puis appuyez sur Entrée. Si vous voyez les informations encerclées dans l'image ci-dessous, vous pouvez passer à l'étape suivante.
        * Si aucune information n'est retournée, il s'agit d'un problème du système de l'ordinateur hôte, nous ne pouvons rien faire. Vous devez changer pour un système fonctionnel ou remplacer l'ordinateur hôte.
        * Si l'instruction `lsusb` n'existe pas, vous pouvez installer cette commande en exécutant :
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" /> 
    * L'ID affiché dans la zone rouge peut être utilisé comme référence mais il n'est pas fixe (`XX` est variable).
        * Certains ordinateurs hôtes peuvent ne pas afficher correctement ou ne pas afficher du tout en raison de problèmes systèmes.
