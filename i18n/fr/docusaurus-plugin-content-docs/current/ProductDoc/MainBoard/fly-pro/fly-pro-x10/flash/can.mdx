    * Appuyez sur la touche `Q`, une fois que le message **Save configuration** apparaît, appuyez à nouveau sur `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration doit maintenant être sauvegardée et vous devez être revenu à l'interface de ligne de commande
    * Tapez `make -j4` pour commencer la compilation, cela peut prendre un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la sortie finale ressemble au texte suivant, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```


## Vérification du mode de programmation

    * Assurez-vous que l'ordinateur principal est correctement connecté à Internet et que le câble de connexion entre l'ordinateur principal et le périphérique inférieur possède la fonctionnalité de transmission de données.
    * Connectez-vous via SSH à l'ordinateur principal puis tapez `lsusb` et appuyez sur Entrée. Si l'information entourée par les cercles dans l'image ci-dessous apparaît, vous pouvez passer à l'étape suivante.
        * Si aucune information n'est retournée, il s'agit d'un problème système de l'ordinateur principal. Nous ne pouvons rien faire à ce sujet. Vous devez soit changer vers un système fonctionnel, soit remplacer l'ordinateur principal.
        * Si la commande `lsusb` n'est pas reconnue, vous pouvez l'installer en exécutant cette commande :
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" /> 
    * L'ID affiché dans la zone rouge peut être utilisé comme référence mais il n'est pas fixe ; le suffixe `stm32h723xx` après `OpenMoko, Inc.` sert de critère de jugement, où `XX` est variable.
        * Certains ordinateurs principaux peuvent ne pas afficher ou ne pas afficher correctement en raison de problèmes système.
