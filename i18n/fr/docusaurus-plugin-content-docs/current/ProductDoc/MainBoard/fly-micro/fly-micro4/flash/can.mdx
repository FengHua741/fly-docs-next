## Vérification de l'entrée en mode de flash

    * Assurez-vous que l'ordinateur principal est correctement connecté à Internet et que le câble de communication entre l'ordinateur principal et l'ordinateur secondaire possède la fonction de transmission de données.
    * Connectez-vous via SSH à l'ordinateur principal et entrez `lsusb` puis appuyez sur Entrée. Si vous voyez des informations entourées dans l'image ci-dessous, vous pouvez passer à l'étape suivante.
        * Si aucune information n'est retournée, il s'agit d'un problème du système de l'ordinateur principal, nous ne pouvons rien faire à ce sujet. Vous devez remplacer par un système confirmé comme fonctionnant ou changer d'ordinateur principal.
        * Si l'erreur indique qu'il n'y a pas de commande `lsusb`, exécutez cette commande pour l'installer :
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/ProductDoc/MainBoard/fly-micro/fly-micro4/flash/img/6177.webp').default} size="70%" align="center" /> 
    * Les ID affichés dans la zone rouge peuvent être utilisés comme référence mais ne sont pas fixes, le suffixe `OpenMoko, Inc. rp2040` après `XX` est une indication variable.
        * Certains ordinateurs principaux peuvent ne pas afficher correctement en raison de problèmes liés au système.
