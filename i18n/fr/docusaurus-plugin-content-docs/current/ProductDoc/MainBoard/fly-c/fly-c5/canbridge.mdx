# PONTAGE CAN

Le bus CAN est un protocole de communication en série conçu pour les applications en temps réel, capable de transmettre des signaux sur une paire de fils torsadés. C'est l'un des systèmes de bus de terrain les plus largement utilisés dans le monde. Le protocole CAN est utilisé pour la communication entre divers composants dans les véhicules automobiles, remplacant ainsi des câbles de distribution coûteux et encombrants. La robustesse de ce protocole a permis son extension à d'autres applications automatisées et industrielles. Les caractéristiques du protocole CAN incluent des communications série complètes, un support en temps réel, un taux de transmission allant jusqu'à 1 Mb/s, une adresse à 11 bits ainsi que la capacité de détecter les erreurs.
Dans les imprimantes 3D, le bus CAN est utilisé pour réduire le nombre de fils reliant la tête d'impression à la carte mère. Alors qu'autrefois il fallait plusieurs dizaines de fils, avec le bus CAN, cela ne nécessite que quatre fils, ce qui réduit considérablement le nombre de fils et facilite la mise en place des câblages.
Ce chapitre donne une vue d'ensemble simplifiée de l'utilisation du pontage CAN.

## Avertissements Importants

    :::danger À savoir

    * Le taux de CAN par défaut du système FLY est `1M`, il est recommandé que les cartes outils et la carte mère soient également flashées avec une version de firmware à `1M`.
    * La carte outil FLY est fournie avec une version de firmware `Katapult` flashée par défaut, avec un taux CAN de `1M`.
    * Si la carte outil est sous le firmware `Katapult`, une LED clignotera sur la carte mère, ce qui indique qu'il faut flasher la version `KLIPPER`, sans quoi la connexion ne fonctionnera pas correctement.
    * Veuillez utiliser un multimètre pour vérifier la résistance entre `CANH` et `CANL` pendant qu'il n'y a pas d'alimentation, cette résistance doit être autour de `60Ω`.

    :::
