    <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />

    * Si la sortie ressemble au texte ci-dessous, la compilation a réussi

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Vérifier si le mode de flash est activé
    * Assurez-vous que l'ordinateur hôte peut se connecter correctement à Internet et que le câble de connexion entre l'hôte et l'appareil sous tension possède la fonctionnalité de transmission de données.
    * Connectez-vous via SSH à l'ordinateur hôte puis entrez `lsusb` et appuyez sur Entrée, si l'information entourée dans l'image ci-dessous apparaît, vous pouvez passer à l'étape suivante.
        * Si aucune information n'est retournée, il s'agit d'un problème du système de l'ordinateur hôte, nous ne pouvons rien faire, vous devez changer de système confirmé ou remplacer l'ordinateur hôte.
        * Si l'erreur "commande lsusb introuvable" apparaît, exécutez cette commande pour l'installer
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/ProductDoc/MainBoard/fly-micro/fly-micro4/flash/img/6177.webp').default} size="70%" align="center" /> 
    * Les ID affichés dans la zone rouge peuvent être utilisés comme référence mais ne sont pas des IDs fixes, ils sont variables. Le suffixe `OpenMoko, Inc. rp2040` après `XX` est un indicateur de reconnaissance variable.
        * Certains ordinateurs hôtes peuvent ne pas afficher complètement ou ne pas afficher du tout en raison de problèmes système.
