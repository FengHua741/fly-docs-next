## 플래싱 모드 진입 확인

    * 상위 기기의 네트워크 연결이 정상적인지 확인하고, 하위 기기와 연결된 데이터선이 데이터 전송 기능을 가졌는지 확인하십시오.
    * 상위 기기에 SSH로 연결한 후 `lsusb`를 입력하고 Enter를 누릅니다. 아래 이미지에서 원으로 표시된 정보가 나타난다면 다음 단계로 진행할 수 있습니다.
        * 아무런 정보도 반응이 없다면 상위 기기의 시스템 문제입니다. 해결 방법은 없으므로 정상적인 시스템으로 교체하거나 상위 기기를 바꾸어야 합니다.
        * `lsusb` 명령어가 없다는 오류 메시지가 뜬다면 다음 명령어를 실행하여 설치할 수 있습니다.
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" /> 
    * 빨간색 사각형 내부에 표시된 ID는 참고용이며 고정된 값은 아닙니다. `OpenMoko, Inc. stm32f407xx`의 뒤쪽에 있는 `XX`는 가변적인 의미입니다.
        * 일부 상위 기기는 시스템 문제로 인해 정보가 부족하거나 전혀 표시되지 않을 수 있습니다.
