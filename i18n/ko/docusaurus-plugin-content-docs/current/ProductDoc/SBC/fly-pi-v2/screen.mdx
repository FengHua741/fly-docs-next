## 화면 구성

<Tabs>
    <TabItem value="flyos-fast" label="FLYOS-FAST 시스템 - TFT 화면 구성">

    ### FLYOS-FAST 시스템 - TFT 화면 구성
    1. 브라우저에서 장치의 IP 주소로 접속합니다. 예를 들어: `http://192.168.6.110/`
       <table>
         <tr>
           <td>
             `fluidd`를 아래 그림처럼 설정합니다: `체크 해제` -> `숨겨진 파일 및 폴더 필터링 비활성화`.
             <ImageView image={require('@site/docs/DebugDoc/flyos-fast/img/config1.webp').default} size="100%" align="center" />
           </td>
           <td>
             `mainsail`를 아래 그림처럼 설정합니다: `체크` -> `숨겨진 파일 표시`.
             <ImageView image={require('@site/docs/DebugDoc/flyos-fast/img/config2.webp').default} size="85%" align="center" />
           </td>
         </tr>
       </table>
  
    2. 이때 `.flyos-config` 폴더가 보일 것입니다. 이 폴더 안에는 `sys-config.conf` 파일이 있습니다.

     <ImageView image={require('@site/docs/DebugDoc/flyos-fast/img/config3.webp').default} size="60%" align="center" />
    
    3. `sys-config.conf` 파일을 열고 수정할 구성 항목을 변경한 후 `저장`하고 `닫기`하세요.
    :::warning

        - 저항성 화면은 `fly-tft-v1`으로 설정해야 합니다. `=` 뒤에 **공백이 있으면 안 됩니다!!** 화면 뒷면의 다이얼을 `Resi`로 설정했는지 확인하세요.
        - 전도성 화면은 `fly-tft-v2-c`으로 설정해야 합니다. `=` 뒤에 **공백이 있으면 안 됩니다!!** 화면 뒷면의 다이얼을 `Cap`로 설정했는지 확인하세요.
        - `klipper_screen`은 `1`으로 설정해야 합니다. `=` 뒤에 **공백이 있으면 안 됩니다!!**
        - 주의: `sys-config.conf` 파일의 구성 항목 형식은 `key=value`이며, 임의로 형식을 수정하지 마세요. 그렇지 않으면 구성이 적용되지 않습니다.
    :::
    
    <ImageView image={require('@site/docs/ProductDoc/SBC/fly-pi-v2/img/flyos-fast_tft_conf.webp').default} size="90%" align="center" />
    
    4. 수정 및 저장 후, `시스템 재시작`하면 적용됩니다.
    </TabItem>


    <TabItem value="flyos-armbian" label="FLYOS-Armbian 시스템 - TFT 화면 구성" default>
    ### FLYOS-Armbian 시스템 - TFT 화면 구성
    :::info
    - 상위 컴퓨터에서 WiFi에 연결하고, 상위 컴퓨터의 IP를 얻은 후, 브라우저에서 `IP+:9999`를 입력하여 FLY TOOLS에 접근할 수 있으며, 이를 통해 화면 관련 구성이 가능합니다.
    - 예를 들어, 상위 컴퓨터의 IP 주소가 `192.168.6.110`인 경우, 브라우저에서 `192.168.6.110:9999`를 입력해야 합니다.
    :::
    1. FLY TOOLS를 열고 아래 그림과 같이 단계를 따라 `Klipper Screen 활성화`, `화면 모델 선택`을 하고 오른쪽 하단의 분홍색 버튼을 클릭하여 `설정 저장`한 후, **재부팅 후 적용됩니다!**
      :::warning
      - 저항성 화면 선택: `fly-tft-v1` 선택, 화면 뒷면의 다이얼을 `Resi`로 설정했는지 확인하세요.
      - 전도성 화면 선택: `fly-tft-v2-c` 선택, 화면 뒷면의 다이얼을 `Cap`로 설정했는지 확인하세요.
      :::

    2. 또한 SSH를 통해 명령줄을 사용해 직접 구성할 수도 있습니다 (이 방법은 권장하지 않습니다!)
        * 저항성 화면 구성은 아래 명령어를 사용하여 `fly-tft-v1`로 설정해야 합니다. 화면 뒷면의 다이얼을 `Resi`로 설정했는지 확인하세요.
        ```cfg
        sudo sed '11s/screen=none/screen=fly-tft-v1/' -i /boot/FlyOS-Env.txt
        ```

        * 전도성 화면 구성은 아래 명령어를 사용하여 `fly-tft-v2-r`로 설정해야 합니다. 화면 뒷면의 다이얼을 `Cap`로 설정했는지 확인하세요.
        ```cfg
        sudo sed '11s/screen=none/screen=fly-tft-v2-r/' -i /boot/FlyOS-Env.txt
        ```

        * 설정이 성공적으로 적용되었는지 확인하려면 아래 명령어를 사용하세요.

        ```cfg
        sed -n '9,11p' /boot/FlyOS-Env.txt | grep -v '^#'
        ```

        * 저항성 화면을 전도성 화면으로 변경하려면 아래 명령어를 사용하세요.
        ```cfg
        sudo sed -i '11{/^#/!s/screen=fly-tft-v1/screen=fly-tft-v2-r/}' /boot/FlyOS-Env.txt
        ```
        * 전도성 화면을 저항성 화면으로 변경하려면 아래 명령어를 사용하세요.
        ```cfg
        sudo sed -i '11{/^#/!s/screen=fly-tft-v2-r/screen=fly-tft-v1/}' /boot/FlyOS-Env.txt
        ```
