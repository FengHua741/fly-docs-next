# **BDsensor-m-m** 설치

## 전선을 메인보드의 EXP1 포트에 연결

    * 전선이 충분하지 않으면 포장된 연장선을 사용하세요.
    * BDsensor-m-m의 CKL 및 SDA 선은 보드의 어떤 GPIO 핀에나 연결할 수 있습니다. 또한 BDsensor 케이블을 Bltouch 포트에 직접 연결할 수도 있습니다. 예:

    ```bash
    BLtouch    |    BDsensor-m
    5V       -->     5V
    GND      -->     GND
    S        -->     CLK/SCL    (입력)
    GND      -->     GND
    Zmin     -->     SDA    (입력/출력) 
    ```

    * 메인보드 커넥터의 일부 핀은 MCU의 gpios에 직접 연결되지 않을 수 있습니다(예: 필터 커패시터가 있거나 MOSFET, 다이오드 또는 옵토커플러로 분리되어 있지만 저항이나 저항 풀업/풀다운으로 분리되어 있으면 됩니다). 따라서 BDsensor-m과 함께 사용할 수 없습니다. 그리고 펌웨어가 연결 오류를 보고할 것입니다. 예를 들어:

    * 팬과 히터의 커넥터는 MOSFET으로 분리되어 있습니다.
    * 일부 보드에서는 온도 센서와 엔드스탑/프로브의 커넥터가 일반적으로 필터 커패시터를 통해 GND에 연결됩니다.

## Marlin 펌웨어에 패치 설치

    BD 센서는 Marlin2.1.x에 통합되었습니다(2022.8.27 이후).

    릴리스 버전을 다운로드할 수 있지만, 최신 버그 수정 버전을 다운로드하는 것이 좋습니다: https://github.com/MarlinFirmware/Marlin

    설정 파일과 핀 파일을 변경해야 합니다.

### Configuration.h 편집

1. BD_SENSOR 활성화

    주석 제거

    ```bash
    #define BD_SENSOR`
    #define Z_SAFE_HOMING
    #define BD_SENSOR_PROBE_NO_STOP // 빠른 베드 레벨링을 위해 추가, 노즐 정지 없이
    ```

    `BD_SENSOR_PROBE_NO_STOP`만 해당됩니다.

    최신 마린 버그 수정: https://github.com/MarlinFirmware/Marlin

    설명: https://github.com/MarlinFirmware/Marlin/pull/25847

2. 프로브로 홈 설정

    `Z_MIN_PROBE_USES_Z_MIN_ENDSTOP_PIN`이 비활성화되어 있어야 하며, `USE_PROBE_FOR_Z_HOMING`을 다음과 같이 활성화해야 합니다.

    ```bash
    //#define Z_MIN_PROBE_USES_Z_MIN_ENDSTOP_PIN
    // Force the use of the probe for Z-axis homing
    #define USE_PROBE_FOR_Z_HOMING
    ```



3. 두 번째 Z 홈 속도 감소

    ```bash
    #define Z_PROBE_FEEDRATE_SLOW (Z_PROBE_FEEDRATE_FAST / 16)
    ```

    여기서 우리는 BDsensor-m 프로세스에서 읽은 엔드스탑이 일반 엔드스탑처럼 실시간이 아니기 때문에 범프 홈 속도와 Z 홈 속도를 줄여야 합니다.

    ### Configuration_adv.h 편집

    실시간 조정을 위해 `#define BABYSTEPPING`을 활성화합니다.

    ```bash
    #define HOMING_BUMP_DIVISOR { 2, 2, 8 }       // Re-Bump Speed Divisor (Divides the Homing Feedrate)
    ```

### pins_boardname.h 편집

    BDsensor-m의 SDA와 SCL 핀을 `pins_boardname.h` 파일에 추가하여 설정합니다(예: `pins_PANDA_PI_V29.h`).

        ```bash
        #define  I2C_BD_SDA_PIN    PC6   // SDA 선이 메인보드에 실제로 연결된 번호로 변경하세요.
        #define  I2C_BD_SCL_PIN    PB2   // SLK 선이 메인보드에 실제로 연결된 번호로 변경하세요.
        #define  I2C_BD_DELAY  20      // 기본값은 20이며, [20,50] 범위 내에 있어야 합니다.
        ```

    BLtouch와 같은 자동 베드 레벨링 프로브(G29)를 인쇄 전에 사용하려면 주석을 제거하세요.

        ```bash
        #define AUTO_BED_LEVELING_BILINEAR
        ```

    다음과 같이 값을 편집하세요.

        ```bash
        #define Z_CLEARANCE_DEPLOY_PROBE   0 // Z Clearance for Deploy/Stow
        #define Z_CLEARANCE_BETWEEN_PROBES  1 // Z Clearance between probe points
        #define Z_CLEARANCE_MULTI_PROBE     1 // Z Clearance between multiple probes
        ```



## LCD 화면에 BD 센서 값 표시

    * 상태 표시가 있는 프린터의 경우(gcode M117 지원), LCD12864 또는 일부 uart 화면(예: ender3V2)...

## 보정

    1. 노즐을 청소한 후 콘솔을 통해 Z축을 이동하여 노즐이 베드에 정확히 닿을 때까지 조정하세요(BDsensor-m은 이 위치를 0점으로 사용하므로 z_offset이 필요 없으며 값을 0으로 설정합니다).
    2. gcode 명령 `M102 S-6`을 보내면 프린터는 Z축을 0.1mm씩 천천히 올려 4mm에 도달할 때까지 조정합니다. 센서 설치 전에는 M102 S-6을 실행하지 마세요. 보정 중 프린터 전원을 끄지 마세요. 그렇지 않으면 이전 보정 데이터가 삭제됩니다. 이런 경우 다시 보정하면 됩니다.

    3. `M102 S-5` 명령을 통해 BD 센서가 성공적으로 보정되었는지 확인할 수 있으며, 이는 BD 센서에 저장된 원시 보정 데이터를 반환합니다.

    다음 도구를 사용하여 보정할 수도 있습니다: https://github.com/markniu/Bed_Distance_sensor/raw/new/marlin/BD_Config_Tool_Marlin.zip ![img](https://raw.githubusercontent.com/markniu/Bed_Distance_sensor/main/doc/images/Read.jpg)

    참고: 데이터 값이 1015 또는 > 1010이면 센서 범위를 벗어났음을 의미합니다. 처음 5점(0~0.5mm) 또는 그 이상의 값이 0에서 1000 사이에 있고 증가한 값의 delta가 >=10이면 보정 성공입니다. 위의 그래프처럼 표시됩니다.

    M102 S-5가 반환하는 첫 번째 원시 보정 데이터가 400보다 크면 센서가 너무 높게 설치된 것을 의미하며, 베드에 더 가까운 위치로 재설치해야 합니다. 또한 두 번째 데이터가 첫 번째 데이터보다 10 이상 커야 합니다.

## 테스트 및 인쇄

    메뉴 베드 레벨링

    자동 베드 레벨링

## 베드를 자동으로 레벨링하는 두 가지 방법:

    **1. M102를 사용한 실시간 레벨링**

        gcode 명령을 보내거나 gcode 파일에 추가하여 이 자동 레벨링을 쉽게 활성화/비활성화할 수 있습니다.

        베드 레벨링을 활성화하려면 프린터 설정의 "시작 G 코드" 부분에 G102(모든 축 홈) G 코드 바로 아래 M28 G 코드를 추가하세요. 예를 들어, G28 아래에 이 코드가 있으면 Z 높이가 0.2mm 이하에서만 베드 레벨링이 수행됩니다. `M102 S2`

        BD 센서를 사용하여 베드 레벨링을 비활성화하려면 `M102 S0` `G28` `M18`를 보내거나 사용하세요.

        ```bash
        M102   S-1     // Read sensor information, and we can use this for connection checking.
        M102   S-2     // Read current distance value
        M102   S-5     // Read raw Calibrate data
        M102   S-6     // Start Calibrate, before that make sure the nozzle is just touched the bed, and then the printer restarted. Don't home z axis before this.
        M102   S4      // Set the adjustable Z height value, e.g. M102 S4 means it will do adjusting while the Z height <=0.4mm, disable it by M102 S0.
        ```

    **2. G29 자동 베드 레벨링**

        G29의 BLtouch와 같은 또 다른 자동 베드 레벨링 방법입니다. G29 아래에 한 줄만 추가하면 됩니다.

        [설치 비디오](https://www.pandapi3d.com/post/install-bed-distance-sensor-video)

        [크리스 지하실 설치 비디오](https://youtu.be/VLUfvkO2NFc?si=PF_6nhw39KhHBhcj)

### Z 엔드스탑 확인 `M119`

    이 단계를 확인하기 전에 Z 축을 홈하지 마세요. 그렇지 않으면 노즐이 프린터 베드에 닿을 수 있습니다.

    이것은 M119 명령(엔드스탑 상태 보고)을 보낸 후의 반환 메시지입니다.

    ```bash
    Send: M119
    Recv: x:open y:open z:open
    ```

    z min이 열리지 않으면 설정을 확인하세요. `#define Z_MAX_ENDSTOP_HIT_STATE HIGH`

    - Z 모터가 꺼져/잠금 해제되었는지 확인하세요.
    - 손으로 Z 축을 아래로 이동하여 노즐이 베드에 닿을 때까지 조정하세요.
    - `M102 S-2`를 보내면 반환 값이 0.00mm여야 하며, 다시 M119를 보내면 z 엔드스탑이 트리거되었음을 볼 수 있습니다.

    ```bash
    Send: M119
    Recv: x:open y:open z:TRIGGERED
    ```

    ### 연결 확인

    `M102 S-1`로 연결을 확인하세요. 이것은 반환 메시지의 예입니다. 연결과 전선 순서가 정확한지 확인하세요.

    ```bash
    Send: M102 S-1
    Recv: V1.0 pandapi3d.com
    ```
