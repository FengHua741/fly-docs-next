# **BDsensor-m** 설치

## 센서 케이블을 메인보드의 EXP1 포트에 연결
    * 센서 케이블 길이가 충분하지 않으면 패키지 내 연장 케이블을 사용하세요.
    * BDsensor-m의 CKL과 SDA 선은 회로판의 어떤 GPIO 핀에도 연결할 수 있습니다. 또한 BDsensor 케이블을 Bltouch 포트에 직접 연결할 수도 있습니다. 예를 들어:

    ```bash
    BLtouch    |    BDsensor-m
    5V       -->     5V
    GND      -->     GND
    S        -->     CLK/SCL    (입력)
    GND      -->     GND
    Zmin     -->     SDA    (입력/출력) 
    ```

    * 주의: 일부 메인보드 커넥터의 핀은 MCU의 GPIO에 직접 연결되지 않을 수 있습니다(예: 필터 커패시터가 있거나 MOSFET, 다이오드, 옵토커플러로 분리될 수 있으나, 저항이나 풀업/풀다운 저항으로 분리된 경우는 사용 가능). 이러한 경우 `BDsensor-m`과 함께 사용할 수 없으며, 펌웨어는 연결 오류를 보고합니다. 예를 들어:
    * 팬과 히터의 커넥터는 MOSFET으로 분리됩니다.
    * 일부 보드에서 온도 서미스터 및 엔드스톱/프로브용 커넥터는 보통 필터링 커패시터로 GND에 연결됩니다.

## 패치를 klipper 펌웨어에 설치

    * 이전에 수정한 klipper 파일을 버리고 klipper를 업데이트하세요

        ```bash
        cd
        cd ~/klipper
        git checkout .
        git pull
        ```

    * 최신 BDsensor 코드를 클론합니다

        ```bash
        cd && git clone https://github.com/markniu/Bed_Distance_sensor.git
        ```

    * 설치

        ```bash
        cd  ~/Bed_Distance_sensor/klipper/
        ./install_BDsensor.sh
        ```

    * 펌웨어 컴파일

        ```bash
        cd ~/klipper/
        make menuconfig
        make clean
        make
        ```

    * 펌웨어를 BDsensor가 연결된 메인보드에 플래시

## Moonraker를 사용하는 경우, moonraker.conf에 다음 부분을 추가하세요. 이후 웹 페이지나 klipperscreen에서 BDsensor를 한 번 클릭하여 업데이트할 수 있습니다.

    ```bash
    [update_manager BDsensor]
    type: git_repo
    primary_branch: new
    channel: dev
    path: ~/Bed_Distance_sensor
    origin: https://github.com/markniu/Bed_Distance_sensor.git
    install_script: ./klipper/install_BDsensor.sh
    is_system_service: False
    managed_services: klipper
    info_tags:
    desc=Bed Distance Sensor
    ```

## printer.cfg 수정

    * 다음 부분을 **printer.cfg**에 복사하고 `sda_pin`과 `scl_pin`을 편집하세요. 또한 다른 프로브 부분(예: **BLtouch**)을 비활성화하지 마세요. BDsensor를 메인보드나 툴헤드 CAN 모듈에 연결할 수 있습니다.
    * `speed`를 0.8로 변경하세요. 이는 z 틸트 및 PROBE_ACCURACY 명령에만 적용됩니다. 값이 작을수록 정밀도가 높아집니다.
    * Z 축 원점 복귀 시 BDsensor를 리미트 스위치로 사용하려면 `endstop_pin`을 `endstop_pin: probe:z_virtual_endstop`로 변경하세요.
    * **printer.cfg**에 `safe_z_home`가 포함되어 있는지 확인하세요.
    * `bed_mesh`와 `z_tilt` 또는 `quad_gantry_level`의 값을 1로 변경하세요(0.7-1.0mm 권장).
    * **노즐 높이는 `z_adjust:`에서만 설정해야 하며, 양수는 노즐이 베드에 가까워지고, 음수는 멀어지게 합니다. 다른 노즐 높이 조절 설정은 버그가 있을 수 있습니다.**
    * 빠른 스캔을 사용하려면 `no_stop_probe:true` 앞의 #을 제거하세요.
    * 아래는 설정 예시입니다.

        ```cfg
        [BDsensor] 
        scl_pin:PC6  # 서보 신호 포트
        sda_pin:PC3  # 리미트 신호 포트
        delay: 20 # 펄스당 20us, 이 값은 >=20이어야 하며 50 미만이어야 합니다.
        z_offset:0 # 이 `z_offset`는 0으로 설정해야 합니다.
        z_adjust:0.0 # Z 축 조정, z_offset 기능을 대체. -0.3에서 0.3mm 사이
        x_offset: -34
        y_offset: 0
        #no_stop_probe:true # 빠른 프로브를 사용하려면 이 옵션을 활성화하세요.
        position_endstop: 0.8 # Z 축이 이 위치(mm)에서 정지합니다, 권장값은 0.4~1.0
        #speed:0.8 # 이 속도는 z 틸트 및 PROBE_ACCURACY 명령에만 적용됩니다.

        [stepper_z]
        endstop_pin: probe:z_virtual_endstop 
        #position_endstop: 0.5
        homing_speed: 5
        second_homing_speed: 0.8

        [bed_mesh]
        speed: 200
        horizontal_move_z:1
        algorithm: bicubic

        [quad_gantry_level]
        horizontal_move_z:1

        ```

## 설치 후 다음 gcode 명령을 통해 확인하세요

    ```bash
    M102   S-1     # 센서 정보 읽기
    M102   S-2     # 거리 값 하나 읽기
    ```

## 연결 확인

    * **콘솔**을 통해 `M102 S-1`을 보내세요. 다음은 반환된 메시지의 예입니다. 빈칸이나 다른 문자열이 반환되면 연결 및 와이어 순서를 확인하세요.

        ```bash
        Send: M102 S-1
        Recv: V1.0 pandapi3d.com
        ```

## 보정

    * 노즐을 청소하고 콘솔을 통해 Z 축을 이동하여 노즐이 베드에 닿을 때까지 조정하세요(BDsensor-m은 이 위치를 제로 위치로 사용하므로 `z_offset`가 필요하지 않습니다. 이는 [BDsensor-m] 부분의 값이 0인 이유입니다).
    * **콘솔**에서 gcode 명령 `M102 S-6`을 보내세요. 프린터는 Z 축을 천천히 0.1mm씩 위로 이동하여 4mm까지 도달합니다. 센서 설치 전에 M102 S-6을 실행하지 말고, 보정 중에는 프린터 전원을 끄지 마세요. 그렇지 않으면 이전 보정 데이터가 삭제됩니다. 이 경우 다시 보정하면 됩니다.
    * 이후 `M102 S-5`를 통해 BDsensor가 성공적으로 보정되었는지 확인할 수 있습니다. 이는 BDsensor에 저장된 원시 보정 데이터를 반환합니다.

**주의사항**:

* Z 축 원점 복귀 속도는 5가 가장 좋습니다.

* M102 S-5가 반환하는 첫 번째 원시 보정 데이터가 400보다 크면 센서가 너무 높이 설치된 것을 의미합니다. 보다 낮은 위치에 다시 설치해야 하며, 첫 번째 데이터의 권장 값은 100입니다. 두 번째 데이터는 첫 번째 데이터보다 10 이상 큰 값이어야 합니다.

  * FAQ: 보정 데이터가 1로 시작하고, 두 번째 값이 9, 세 번째 값이 24라면 이는 무엇을 의미합니까?

  * 이는 0-0.1mm 사이의 해상도가 9이고, 0.1-0.2mm 사이의 해상도가 15임을 의미합니다. 따라서 첫 번째 해상도 0-0.1mm를 10 이상으로 조정하여 다시 보정하는 것이 좋습니다.

* G28을 실행한 후 또는 `Z_tilt`와 `quad_gantry_level` 명령에 대해 Z 축 높이를 조정하는 것을 잊지 마세요.

* 부분 이름의 대소문자를 정확히 맞춰야 합니다. 그렇지 않으면 klipper는 `Unknown pin chip name 'probe'`를 보고합니다.
