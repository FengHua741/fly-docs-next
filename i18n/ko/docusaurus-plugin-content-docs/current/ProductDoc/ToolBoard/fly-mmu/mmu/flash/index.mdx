# 펌웨어 플래시 설명

    * MMU는 RS232, CAN, USB 세 가지 연결 방식으로 상위 기기와 연결할 수 있습니다.
    * 그 중 RS232와 CAN은 `XT30(2+2)-F`를 사용하여 연결해야 하며, USB는 데이터 전송 기능이 있는 `TYPE-C` 데이터 케이블을 사용해야 합니다.
    * 아래는 `XT30(2+2)-F` 데이터 케이블의 모양입니다.
        <ImageView image={require('@site/docs/General/get-id/img/xt30.webp').default} size="80%" align="left" />

:::경고
* 메인보드 펌웨어 플래시는 Katapult를 통해 Klipper를 플래시해야 합니다.
* 메인보드에 Katapult 펌웨어가 이미 플래시되었다면 마지막 단계에서 Katapult 펌웨어를 플래시하지 않아도 됩니다. 이것은 메인보드의 Katapult 펌웨어 플래시 실패를 방지하기 위한 것입니다.
* 메인보드의 LED가 두 번 빠르게 깜빡이는 것을 확인하면 메인보드가 Katapult 펌웨어로 업데이트되었음을 나타냅니다.
:::

## 어느 종류의 펌웨어가 필요한지 어떻게 판단합니까?

    * MMU는 다음 방법으로 상위 기기에 연결할 수 있습니다.
    * <Button variant="contained" disableElevation href="./can">CAN 펌웨어 플래시</Button>
        * 장점: `Katapult` 펌웨어를 플래시하면 메인보드를 만지지 않고 펌웨어를 업데이트할 수 있으며, 버스 연결을 지원합니다.
        * 단점: `KlipperScreen`과 함께 사용할 경우 리부팅 시간 초과 등의 오류가 발생할 수 있습니다.
        * 주의: CAN을 사용하려면 상위 기기가 CAN을 지원하고, UTOC 또는 브리지 펌웨어가 플래시된 메인보드가 필요합니다.
    * <Button variant="contained" disableElevation href="./rs232">RS232 펌웨어 플래시</Button>
        * 장점: 특수 소프트웨어 없이 사용할 수 있으며, 통신이 안정적이고 신뢰할 수 있습니다.
        * 단점: 펌웨어를 다시 플래시하려면 수동 DFU로 들어가야 하거나, RST 버튼을 두 번 클릭하여 `Katapult`로 들어갈 수 있어야 합니다.
        * 주의: G2T 또는 UTOR와 함께 사용해야 합니다.
    * <Button variant="contained" disableElevation href="./usb">USB 펌웨어 플래시</Button>
        * 장점: `Katapult` 펌웨어를 플래시하면 메인보드를 만지지 않고 펌웨어를 업데이트할 수 있으며, 메모리 카드 또는 DFU로 플래시된 경우 여전히 원래 방법으로 펌웨어를 업데이트해야 합니다. 통신이 안정적이고 신뢰할 수 있습니다.
        * 단점: 여러 USB 장치가 있을 경우 데이터 전송에 영향을 줄 수 있습니다.
