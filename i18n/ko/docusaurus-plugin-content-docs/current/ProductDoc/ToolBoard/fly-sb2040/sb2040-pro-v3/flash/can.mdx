    * **GPIO 핀을 마이크로컨트롤러 시작 시 설정**을 선택하고, 입력: **!gpio5**
    :::warning
        `!gpio5`를 입력할 때 영문 입력 상태여야 합니다.
    :::
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro-v3/flash/img/can.webp').default} size="100%" align="center" />
    * `Q` 키를 누르면 **구성 저장**이 나타나고, 그 다음에 `Y` 키를 누릅니다.
    <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * 이제 구성이 저장되고 명령줄 인터페이스로 돌아왔습니다.
    * `make -j4`를 입력하여 빌드를 시작합니다. 시간이 좀 걸립니다.
    <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * 마지막으로 다음 내용이 출력되면 빌드가 성공한 것입니다.

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## 펌웨어 프래싱

    * 펌웨어 프래싱 전에 XT30 데이터 케이블로 도구 보드를 UTOC 또는 브리지 펌웨어가 깔린 메인보드에 연결합니다. 아래는 참조 배선도입니다.
    * 다이얼 스위치가 적절한 위치에 있는지 확인하십시오.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/sb2040v3_utoc.webp').default} size="80%" align="left" />
    * CAN H와 CAN L의 저항값이 끊김 상태에서 약 `60Ω`인지 확인하십시오. 만약 `120Ω`라면 `120Ω` 다이얼 스위치를 켜야 합니다. 만약 `40Ω` 정도라면 다이얼 스위치를 끄십시오.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/120.webp').default} size="100%" align="center" />
