# 시스템 구성 변경

:::tip 필수 읽기

- 구성 파일의 `부분` 설정은 플래시 후 처음으로 시스템을 시작할 때만 적용됩니다.
- 따라서 플래시 후에 먼저 플러시 가능한 디스크 `FlyOS-Conf`의 구성 파일 `config.txt`를 수정해야 합니다.
- 그런 다음 카드를 보드에 삽입하여 시스템을 시작하세요.

:::

:::warning 주의

- Windows 시스템의 기본 메모장으로 구성 파일을 편집하는 것은 피하십시오. VSCode나 다른 전문 텍스트 편집기를 사용할 수 있습니다.
- 구성 파일에는 중국어 또는 특수 문자를 입력하지 않는 것이 좋습니다.
- 구성 파일을 수정한 후에는 시스템을 재시작해야 적용됩니다.

:::

:::danger

- 구성 파일에서 `######### PLEASE DO NOT EDIT THE FOLLOWING #########` 아래의 내용은 수정하지 마십시오.

:::

## 구성 파일 변경 방법

### 플래시 완료 후 구성 파일 변경

- `처음 시작` 시 적용되는 구성 항목은 플래시 후 전문 텍스트 편집기를 사용하여 플러시 가능한 디스크 `FlyOS-Conf`의 구성 파일 `config.txt`를 열고 수정해야 합니다.

### 시스템이 시작된 후 온라인 구성 변경

- 일반 구성 항목은 시스템 웹 구성 페이지에서 수정하고 시스템을 재시작하면 적용됩니다.
    - 브라우저에서 장치 IP 주소를 열어주세요. 예를 들어: `http://192.168.1.2/`
    - fluidd는 아래 그림 왼쪽에 표시된대로 `비선택` -> `숨겨진 파일 및 폴더 필터링`을 해제합니다.
    - mainsail은 아래 그림 오른쪽에 표시된대로 `선택` -> `숨겨진 파일 표시`를 합니다.

    - 이제 `.flyos-config` 폴더를 볼 수 있으며 이 폴더로 들어가면 `sys-config.conf` 파일이 있습니다.
    - `sys-config.conf` 파일은 플러시 가능한 디스크 `FlyOS-Conf`의 구성 파일 `config.txt`의 소프트 링크입니다.

    - 주의: `sys-config.conf` 파일의 구성 항목 형식은 `키=값`이며 `키: 값`으로 수정하지 마십시오.
    - 수정 및 저장 후에는 시스템을 재시작해야 적용됩니다.

## 구성 항목

### 무선 네트워크 구성

- 다음 구성 항목은 시스템이 `매번 시작`될 때마다 적용됩니다.

| 구성 항목 | 기본값 | 사용 가능한 값 | 설명 |
| :---: | :---: | :--- | :--- |
| `WIFI_SSID` | `빈 값` | 임의의 WIFI 이름 | WIFI 이름, 작성된 경우 시스템 시작 시 우선적으로 연결 |
| `WIFI_PASS` | `빈 값` | 비밀번호 없으면 비워두세요 | WIFI 비밀번호, 작성된 경우 비밀번호로 연결 |

| :---: | :---: | :--- | :--- |

### 시스템 구성

- 다음 구성 항목은 시스템이 `매번 시작`될 때마다 적용됩니다.

| 구성 항목 | 기본값 | 사용 가능한 값 | 설명 |
| :---: | :-------: | :--- | :--- |
| `console` | `serial` | `serial`, `both` | 이 구성 항목은 수정하지 마십시오. `serial`만 유효 |
| `board` | H5: `fly-pi-v2`<br/>H3: `fly-lite2.1` | <details><summary>H5 시리즈 확장</summary>`fly-pi`, `fly-pi-v2`, `fly-gemini-v1`, `fly-gemini-v2`, `fly-gemini-v3`, `fly-c8`</details><br/><details><summary>H3 시리즈 확장</summary>`fly-lite2`, `fly-lite2.1`, `fly-minipad`</details> | 메인보드 모델, 주로 자동 플래시 펌웨어 등에 사용 |
| `shutdown_pin_state` | `1` | `0`, `1` | 종료 버튼의 전압 상태 설정<br/>1: 저전압 트리거<br/>0: 고전압 트리거 |
| `shutdown_pin` | `none` | `PA21`, `PA4`... | 종료 버튼의 트리거 핀 설정, 기본값 `none`은 비활성화 |
| `spidev` | `spidev0.0 spidev1.0` | `spidev0.0`, `spidev0.1`, `spidev1.0`, `spidev1.1` | 활성화할 spidev 포트 설정<br/>여러 개를 활성화하려면 공백으로 구분 |

### Klipper 관련 구성

- 다음 구성 항목은 시스템이 `매번 시작`될 때마다 적용됩니다.

| `moon2uart` | `0` | `0`, `1` | moon2uart를 활성화할지 설정<br/>1: 활성화<br/>0: 비활성화<br/>`printer=klipper`일 때 유효 |
| `m2u_port` | `/dev/ttyS1` | 유효한 시리얼 포트 | moon2uart의 시리얼 포트, `moon2uart=1`일 때 유효 |
