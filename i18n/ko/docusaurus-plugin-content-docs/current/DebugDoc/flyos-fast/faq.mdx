# FlyOS-Fast 자주 묻는 질문

## 질문: 명령을 실행하면 `명령을 찾을 수 없습니다`라는 메시지가 표시됨
- **원인**: 시스템에 해당 명령이 없습니다.
- **해결 방법**: 시스템에는 많은 일반적인 소프트웨어 패키지가 내장되어 있습니다. 필요하다면 다른 패키지를 제출해 주세요. 필요하다고 판단되면 다음 버전에 사전 설치될 예정입니다.

## 질문: `apt update` 명령을 실행할 수 없음
- **원인**: FlyOS-Fast는 가벼운 빠른 시스템으로, apt 패키지 관리 도구를 사용하지 않습니다. 따라서 `apt update` 명령을 실행할 수 없습니다.
- **해결 방법**: 시스템에는 많은 일반적인 소프트웨어 패키지가 내장되어 있습니다. 필요하다면 다른 패키지를 제출해 주세요. 필요하다고 판단되면 다음 버전에 사전 설치될 예정입니다.

## 질문: 번인 명령을 실행하려 하니 `-bash: ~/klippy-env/bin/python: No such file or directory`라는 오류 발생
- **원인**: FlyOS-Fast는 venv 가상 환경을 사용하지 않습니다.
- **해결 방법**: `~/klippy-env/bin/python`를 `python`으로 교체하여 실행하면 됩니다.

## 질문: `fly` 사용자로 로그인할 수 없음
- **원인**: FlyOS-Fast에는 root 사용자만 있으며 모든 작업은 root 사용자에서 수행됩니다.
- **해결 방법**: 시스템에는 `fly` 사용자가 없습니다. root 사용자로 직접 로그인하시면 됩니다.

## 질문: 9999 포트의 fly-tools를 열 수 없음
- **원인**: FlyOS-Fast에는 fly-tools가 설치되지 않았습니다.
- **해결 방법**: fluidd와 mainsail은 모두 ID를 검색하는 기능을 지원합니다. 화면 설정은 [시스템 구성](/docs/DebugDoc/flyos-fast/config)에서 확인할 수 있습니다.

## 질문: 화면에 로고만 표시되고 KlipperScreen 인터페이스가 나타나지 않음
- **원인**: KlipperScreen이 활성화되지 않았습니다.
- **해결 방법**: [시스템 구성](/docs/DebugDoc/flyos-fast/config)에서 KlipperScreen을 활성화하고 `klipper_screen`을 `1`로 설정하세요.

## 질문: 시스템에서 웹 페이지를 통해 klipper 등 구성 요소를 업데이트할 수 없음
- **원인**: FlyOS-Fast에 포함된 klipper, moonraker 등의 구성 요소는 자체 개발 버전입니다.
- **해결 방법**: 필요한 경우 명령줄에서 직접 업데이트할 수 있지만 권장하지 않습니다. 시스템 업데이트를 기다리시기 바랍니다.

## 질문: Happy Hare와 같은 서드파티 구성 요소 설치 스크립트를 사용할 수 없음
- **원인**: FlyOS-Fast의 기본 사용자는 `root`이며, 이는 서드파티 설치 스크립트에서 root로 인식되어 설치되지 않습니다.
- **해결 방법**: 설치 스크립트의 root 감지 부분을 간단히 수정하면 됩니다. 자세한 내용은 [여기](https://github.com/kluoyun/Happy-Hare/tree/flyos-dev)를 참조하시기 바랍니다. 다른 설치 스크립트도 거의 동일합니다.
