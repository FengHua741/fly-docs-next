# 펌웨어 설명

    :::danger 주의

    * Klipper 펌웨어는 가장 잘 작동하는 방법으로 당신의 업스트림 펌웨어로 컴파일해야 합니다.
    * 다른 사람이 컴파일하거나 다른 버전의 펌웨어를 사용하면 여러 문제가 발생할 수 있습니다.
    * 가능한 한 수정된 업스트림 펌웨어는 사용하지 않는 것이 좋습니다.

    :::

## 특별한 주의사항

    :::info 특별한 주의사항
    * 대부분의 FLY 메인보드는 출하 시 `Katapult` 펌웨어가 이미 프로그램되었습니다.
    * FLY 메인보드는 현재 `Katapult` 펌웨어를 사용하여 `Klipper` 펌웨어를 프로그래밍합니다.
    * DP5, D5, D7 이 세 개의 메인보드는 수동으로 `Katapult` 펌웨어를 프로그래밍해야 합니다.
    * 이전 펌웨어의 영향으로 `Katapult` 펌웨어를 프로그래밍한 후에 메인보드 ID를 검색하려면 두 번 메인보드 리셋을 클릭해야 합니다. 그렇지 않으면 펌웨어 프로그래밍이 불가능합니다.
    :::

## 펌웨어 컴파일 설명

    :::tip

    * FLY 메인보드는 `Katapult`와 메모리 카드를 사용하여 부트로더를 프로그래밍합니다.
    * `Katapult` 펌웨어의 장점은 메인보드에 직접 접촉하지 않고도 펌웨어를 업데이트할 수 있다는 것입니다.
    * 부트로더 펌웨어의 장점은 잘못된 펌웨어를 설치할 걱정이 없다는 것입니다.
    * 펌웨어 구성에는 `MobaXterm`을 사용하는 것이 가장 좋습니다. 일부 특수 SSH 소프트웨어가 컴파일 실패를 초래한다는 그룹 친구들의 피드백을 고려할 때입니다.
    
    :::

    * 어떤 펌웨어를 사용하더라도 펌웨어가 정확하게 컴파일되었는지 확인하는 것이 중요합니다. 그렇지 않으면 정상적인 인쇄를 찾을 수 없거나 장치를 찾을 수 없게 될 수 있습니다.

## 펌웨어 컴파일 설명

    :::tip
    * `Enable extra low-level configuration options` 옵션을 선택하지 않으면 자세한 내용이 표시되지 않습니다.
    :::

### STM32

    * STM32 펌웨어를 컴파일할 때 `GPIO pins to set at micro-controller startup`를 설정해야 하는 경우 반드시 대문자로 입력해야 합니다.

### RP2040

    * RP2040 펌웨어를 컴파일할 때 `GPIO pins to set at micro-controller startup`를 설정해야 하는 경우 반드시 소문자로 입력해야 합니다.

## 펌웨어 프로그래밍 설명

    :::danger 주의

    * `Katapult` 펌웨어를 사용하여 `Klipper` 펌웨어를 프로그래밍하기 위해서는 장치를 찾을 수 있어야 합니다. 장치를 찾을 수 없는 경우에는 수동으로 두 번 메인보드 리셋을 클릭해야 합니다.
    * 교육 외의 방법으로 펌웨어를 프로그래밍하지 마십시오.
    * 펌웨어에 컴파일 오류가 있는지 확인하십시오. 컴파일 오류가 있으면 펌웨어를 정상적으로 프로그래밍할 수 없습니다.

    :::

### Katapult 펌웨어

    * `Katapult` 펌웨어로 컴파일 오류가 있는 펌웨어를 프로그래밍하는 해결 방법
    * 두 번 메인보드 리셋을 클릭하면 됩니다. 리셋을 클릭한 후 일반적으로 LED 하나가 깜빡이는 것을 볼 수 있습니다.
    * 메인보드 펌웨어를 지우고 다시 `Katapult` 펌웨어를 프로그래밍합니다.

### 부트로더 펌웨어

    * `bootloader` 펌웨어로 컴파일 오류가 있는 펌웨어를 프로그래밍하는 해결 방법
    * TF 카드를 사용하여 다시 프로그래밍하면 됩니다.
    * 프로그래밍이 안 되면 전원을 끄고 메모리 카드를 메인보드에 설치한 후 다시 전원을 켜십시오. 5초 이내에 프로그래밍이 가능합니다.
