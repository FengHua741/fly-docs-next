## Z-Offset 미세 조정 방법
* 첫 번째 단계: Prusa/OrcaSlicer/Cura/Bambu Studio를 사용하여 높이 0.2mm의 200x200 크기의 정사각형 조각을 잘라내어 Z-offset 미세 조정을 위해 인쇄합니다.
* 두 번째 단계: 프린터의 IP를 통해 `fluidd` `mainsail`에 접속하여 대시보드의 `이동 제어`에서 Z축 오프셋을 계속 미세 조정한 후 조정한 값을 기억합니다.
<ImageView image={require('@site/docs/DebugDoc/faq/img/Zoff.webp').default} size="80%" align="left" />
* 세 번째 단계: 아래 명령어로 klipper를 재시작합니다.
```bash
FIRMWARE_RESTART
```
* 네 번째 단계: 아래 명령어로 Z-offset을 조정할 수 있는 UI 인터페이스를 띄워 인쇄 중에 조정한 값을 `z-offset`으로 설정합니다.
```bash
G28
PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
```
<ImageView image={require('@site/docs/DebugDoc/faq/img/UI.webp').default} size="50%" align="left" />

:::info
* `Eddy current sensor error` 오류가 발생하면 eddy 설정의 z_offset 값을 높여보세요, 예시를 참고하세요.

:::


## 작업 설명

    :::warning 주의 사항

    * 먼저 전류 보정을 하고 나서 코일 주파수 보정을 해야 합니다.
    * 다중 Z축 기기는 수동으로 한 번 정렬하고 모든 보정을 완료한 후 다시 한 번 정렬하여 다시 보정을 해야 합니다.
    * 하위 펌웨어는 Kliiper 버전과 완전히 일치해야 합니다.

    :::

    * 아래 설정 후 `FUIDD` 웹 페이지에서 다음 작업을 수행하십시오.
        ```cfg
        [force_move]
        enable_force_move: true
        ```

    ### 첫 번째 단계: EDDY 전류 보정

        * 1: 콘솔에 아래 명령어를 입력합니다.
            ```bash
            SET_KINEMATIC_POSITION z=80
            ```
        * 2: X와 Y축을 원점으로 이동하고 도구 헤드를 히트베드 중앙으로 이동합니다.
            ```bash
            G28 X Y
            ```
        * 위의 단계를 완료한 후, Eddy 전류를 보정해야 합니다. 먼저 EDDY를 히트베드 중앙으로 이동한 후 노즐이 히트베드에 약간 닿을 때까지 이동합니다. 그런 다음 콘솔에서 아래 명령어를 실행하여 자동으로 전류를 얻고 마지막으로 `SAVE_CONFIG` 명령으로 매개변수를 저장합니다. 피드백 값이 15 정도면 정상입니다. 피드백 값이 0이거나 30이면 연결이 잘못되었는지 확인하세요 (접촉 불량).
            ```bash
            LDC_CALIBRATE_DRIVE_CURRENT CHIP=fly_eddy_probe
            ```
        * 정상 반환 값은 `15` 정도여야 합니다.
        * 보정이 완료되면 `SAVE_CONFIG` 명령으로 매개변수를 저장합니다.

    ### 두 번째 단계: Eddy 주파수 보정


        :::info

        * Eddy를 설치할 때, 바닥면은 노즐 위에 최소 1-2mm 위치해야 합니다.
        * Eddy 주파수 보정을 할 때, 코일 PCB는 히트베드에서 최소 5mm 이상 떨어져 있어야 합니다.

        :::

        * 그런 다음 노즐을 플랫폼 중앙으로 이동합니다（주의: 이 단계에서는 높이맵이 로드되지 않은 상태여야 합니다）
        * 수동 Z 오프셋 보정을 시작합니다 [Paper test](https://www.klipper3d.org/Bed_Level.html#the-paper-test)
            ```bash
            PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
            ```
        * 보정이 완료되면 `SAVE_CONFIG` 명령으로 매개변수를 저장합니다.
        :::info
        * `z_tilt` 또는 `quad_gantry_level（QGL）` 기능이 있는 프린터는 먼저 평행도를 조정한 후 `i3는 X축의 평행도를 조정하고, Trident는 히트베드의 평행도를 조정하며, Voron 2.4는 게이트리 프레임의 평행도를 조정한 후 다시 한 번 Eddy 주파수를 보정합니다.
        :::
    ### 세 번째 단계: 온도 보상

        * 모든 축을 원점으로 이동합니다.
       ```bash
       G28
       ```

        * `quad_gantry_level` 또는 `z_tilt`가 있으면, 먼저 `Z_TILT_ADJUST` 또는 `QUAD_GANTRY_LEVEL`을 한 번 실행한 후, 다시 한 번 원점으로 이동합니다.
        * 콘솔에 G0 Z5를 입력하거나 웹 페이지를 통해 Z축을 히트베드 위 5mm로 이동합니다.
        * 아래 코드를 실행하여 장비의 `idle timeout`을 길게 설정하고, 온도 상승 중 타임아웃을 방지합니다.
            ```bash
            SET_IDLE_TIMEOUT TIMEOUT=65000
            ```
        * 아래 명령을 실행하여 보정을 시작합니다.
            ```bash
            TEMPERATURE_PROBE_CALIBRATE PROBE=fly_eddy_probe TARGET=56 STEP=4
            ```
        :::warning 주의 사항

        * 힌트: 위의 명령에서 EDDY의 목표 온도는 `56℃`로 설정되어 대부분의 장비에 적합합니다.
        * 3D 프린터 실내의 실제 온도가 더 높으면 목표 온도를 적절히 높일 수 있습니다.
        * 목표 온도가 높을수록 Eddy가 온도 균형을 맞추는 데 더 오래 걸리므로 보정 시간이 길어집니다.

        :::
    
        1. 이 작업을 수행하면 UI에 Z축 조정 창이 표시됩니다. 위에 언급된 수동 [Z축 오프셋 보정（Paper Test）방법](https://www.klipper3d.org/Bed_Level.html#the-paper-test)을 사용하여 노즐과 베드 사이에 종이를 끼우고 값을 확인하세요.
        2. 값을 수락한 후, 히트베드 온도를 최대로 설정하고 노즐 온도를 220℃로 설정합니다.
        3. 에어컨이나 창문을 열어놓은 방에서 작업 중일 경우, Eddy의 온도 상승을 보장하기 위해 에어컨이나 창문을 닫는 것이 좋습니다. 바람이 온도 상승에 영향을 미칩니다.
        4. Eddy의 온도가 상승함에 따라 시스템은 4℃마다 수동 Z 오프셋 보정을 요청합니다. 히트베드 온도가 매우 높으므로 화상에 주의하십시오!!!
        5. Eddy의 온도가 더 이상 상승하지 않으면 아래 관련 명령을 사용하여 보정을 조기에 종료할 수 있습니다.
        
        * 드리프트 보정 중에 사용 가능한 추가 gcode 명령은 다음과 같습니다: TEMPERATURE_PROBE_NEXT
            * `TEMPERATURE_PROBE_NEXT`    EDDY가 설정된 목표 온도 `56℃`에 도달하기 전에 강제로 EDDY가 새 데이터를 샘플링하도록 합니다.
            * `TEMPERATURE_PROBE_COMPLETE`    EDDY가 설정된 온도 `56℃`에 도달하기 전에 보정을 완료합니다.
            * `ABORT`  보정을 종료하고 결과를 무시합니다.
        * 이러한 단계를 완료하면 Eddy는 광범위한 온도 범위에서 훌륭한 첫 층 인쇄 성능을 제공할 수 있습니다!
## 다양한 최적화 매크로

    ### `[quad_gantry_level]` 스캔 최적화
        * `[quad_gantry_level]`이 구성되어 있으면 아래 매크로를 설정에 추가하세요, 그렇지 않으면 필요 없습니다.
        * 이 매크로는 게이트리 프레임의 높이 차이에 따라 탐색 높이를 낮춥니다.
            ```cfg
            [gcode_macro QUAD_GANTRY_LEVEL]
            rename_existing: _QUAD_GANTRY_LEVEL
            gcode:
                SAVE_GCODE_STATE NAME=STATE_QGL
                BED_MESH_CLEAR
                {% if not printer.quad_gantry_level.applied %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_QGL
            ```

    ### `[z_tilt]` 스캔 최적화

        * `[z_tilt]`이 구성되어 있으면 아래 매크로를 설정에 추가하세요, 그렇지 않으면 필요 없습니다.
        * 이 매크로는 게이트리 프레임의 높이 차이에 따라 탐색 높이를 낮춥니다.
            ```cfg
            [gcode_macro Z_TILT_ADJUST]
            rename_existing: _Z_TILT_ADJUST
            gcode:
                SAVE_GCODE_STATE NAME=STATE_Z_TILT
                BED_MESH_CLEAR
                {% if not printer.z_tilt.applied %}
                _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _Z_TILT_ADJUST horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_Z_TILT
            ```

    ### 스캔 최적화

        * 대형 AC 히트베드(500W 이상)를 사용하는 경우
        * 아래 매크로는 스캔 중에 히트베드를 끄고 스캔이 완료되면 다시 가열합니다.
            ```cfg
            [gcode_macro BED_MESH_CALIBRATE]
            rename_existing: _BED_MESH_CALIBRATE
            gcode:
                {% set TARGET_TEMP = printer.heater_bed.target %}
                M140 S0
                _BED_MESH_CALIBRATE {rawparams}
                M140 S{TARGET_TEMP}
            ```
    ## 베드 메시 명령어
    
        * `printer.cfg` 설정 파일에 아래 매크로를 추가합니다.
            ```bash
            [gcode_macro BED_MESH_CALIBRATE]
             rename_existing: _BED_MESH_CALIBRATE
             gcode: 
                   _BED_MESH_CALIBRATE METHOD=rapid_scan {rawparams}
            ```
        * 네트워크 베드 명령어는 Klipper가 객체 제외 기능을 사용할 수 있어야 하며, 시작 매크로에 추가하는 것이 좋습니다.
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan adaptive=1 
            ```
