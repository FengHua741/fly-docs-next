## z-오프셋을 어떻게 미세 조정하나요?

* 첫 번째 단계: `pursa`/`OrcaSlicer`/`cura`/`Bambu Studio`를 사용하여 높이 0.2mm의 200x200 크기의 사각형을 잘라 z-오프셋을 미세 조정하기 위해 인쇄합니다.
* 두 번째 단계: 프린터의 IP를 통해 `fluidd` 또는 `mainsail`에 접속하여 계기판의 `이동 제어`에서 z축 오프셋을 계속 미세 조정한 후, 조정된 값을 기억합니다.
<ImageView image={require('@site/docs/DebugDoc/faq/img/Zoff.webp').default} size="80%" align="left" />
* 세 번째 단계: 아래 명령어로 klippper를 재시작합니다.
```bash
FIRMWARE_RESTART
```
* 네 번째 단계: 아래 명령어를 통해 z-오프셋을 조정하는 UI 화면이 나타나며, 인쇄 중에 조정된 값을 z-오프셋에 적용합니다.
```bash
G28
PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
```
<ImageView image={require('@site/docs/DebugDoc/faq/img/UI.webp').default} size="50%" align="left" />

:::info
* `Eddy current sensor error` 오류가 발생하면, eddy 설정의 z_offset 값을 크게 조정하세요. 예시를 참조하세요.

<ImageView image={require('@site/docs/DebugDoc/faq/img/EDDYZOFF.webp').default} size="50%" align="left" />
:::

:::info
 * 위의 모든 작업을 완료했지만 스캔이 실패하면 z 제한을 가상 제한으로 변경하세요.
:::


## 작업 설명

    :::warning 주의사항

    * 먼저 전류 보정을 한 후 코일 주파수 보정을 하세요.

    * 하위 기기 펌웨어는 Kliiper 버전과 완전히 일치해야 합니다.

    * `FUIDD` 웹 페이지에 아래 설정을 추가한 후 다음 작업을 실행하세요.
        ```cfg
        [force_move]
        enable_force_move: true
        ```

    ### 첫 번째 단계: EDDY 전류 보정

        * 1: 콘솔에 아래 명령을 입력하세요.
            ```bash
            SET_KINEMATIC_POSITION z=80
            ```
        * 2: X와 Y를 원점으로 이동시키고 도구 헤드를 히트베드 중앙으로 이동시킵니다.
            ```bash
            G28 X Y
            ```
        * 위의 단계를 완료한 후, Eddy 전류를 보정해야 합니다. 먼저 EDDY를 히트베드 중앙으로 이동시키고 노즐이 히트베드에 막 닿을 때까지 이동시킵니다. 그런 다음 콘솔에서 아래 명령을 실행하여 자동으로 전류 값을 얻은 후 `SAVE_CONFIG`로 매개변수를 저장합니다. 피드백 값은 대략 15가 정상 값이며, 0 또는 30이 나오면 연결 상태를 확인하세요(접촉 불량).
            ```bash
            LDC_CALIBRATE_DRIVE_CURRENT CHIP=fly_eddy_probe
            ```
        * 정상적인 반환 값은 약 `15`여야 합니다.
        * 보정 후 `SAVE_CONFIG`로 매개변수를 저장하세요.

    ### 두 번째 단계: Eddy 주파수 보정


        :::info

        * Eddy 설치 시, 노즐과의 거리는 최소 1-2mm 이상이어야 합니다.
        * Eddy 주파수 보정 시, 코일 PCB는 히트베드에서 최소 5mm 떨어져 있어야 합니다.

        :::

        * 그런 다음 노즐을 플랫폼 중앙으로 이동시킵니다(주의: 이 단계는 기계가 heightmap을 로드하지 않았는지 확인하세요).
        * 수동 z 오프셋 보정을 시작합니다 [Paper test](https://www.klipper3d.org/Bed_Level.html#the-paper-test)
            ```bash
            PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
            ```
        * 보정 후 `SAVE_CONFIG`로 매개변수를 저장하세요.
        :::info
        * `z_tilt` 또는 `quad_gantry_level（QGL）` 기능이 있는 프린터는, 먼저 평행도를 조정한 후 다시 한 번 Eddy 주파수를 보정하세요. i3는 X축의 평행도를 조정하고, 삼지창은 히트베드의 평행도를 조정하고, voron 2.4는 간트리 프레임의 평행도를 조정하세요.
        :::
    ### 세 번째 단계: 온도 보정

        * 모든 축을 원점으로 이동시킵니다.
       ```bash
       G28
       ```

        * `quad_gantry_level` 또는 `z_tilt`가 있는 경우, 먼저 `Z_TILT_ADJUST` 또는 `QUAD_GANTRY_LEVEL`을 실행한 후 다시 한 번 원점으로 이동합니다.
        * 콘솔에 G0 Z5를 입력하거나 웹 페이지를 통해 Z축을 히트베드 위로 5mm 이동시킵니다.
        * 아래 코드를 실행하여 기계의 `idle timeout`을 길게 설정하여, 온도 상승 중에 타임아웃을 방지합니다.
            ```bash
            SET_IDLE_TIMEOUT TIMEOUT=65000
            ```
        * 아래 명령을 실행하여 보정을 시작합니다.
            ```bash
            TEMPERATURE_PROBE_CALIBRATE PROBE=fly_eddy_probe TARGET=56 STEP=4
            ```
        :::warning 주의사항

        * 참고: 위 명령에서 EDDY의 목표 온도는 `56℃`로 설정되었으며, 대부분의 장치에 적합합니다.
        * 3D 프린터의 실제 챔버 온도가 더 높다면, 목표 온도를 적절히 높이세요.
        * 목표 온도가 높을수록 보정에 걸리는 시간이 길어지므로, Eddy가 온도 균형을 맞추는 데 더 오래 걸립니다.

        :::
    
        1. 이 작업을 실행한 후, UI에 Z축 조정 상자가 표시됩니다. 위에서 언급한 수동 [Z축 오프셋 보정(Paper Test) 방법](https://www.klipper3d.org/Bed_Level.html#the-paper-test)을 사용하여 종이를 노즐과 베드 사이에 끼우고 값을 확인하세요.
        2. 값을 수락한 후, 히트베드 온도를 최대로 설정하고 노즐 온도를 220℃로 설정합니다.
        3. 에어컨이나 창문을 닫아 Eddy의 온도가 올라갈 수 있도록 하세요. 바람이 온도 상승을 방해할 수 있습니다.
        4. Eddy의 온도가 상승함에 따라 시스템은 매 4℃마다 수동 Z 오프셋 보정을 자동으로 요청합니다. 히트베드 온도가 매우 높으므로 화상에 주의하세요!!!
        5. 수동 Z 오프셋 보정(Paper Test)을 반복하여 보정을 완료할 때까지 진행합니다. Eddy의 온도가 더 이상 오르지 않으면 아래 명령을 사용하여 보정을 조기에 종료할 수 있습니다.
        
        * 드리프트 보정 중 사용할 수 있는 추가적인 gcode 명령어는 다음과 같습니다: `TEMPERATURE_PROBE_NEXT`
            * `TEMPERATURE_PROBE_NEXT`    EDDy가 설정된 목표 온도 `56℃`에 도달하기 전에 강제로 새로운 데이터를 샘플링합니다.
            * `TEMPERATURE_PROBE_COMPLETE`    EDDy가 설정된 온도 `56℃`에 도달하지 못할 때 보정을 완료합니다.
            * `ABORT`  보정을 종료하고 결과를 무시합니다.
        * 이러한 단계를 완료하면, Eddy는 광범위한 온도 범위에서 뛰어난 첫 번째 층 인쇄 효과를 제공할 수 있습니다!
## 다양한 최적화 매크로

    ### `[quad_gantry_level]` 스캔 최적화
        * `quad_gantry_level`이 설정되어 있다면 아래 매크로를 구성 파일에 추가하세요. 그렇지 않으면 추가할 필요가 없습니다.
        * 이 매크로는 간트리의 높이 차이가 줄어들수록 탐지 높이를 낮춥니다.
            ```cfg
            [gcode_macro QUAD_GANTRY_LEVEL]
            rename_existing: _QUAD_GANTRY_LEVEL
            gcode:
                SAVE_GCODE_STATE NAME=STATE_QGL
                BED_MESH_CLEAR
                {% if not printer.quad_gantry_level.applied %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_QGL
            ```

    ### `[z_tilt]` 스캔 최적화

        * `z_tilt`가 설정되어 있다면 아래 매크로를 구성 파일에 추가하세요. 그렇지 않으면 추가할 필요가 없습니다.
        * 이 매크로는 간트리의 높이 차이가 줄어들수록 탐지 높이를 낮춥니다.
            ```cfg
            [gcode_macro Z_TILT_ADJUST]
            rename_existing: _Z_TILT_ADJUST
            gcode:
                SAVE_GCODE_STATE NAME=STATE_Z_TILT
                BED_MESH_CLEAR
                {% if not printer.z_tilt.applied %}
                _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _Z_TILT_ADJUST horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_Z_TILT
            ```

    ### 스캔 최적화

        * 대형 AC 히트베드(500W 이상)를 사용하는 경우
        * 아래 매크로는 스캔 중에 히트베드를 껐다가 스캔이 완료되면 다시 가열합니다.
            ```cfg
            [gcode_macro BED_MESH_CALIBRATE]
            rename_existing: _BED_MESH_CALIBRATE
            gcode:
                {% set TARGET_TEMP = printer.heater_bed.target %}
                M140 S0
                _BED_MESH_CALIBRATE {rawparams}
                M140 S{TARGET_TEMP}
            ```
    ## 네트워크 베드 명령

        * `printer.cfg` 구성 파일에 아래 매크로를 추가하세요.
            ```bash
            [gcode_macro BED_MESH_CALIBRATE]
             rename_existing: _BED_MESH_CALIBRATE
             gcode: 
                   _BED_MESH_CALIBRATE METHOD=rapid_scan {rawparams}
            ```
        * 네트워크 베드 명령어는 Klipper가 객체 배제 기능을 사용할 수 있도록 해야 하며, 시작 매크로에 추가하는 것을 권장합니다.
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan adaptive=1 
            ```
