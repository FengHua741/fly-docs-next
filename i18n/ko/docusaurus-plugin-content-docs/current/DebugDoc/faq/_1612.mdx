## z-오프셋을 어떻게 조정하나요?
* 첫 번째 단계: Prusa/OrcaSlicer/Cura/Bambu Studio를 사용하여 높이 0.2의 200x200 크기의 사각형 조각을 잘라 z-오프셋을 조정하여 인쇄합니다.
* 두 번째 단계: 프린터 IP를 통해 `fluidd` `mainsail`에 들어가 대시보드의 `이동 제어`에서 z축 오프셋을 계속해서 조정하고, 조정한 값을 기억합니다.
<ImageView image={require('@site/docs/DebugDoc/faq/img/Zoff.webp').default} size="80%" align="left" />
* 세 번째 단계: 아래 명령어를 통해 klippper를 재시작합니다
```bash
FIRMWARE_RESTART
```
* 네 번째 단계: 아래 명령어를 통해 z-오프셋을 조정하는 UI 인터페이스를 열어 인쇄 중 조정한 값을 z-오프셋으로 설정합니다.
```bash
G28
PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
```

    * 하위 기기 펌웨어는 Kliiper 버전과 완전히 일치해야 합니다.

        [force_move]

    ### 첫 번째 단계: EDDY 전류 보정

        * 1: 콘솔에 아래 명령을 입력하세요
            ```bash
            SET_KINEMATIC_POSITION z=80
            ```
        * 2: X와 Y를 원점으로 되돌리고 도구 헤드를 히트베드 중앙으로 이동합니다
            ```bash
            G28 X Y
            ```
        * 위의 단계가 완료되면 Eddy 전류를 보정해야 합니다. 먼저 EDDY를 히트베드 중앙으로 이동하고 노즐이 히트베드에 막 닿을 때까지 이동합니다. 그런 다음 콘솔에서 아래 명령을 실행하여 자동으로 전류를 얻고, 마지막으로 `SAVE_CONFIG`를 통해 매개변수를 저장합니다. 피드백 값이 15 정도면 정상입니다. 0이나 30이 나오면 연결이 잘못되었는지 확인하세요 (접촉 불량).
            ```bash
            LDC_CALIBRATE_DRIVE_CURRENT CHIP=fly_eddy_probe
            ```
        * 정상적인 반환 값은 `15` 정도여야 합니다.
        * 보정이 완료되면 `SAVE_CONFIG`를 통해 매개변수를 저장합니다.

    ### 두 번째 단계: Eddy 주파수 보정


        :::info

        * Eddy 설치 시 노즐 위치보다 최소 1-2mm 위에 있어야 합니다.
        * Eddy 주파수 보정 시, 코일 PCB는 히트베드에서 최소 5mm 이상 떨어져 있어야 합니다.

        :::

        * 그런 다음 노즐을 플랫폼 중앙으로 이동합니다 (주의: 이 단계에서는 기기가 heightmap을 로드하지 않았는지 확인하세요)
        * 수동 z-오프셋 보정을 시작합니다 [Paper test](https://www.klipper3d.org/Bed_Level.html#the-paper-test)
            ```bash
            PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
            ```
        * 보정이 완료되면 `SAVE_CONFIG`를 통해 매개변수를 저장합니다
        :::info
        * `z_tilt` 또는 `quad_gantry_level（QGL）` 기능이 있는 프린터는 평행도를 먼저 조정한 후 `i3는 X축 평행도를 조정하고, 삼지창은 히트베드 평행도를 조정하며, voron 2.4는 갠트리 프레임의 평행도를 조정한 후 Eddy 주파수를 다시 보정합니다.
        :::
    ### 세 번째 단계: 온도 보정

        * 모든 축을 원점으로 되돌립니다
       ```bash
       G28
       ```

        * `quad_gantry_level` 또는 `z_tilt`가 있는 경우, 먼저 `Z_TILT_ADJUST` 또는 `QUAD_GANTRY_LEVEL`을 한 번 실행한 후 다시 한 번 원점으로 되돌립니다.
        * 콘솔에 G0 Z5를 입력하거나 웹 페이지 등을 통해 Z축을 히트베드 위 `5mm`로 이동합니다.
        * 아래 코드를 실행하여 기기의 `idle timeout`을 조금 더 길게 설정하여 가열 과정에서 timeout을 피합니다
            ```bash
            SET_IDLE_TIMEOUT TIMEOUT=65000
            ```
        * 보정을 시작하려면 아래 명령을 실행합니다
            ```bash
            TEMPERATURE_PROBE_CALIBRATE PROBE=fly_eddy_probe TARGET=56 STEP=4
            ```
        :::warning 주의사항

        * 참고: 위 명령에서 EDDY의 목표 온도는 `56℃`로 설정되어 있어 대부분의 장치에 적합합니다.
        * 3D 프린터 내부의 실제 온도가 더 높다면 목표 온도를 적절히 올릴 수 있습니다.
        * 목표 온도가 높을수록 Eddy가 온도 균형에 도달하는 데 더 많은 시간이 필요하므로 보정 시간이 길어집니다.

        :::
    
        1. 이 작업을 실행하면 UI에 Z 축 조정 창이 표시됩니다. 위에서 언급한 수동 [Z축 오프셋 보정（Paper Test）방법](https://www.klipper3d.org/Bed_Level.html#the-paper-test)을 사용하여 종이를 노즐과 베드 사이에 끼우고 값을 확인하세요.
        2. 값을 수락한 후, 히트베드 온도를 최대로, 노즐 온도를 220℃로 설정합니다.
        3. 에어컨이나 창문을 열어둔 방에서는 Eddy의 온도 상승을 보장하기 위해 에어컨이나 창문을 닫는 것이 좋습니다. 바람이 온도 상승에 영향을 미칠 수 있습니다.
        4. Eddy의 온도가 상승함에 따라 시스템은 4℃마다 수동 Z 오프셋 보정을 실행하도록 안내합니다. 히트베드 온도가 매우 높으므로 화상을 조심하세요!!!
        5. 수동 Z 오프셋 보정（Paper Test）을 반복하여 보정이 완료될 때까지 진행합니다. Eddy의 온도가 더 이상 상승하지 않는다면 아래의 관련 명령을 사용하여 보정을 조기에 종료할 수 있습니다.
        
        * 드리프트 보정 중 사용 가능한 추가 gcode 명령어는 다음과 같습니다: TEMPERATURE_PROBE_NEXT
            * `TEMPERATURE_PROBE_NEXT`    EDDY가 설정된 목표 온도 `56℃`에 도달하기 전에 강제로 새 데이터를 샘플링합니다.
            * `TEMPERATURE_PROBE_COMPLETE`    EDDY가 설정된 온도 `56℃`에 도달하기 전에 보정을 완료합니다.
            * `ABORT`  보정을 중단하고 결과를 무시합니다.
        * 이러한 단계를 완료하면 Eddy는 광범위한 온도 범위에서 탁월한 첫 번째 레이어 인쇄 효과를 제공할 수 있습니다!
## 다양한 최적화 매크로

    ### `[quad_gantry_level]` 스캔 베드 최적화
        * `quad_gantry_level`이 구성되어 있다면 아래 매크로를 설정에 추가하세요, 그렇지 않으면 추가할 필요가 없습니다.
        * 이 매크로는 갠트리의 높이 차이에 따라 탐지 높이를 낮춥니다.
            ```cfg
            [gcode_macro QUAD_GANTRY_LEVEL]
            rename_existing: _QUAD_GANTRY_LEVEL
            gcode:
                SAVE_GCODE_STATE NAME=STATE_QGL
                BED_MESH_CLEAR
                {% if not printer.quad_gantry_level.applied %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_QGL
            ```

    ### `[z_tilt]` 스캔 베드 최적화

        * `z_tilt`가 구성되어 있다면 아래 매크로를 설정에 추가하세요, 그렇지 않으면 추가할 필요가 없습니다.
        * 이 매크로는 갠트리의 높이 차이에 따라 탐지 높이를 낮춥니다.
            ```cfg
            [gcode_macro Z_TILT_ADJUST]
            rename_existing: _Z_TILT_ADJUST
            gcode:
                SAVE_GCODE_STATE NAME=STATE_Z_TILT
                BED_MESH_CLEAR
                {% if not printer.z_tilt.applied %}
                _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _Z_TILT_ADJUST horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_Z_TILT
            ```

    ### 스캔 베드 최적화

        * 대형 AC 히트베드(500W 이상)를 사용하는 경우
        * 아래 매크로는 스캔 중 히트베드를 끄고 스캔이 완료된 후 다시 가열합니다.
            ```cfg
            [gcode_macro BED_MESH_CALIBRATE]
            rename_existing: _BED_MESH_CALIBRATE
            gcode:
                {% set TARGET_TEMP = printer.heater_bed.target %}
                M140 S0
                _BED_MESH_CALIBRATE {rawparams}
                M140 S{TARGET_TEMP}
            ```
    ## 베드 메시 명령
    
        * 전체 베드 메시 스캔 명령
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan
            ```
        * 로컬 네트워크 베드 메시 명령, Klipper가 객체 제외 기능을 사용할 수 있는지 확인하고 시작 매크로에 추가하는 것이 좋습니다.
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan adaptive=1 
            ```
