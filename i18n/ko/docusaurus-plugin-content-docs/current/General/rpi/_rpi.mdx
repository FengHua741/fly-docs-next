주의사항

    1. USB 이외의 다른 장치를 전원이 켜져 있을 때 꽂거나 빼는 것을 금지합니다.
    2. Type C 인터페이스는 전원 공급용으로도 사용될 수 있으며, USB SLAVE 인터페이스로 이미지를 번들링할 수도 있습니다.
    3. CM4 전원 공급을 정상적으로 보장하기 위해, Type C 인터페이스로 이미지를 번들링할 때 다른 장치들을 연결하지 마십시오.
    4. 모듈에는 어떠한 보호 장치도 내장되어 있지 않으므로, 전원 단락을 일으키지 않도록 주의하십시오.
    5. FLY 메인보드는 POE 기능을 지원하지 않습니다! POE 전원을 사용하지 마십시오.

:::danger 주의
* 주의: EMMC 버전은 SD 카드를 사용할 수 없습니다.
* 주의: EMMC가 없는 버전은 MICRO SD 카드로 부팅하려면 리더기를 사용하여 이미지를 번들링해야 합니다.
* 메모리 카드는 `class10` 등급 또는 그 이상의 전송 속도를 가진 TF 카드를 권장하며, SanDisk와 같은 브랜드의 TF 카드를 사용하는 것이 좋습니다.
:::

## EMMC 또는 SD 카드 포맷

    * SDFormatter.exe 소프트웨어를 사용하여 SD 카드를 포맷하십시오 [여기 클릭](https://www.waveshare.net/w/upload/d/d7/Panasonic_SDFormatter.zip)

        <ImageView image={require('@site/docs/General/rpi/img/sdformatter.webp').default} size="80%" align="left" />


    * 또는 Windows 기본 내장 도구를 사용하여 파티션을 삭제한 후, 파티션을 다시 생성할 수 있습니다. 파티션을 삭제한 후 새로운 파티션을 생성하면 됩니다.
    * 이 컴퓨터 오른쪽 클릭 -> 관리 -> 디스크 관리 -> 새롭게 추가된 드라이브 문자 찾기 (선택을 잘못하면 안 됩니다.) -> 모든 파티션 삭제 -> 임의의 파일 시스템으로 새로운 파티션 생성 만약 시스템을 번들링했었다면 두 개의 드라이브 문자가 있을 것입니다. 무시하고 포맷화하거나 파티션을 삭제해도 됩니다.

## 이미지 번들링

    1. Raspberry Pi Imager [여기 클릭](https://www.raspberrypi.com/software/)를 사용하여 이미지를 번들링하십시오.
    2. 번들링 전에 계정 비밀번호와 SSH를 설정해야 합니다.

            <ImageView image={require('@site/docs/General/rpi/img/imager.webp').default} size="80%" align="left" />


    3. 번들링 완료 후, 내 컴퓨터에서 U드라이브의 드라이브 문자가 나타납니다.

## 번들링 완료 및 재시작

    * 번들링 완료 후 전원을 차단하고, 컴퓨터와의 연결선을 끊고, BOOT 스위치를 원래대로 돌려놓습니다. 전원을 다시 공급하면 됩니다.

## Raspberry Pi Zero2와 Zero2W 주의사항

    * 이 주의사항은 G2T와 함께 FLY-TFT 화면을 사용하기 위한 설명입니다.
    * Raspberry Pi Zero2와 Zero2W는 기본 표시 장치를 수정하려면 아래 코드를 실행하여 직접 수정해야 합니다.
    * 실행 완료 후 재시작해야 사용할 수 있습니다.
    ```cfg
    sudo sed -i 's/\/dev\/fb0/\/dev\/fb1/g' /etc/X11/xorg.conf.d/99-fbdev.conf
    ```
