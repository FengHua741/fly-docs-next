## 장치 검색 전 주의사항

    * ID를 가져가기 전에 먼저 [SSH 연결하기](/docs/DebugDoc/BasicTutorial/index.mdx "클릭하여 이동") 
    * 하위 기기가 상위 기기에 데이터선을 연결했는지 확인하고 데이터 전송 기능이 있는지 확인하세요.
    * SSH로 상위 기기에 연결한 후 `lsusb`를 입력하고 엔터를 누르면 아래와 같은 몇 가지 장치가 표시될 수 있습니다.
    * 시스템이 `lsusb`를 지원하지 않는 경우 다음 명령어를 실행하여 설치할 수 있습니다.
    * `lsusb` 후에 장치가 없다면 상위 기기를 교체하세요!
    ```
    sudo apt-get install usbutils
    ```

    * 메인보드 펌웨어가 올바른 펌웨어로 업데이트되었는지 확인하세요.
    * USB 펌웨어의 경우 `lsusb` 후에 `1d50:614e`가 나타나야 합니다.
    * USB-CAN 브리지 펌웨어의 경우 `lsusb` 후에 `1d50:606f`가 나타나야 합니다.
    * RS232 펌웨어의 경우 UTOR 또는 G2T와 함께 사용해야 하며, G2T는 펌웨어 검색이 필요하지 않으며 UTOR의 경우 `lsusb` 후에 `1a86:7523`이 나타나야 합니다.

## 장치 검색 시 주의사항

:::danger 주의
ID 검색 명령은 혼용할 수 없습니다!!!!
:::

    * USB 펌웨어는 USB ID 검색 방식만 사용할 수 있다는 점에 유의하세요.
    * USB-CAN 브리지 펌웨어는 CAN ID 검색 방식만 사용할 수 있다는 점에 유의하세요.
    * RS232 펌웨어는 RS232 ID 검색 방식만 사용할 수 있다는 점에 유의하세요.
    * `0483:df11`이 나타난다면 펌웨어는 이미 업데이트되었지만 메인보드가 재부팅되지 않았을 가능성이 있습니다. 메인보드를 끄고 다시 켜도 여전히 동일하다면 DFU (점선/撥碼/버튼)이 분리되지 않았는지 확인하세요.
    * `2e8a:0003`이 나타난다면 잘못된 펌웨어가 업데이트되었을 가능성이 있습니다. RP2040은 UF2 펌웨어만 컴파일해서 업데이트할 수 있으며, 다른 펌웨어는 업데이트할 수 없습니다. `2e8a:0003`이 나타난 후 부트로더(점선/撥碼/버튼)를 분리하지 않으면 프로그래밍 실패가 발생할 수 있습니다!!!!
    * RP2040의 UF2 펌웨어는 컴파일 시 `No bootloader` 옵션을 선택하면 UF2 펌웨어가 생성됩니다.
