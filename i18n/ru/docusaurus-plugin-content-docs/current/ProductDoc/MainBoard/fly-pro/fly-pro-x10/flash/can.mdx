## Проверка режима прошивки

    * Убедитесь, что компьютер верхнего уровня подключен к сети и шнур данных, соединяющий компьютер верхнего уровня с нижним уровнем, имеет функцию передачи данных.
    * Подключитесь по SSH к компьютеру верхнего уровня и введите команду `lsusb` и нажмите Enter. Если появляется информация, заключенная в красном круге на изображении ниже, можно переходить к следующему шагу.
        * Если нет никакого ответа, это проблема системы компьютера верхнего уровня, мы не можем помочь. Вам нужно заменить рабочую систему или заменить компьютер верхнего уровня.
        * Если отсутствует команда `lsusb`, выполните эту команду для установки:
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" /> 
    * ID, показанный в красном прямоугольнике, может служить ориентиром, но он не является фиксированным — он может изменяться. Суффикс `stm32h723xx` после `OpenMoko, Inc.` имеет диагностическое значение, где `XX` может меняться.
        * Некоторые компьютеры верхнего уровня могут показывать недостаточно или вообще не показывать данные из-за проблем с системой.
