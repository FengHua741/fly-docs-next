        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Проверка режима записи

    * Убедитесь, что компьютер может нормально подключаться к сети, и убедитесь, что соединительный кабель между компьютером и устройством имеет функцию передачи данных.
    * Подключитесь через SSH к компьютеру и введите `lsusb` и нажмите Enter. Если появится информация, заключённая в круглые скобки на изображении ниже, можно переходить к следующему шагу.
        * Если нет никакой информации, это может быть проблема с системой компьютера, и мы не можем помочь вам. Вам нужно заменить систему или компьютер.
        * Если нет команды `lsusb`, выполните следующую команду для её установки:
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" /> 
    * ID, отображаемый в красном прямоугольнике, может служить ориентиром, но он **не является фиксированным**; суффикс `stm32h723xx` после `OpenMoko, Inc.` имеет значение, где `XX` означает переменные значения.
        * Некоторые компьютеры могут показывать недостаточную информацию или вообще ничего из-за системных проблем.
