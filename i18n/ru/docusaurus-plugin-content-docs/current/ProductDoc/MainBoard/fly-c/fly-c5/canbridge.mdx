# Мост CAN

CAN-шина представляет собой протокол последовательной коммуникации, используемый для реального времени, который может передавать сигналы по двужильному кабелю и является одним из самых广泛应用ых полей в мире. Протокол CAN используется для коммуникации между различными элементами автомобиля, заменяя дорогостоящие и громоздкие проводные соединения. Стойкость этого протокола расширила его применение на другие автоматизированные и промышленные системы. Характеристики протокола CAN включают целостную передачу данных по последовательному каналу, предоставление поддержки в реальном времени, скорость передачи данных до 1 Мбит/с, а также 11-битную адресацию и возможность обнаружения ошибок.
В 3D-принтерах используется CAN-шина для уменьшения количества проводов, связывающих головку печати с материнской платой. Раньше требовалось около десяти проводов, но с использованием CAN достаточно четырех проводов, что значительно снижает количество проводов и упрощает их разводку.
В данном разделе дается только краткое описание использования моста CAN.



## Важное примечание

    :::danger Важно

    * Стандартная скорость CAN в системе FLY составляет `1 Мбит/с`. Рекомендуется использовать прошивку с такой же скоростью на панели инструментов и материнской плате.
    * Панель инструментов出厂 по умолчанию прошита прошивкой `Katapult`, скорость CAN в которой составляет `1 Мбит/с`.
    * Если панель инструментов находится под прошивкой `Katapult`, то на материнской плате будет мигать светодиод, что указывает на необходимость прошивки `KLIPPER`, иначе связь не установится.
    * Перед проверкой сопротивления рекомендуется отключить питание и измерить сопротивление между `CANH` и `CANL`, которое должно быть около `60 Ом`.

    :::
