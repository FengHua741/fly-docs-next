## Проверка режима записи

    * Убедитесь, что хост-компьютер подключен к сети и шнур данных, соединяющий хост-компьютер с целевым устройством, способен передавать данные.
    * Подключитесь по SSH к хост-компьютеру и введите `lsusb` и нажмите Enter. Если появляется информация, заключённая в красных рамках на изображении ниже, можно переходить к следующему шагу.
        * Если никакой информации не выводится, это проблема системы хост-компьютера, мы не можем помочь. Вам нужно заменить систему или хост-компьютер.
        * Если отсутствует команда `lsusb`, выполните эту команду для её установки
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" /> 
    * ID, показанный в красном框е, может служить ориентиром, но он **не является фиксированным** и может изменяться. Последовательность `OpenMoko, Inc. stm32f407xx` с суффиксом `XX` означает, что суффикс может меняться.
        * Некоторые хост-компьютеры могут показывать недостаточно данных или вообще не показывать их из-за системных проблем.
