## Проверка режима записи

    * Убедитесь, что компьютер верхнего уровня подключен к сети и шнур данных, соединяющий компьютер верхнего уровня с нижним уровнем, имеет функцию передачи данных.
    * Подключитесь к компьютеру верхнего уровня через SSH и введите команду `lsusb` и нажмите Enter. Если появляется информация, выделенная на рисунке ниже, можно переходить к следующему шагу.
        * Если никакой информации не отображается, это может быть проблема системы компьютера верхнего уровня, мы бессильны. Вам нужно заменить рабочую систему или компьютер верхнего уровня.
        * Если появляется сообщение об отсутствии команды `lsusb`, выполните эту команду для установки:
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" /> 
    * ID, выделенный красным框ом, может служить ориентиром, но он **не является фиксированным** (ID может изменяться). Следующая часть `OpenMoko, Inc. stm32f407xx` указывает на то, что суффикс имеет переменную природу.
        * Некоторые компьютеры верхнего уровня могут из-за системной проблемы отображать информацию неполностью или вообще не отображать.
