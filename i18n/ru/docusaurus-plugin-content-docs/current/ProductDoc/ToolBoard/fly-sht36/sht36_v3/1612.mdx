# Использование Eddy

    :::info
    * При установке Eddy, нижняя часть должна быть расположена на расстоянии минимум 1-2 мм выше сопла
    * При калибровке частоты Eddy, катушка PCB должна находиться на расстоянии минимум 5 мм от нагревательной платформы
    * Пожалуйста, не проводите тесты на горячей платформе при комнатной температуре и не включайте нагревательную платформу во время тестирования

    :::

## Рекомендуемая конфигурация и инструкция по использованию

##### Добавьте следующую конфигурацию в файл `printer.cfg` для отладки Eddy

    ### Конфигурация сканирования платформы

        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 43
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

        * Обратите внимание, что в настоящее время нет внешнего модуля 1612, поэтому данная конфигурация пока не работает
        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 42
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

    ### Конфигурация температурной компенсации

        ```cfg
        [temperature_probe fly_eddy_probe]
        sensor_type:Generic 3950
        sensor_pin:SHT36:gpio28
        ```

    ## Команды сетки платформы
    
        * Добавьте следующий макрос в `printer.cfg`
            ```bash
            [gcode_macro BED_MESH_CALIBRATE]
             rename_existing: _BED_MESH_CALIBRATE
             gcode: 
                   _BED_MESH_CALIBRATE METHOD=rapid_scan {rawparams}
            ```
        * Команды локальной сетки платформы, убедитесь, что Klipper может использовать функцию исключения объектов, рекомендуется добавить в начальный макрос
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan adaptive=1 
            ```

### Конфигурация bed_mesh

    * В `[bed_mesh]` параметр `horizontal_move_z` рекомендуется установить на 2, чтобы Eddy был как можно ближе к платформе при сканировании

    ```cfg
    [bed_mesh]
    horizontal_move_z: 2
    ```
## Анти-столкновение макросы

    ### Оптимизация сканирования платформы для `[quad_gantry_level]`
        * Если у вас настроен `[quad_gantry_level]`, добавьте следующий макрос в конфигурацию, если нет, то добавлять не нужно
        * Этот макрос уменьшает высоту сканирования в зависимости от разницы высот портала
            ```cfg
            [gcode_macro QUAD_GANTRY_LEVEL]
            rename_existing: _QUAD_GANTRY_LEVEL
            gcode:
                SAVE_GCODE_STATE NAME=STATE_QGL
                BED_MESH_CLEAR
                {% if not printer.quad_gantry_level.applied %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_QGL
            ```

    ### Оптимизация сканирования платформы для `[z_tilt]`

        * Если у вас настроен `[z_tilt]`, добавьте следующий макрос в конфигурацию, если нет, то добавлять не нужно
        * Этот макрос уменьшает высоту сканирования в зависимости от разницы высот портала
            ```cfg
            [gcode_macro Z_TILT_ADJUST]
            rename_existing: _Z_TILT_ADJUST
            gcode:
                SAVE_GCODE_STATE NAME=STATE_Z_TILT
                BED_MESH_CLEAR
                {% if not printer.z_tilt.applied %}
                _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _Z_TILT_ADJUST horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_Z_TILT
            ```

    ### Оптимизация макроса сканирования платформы для вероятности помех от нагревательной платформы Eddy

        * Если вы используете мощную переменную нагревательную платформу (более 500 Вт)
        * Ниже приведен макрос, который отключает нагревательную платформу во время сканирования и включает ее после завершения
            ```cfg
            [gcode_macro BED_MESH_CALIBRATE]
            rename_existing: _BED_MESH_CALIBRATE
            gcode:
                {% set TARGET_TEMP = printer.heater_bed.target %}
                M140 S0
                _BED_MESH_CALIBRATE {rawparams}
                M140 S{TARGET_TEMP}
            ```

   # Шаги отладки:
   ## Первый шаг - изменение частоты

    ### Изменение частоты по умолчанию для ldc1612
    Этот метод применим только для sht36-v3/SB2040-v3 или SB2040-Pro-v3
    Можно использовать инструмент Mobaxterm для выполнения команд

    Скачайте необходимый <Button variant="contained" disableElevation href="/docs/ResDownload/auxiliary_software/">инструмент Mobaxterm</Button>

    [Подробное использование смотрите здесь](/docs/ProductDoc/SBC/fly-pi-v2/ssh)


    Выполните первую команду и нажмите Enter

    ```cfg
sed -i 's/LDC1612_FREQ = 12000000/LDC1612_FREQ = 40000000/g' ~/klipper/klippy/extras/ldc1612.py
    ```

    Выполните вторую команду и нажмите Enter
  ```cfg
grep "LDC1612_FREQ = 40000000" ~/klipper/klippy/extras/ldc1612.py
  ```
    Успешная замена вернет строку с `LDC1612_FREQ = 40000000`. Если нет возврата, замена не удалась.
    Или используйте эту команду для проверки успеха
    <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/1612.webp').default} size="110%" align="left" />
```cfg
sed -n '10,15p' ~/klipper/klippy/extras/ldc1612.py
```
