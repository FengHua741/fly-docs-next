# Использование Eddy

    :::info
    * При установке Eddy, расстояние от дна до сопла должно быть как минимум на 1-2 мм выше сопла
    * При калибровке частоты Eddy, катушка PCB должна быть на расстоянии не менее 5 мм от нагревательной платформы
    * Пожалуйста, убедитесь, что вся нагревательная платформа находится при комнатной температуре при тестировании, и не включайте нагревательную платформу во время тестирования

    :::

## Справочная конфигурация и инструкция по использованию
   
    ### Изменение стандартной частоты ldc1612
    Этот метод применяется только для sht36-v3/SB2040-v3 или SB2040-Pro-v3
    Можно использовать инструмент Mobaxterm для выполнения команд
    
    Скачать необходимый <Button variant="contained" disableElevation href="/docs/ResDownload/auxiliary_software/">инструмент Mobaxterm</Button>
    
    [Подробная инструкция см. здесь](/docs/ProductDoc/SBC/fly-pi-v2/ssh)


    Выполните первую команду и нажмите Enter

    ```cfg
sed -i 's/LDC1612_FREQ = 12000000/LDC1612_FREQ = 40000000/g' ~/klipper/klippy/extras/ldc1612.py
    ```

    Выполните вторую команду и нажмите Enter
  ```cfg
grep "LDC1612_FREQ = 40000000" ~/klipper/klippy/extras/ldc1612.py
  ```
    Если замена прошла успешно, будет возвращена строка с `LDC1612_FREQ = 40000000`. Если нет возврата, значит замена не удалась.
    Либо используйте эту команду для проверки успешности
    <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/1612.webp').default} size="110%" align="left" />
```cfg
sed -n '10,15p' ~/klipper/klippy/extras/ldc1612.py
```
   <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/ldc161240mhz.webp').default} size="30%" align="left" />

   Важно: После изменения используйте эту команду для перезапуска службы Klipper и повторного выполнения калибровки частоты Eddy
   ```cfg
   systemctl restart klipper
   ```
   Команда калибровки частоты Eddy
   ```cfg
   PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
   ```


    ### Настройка сканирования платформы

        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 43
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

        * Обратите внимание, что пока нет внешнего модуля 1612, эта конфигурация пока не работает
        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 42
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

    ### Настройка температурной компенсации

        ```cfg
        [temperature_probe fly_eddy_probe]
        sensor_type:Generic 3950
        sensor_pin:SHT36:gpio28
        ```

### Настройка bed_mesh

    * В [bed_mesh] параметр `horizontal_move_z` рекомендуется установить на 2, чтобы Eddy был как можно ближе к платформе при сканировании

    ```cfg
    [bed_mesh]
    horizontal_move_z: 2
    ```
