# Использование Eddy

    :::info
    * При установке Eddy, дно должно находиться на высоте не менее 1-2 мм над соплом
    * При калибровке частоты Eddy, катушка PCB должна быть на расстоянии не менее 5 мм от нагревательного стола
    * Пожалуйста, не проводите тесты при комнатной температуре нагревательного стола и не включайте его во время тестов

    :::

## Рекомендуемая конфигурация и руководство по использованию

##### Добавьте следующую конфигурацию в файл `printer.cfg` для отладки Eddy

    ### Конфигурация сканирования стола

        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 43
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

        * Обратите внимание, что в настоящее время нет внешнего модуля 1612, и данная конфигурация пока не может быть использована
        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 42
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

    ### Конфигурация температурной компенсации

        ```cfg
        [temperature_probe fly_eddy_probe]
        sensor_type:Generic 3950
        sensor_pin:SHT36:gpio28
        ```

    ## Команды для сканирования стола
    
        * Добавьте следующий макрос в конфигурацию `printer.cfg`
            ```bash
            [gcode_macro BED_MESH_CALIBRATE]
             rename_existing: _BED_MESH_CALIBRATE
             gcode: 
                   _BED_MESH_CALIBRATE METHOD=rapid_scan {rawparams}
            ```
        * Команды для сканирования стола в локальной сети, убедитесь, что Klipper может использовать функцию исключения объектов, рекомендуется добавить в макрос запуска
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan adaptive=1 
            ```

### Конфигурация bed_mesh

    * Параметр `horizontal_move_z` в [bed_mesh] рекомендуется установить на 2, чтобы Eddy был как можно ближе к платформе при сканировании

    ```cfg
    [bed_mesh]
    horizontal_move_z: 2
    ```

## Макросы для предотвращения столкновений

    ### Оптимизация сканирования стола с `[quad_gantry_level]`
        * Если у вас настроен `[quad_gantry_level]`, добавьте ниже макрос в конфигурацию, если нет, то не нужно
        * Этот макрос уменьшает высоту сканирования в зависимости от разницы высот портала
            ```cfg
            [gcode_macro QUAD_GANTRY_LEVEL]
            rename_existing: _QUAD_GANTRY_LEVEL
            gcode:
                SAVE_GCODE_STATE NAME=STATE_QGL
                BED_MESH_CLEAR
                {% if not printer.quad_gantry_level.applied %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_QGL
            ```

    ### Оптимизация сканирования стола с `[z_tilt]`

        * Если у вас настроен `[z_tilt]`, добавьте ниже макрос в конфигурацию, если нет, то не нужно
        * Этот макрос уменьшает высоту сканирования в зависимости от разницы высот портала
            ```cfg
            [gcode_macro Z_TILT_ADJUST]
            rename_existing: _Z_TILT_ADJUST
            gcode:
                SAVE_GCODE_STATE NAME=STATE_Z_TILT
                BED_MESH_CLEAR
                {% if not printer.z_tilt.applied %}
                _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _Z_TILT_ADJUST horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_Z_TILT
            ```

    ### Макрос оптимизации сканирования стола при сильном электромагнитном воздействии нагревательного стола

        * Если вы используете мощный AC нагревательный стол (более 500 Вт)
        * Следующий макрос отключает нагревательный стол во время сканирования и снова включает его после завершения
            ```cfg
            [gcode_macro BED_MESH_CALIBRATE]
            rename_existing: _BED_MESH_CALIBRATE
            gcode:
                {% set TARGET_TEMP = printer.heater_bed.target %}
                M140 S0
                _BED_MESH_CALIBRATE {rawparams}
                M140 S{TARGET_TEMP}
            ```

# Шаги отладки:
## Шаг 1: Изменение частоты

    ### Изменение частоты по умолчанию для ldc1612
    Этот метод используется только для sht36-v3/SB2040-v3 или SB2040-Pro-v3
    Вы можете использовать инструмент Mobaxterm для выполнения команд
    
    Скачайте необходимый <Button variant="contained" disableElevation href="/docs/ResDownload/auxiliary_software/">Инструмент Mobaxterm</Button>
    
    [Подробные инструкции см. здесь](/docs/ProductDoc/SBC/fly-pi-v2/ssh)


    Выполните первую команду и нажмите Enter

    ```cfg
sed -i 's/LDC1612_FREQ = 12000000/LDC1612_FREQ = 40000000/g' ~/klipper/klippy/extras/ldc1612.py
    ```

    Выполните вторую команду и нажмите Enter
    ```cfg
grep "LDC1612_FREQ = 40000000" ~/klipper/klippy/extras/ldc1612.py
    ```
    Если замена прошла успешно, вернется строка с `LDC1612_FREQ = 40000000`. Если ничего не возвращено, то замена не удалась.
    Или используйте эту команду для проверки успеха замены
    <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/1612.webp').default} size="110%" align="left" />
```cfg
sed -n '10,15p' ~/klipper/klippy/extras/ldc1612.py
```
