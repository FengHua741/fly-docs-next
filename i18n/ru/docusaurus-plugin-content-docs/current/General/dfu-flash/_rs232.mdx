* Убедитесь, что ПК может нормально подключаться к сети
* Убедитесь, что нижестоящее устройство подключено к ПК с кабелем, поддерживающим передачу данных
* После подключения по SSH к ПК введите команду `lsusb` и нажмите Enter. Возможно, появится одно из следующих трех состояний
    * Отображаются опознанные USB-устройства
    * Появляется сообщение об отсутствии команды `ls`, можно выполнить эту команду для установки
        ```bash
        sudo apt-get install usbutils
        ```
    * Нет ошибок, никакой другой информации (возможна проблема системы, требуется замена системы)
* Если после ввода команды `lsusb` не происходит никакой реакции, это указывает на системную проблему, и мы бессильны, вам нужно заменить рабочую систему
* Если появляется информация, заключенная в круги на нижеследующем изображении, можно переходить к следующему шагу

    <ImageView image={require('@site/docs/General/get-id/img/df11.webp').default} size="80%" align="left" />

* `0843:df11` относится к устройству, которое вы будете использовать. Последующие подсказки можно игнорировать, так как проблемы системы могут привести к частичному отображению или даже полному отсутствию отображения
* Если нет, проверьте самостоятельно, надежно ли подсоединен кабель или попала ли плата в DFU

:::warning Важные замечания

Продолжайте только при наличии `0843:df11`

:::
