## Поиск ID
    * В командной строке введите `lsusb` и нажмите Enter. Если появляется информация, выделенная на изображении, можно переходить к следующему шагу.
        * Если никакой информации не отображается, это проблема системы верхнего уровня, с которой мы не можем справиться. Вам нужно заменить рабочую систему или верхний уровень.
        * Если появляется сообщение об отсутствии команды `lsusb`, выполните эту команду для установки:
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" />
    * `1d50:6177` относится к устройству, которое вы будете использовать. Заданный суффикс `OpenMoko, Inc. stm32f407xx` имеет лишь информативное значение и не является критерием определения.
        * Некоторые верхние уровни могут показывать неполные или недоступные данные из-за системных проблем.
        * Если при вводе команды `lsusb` устройство видно, но нет `1d50:6177`, попробуйте заменить кабель или проверьте положение переключателей.

    :::warning Внимание
       Действия можно продолжать только после обнаружения устройства `1d50:6177`.
    :::

## Поиск USBID

    ```bash
    ls /dev/serial/by-id/*
    ```

    * После ввода команды должен появиться один синий ID (ниже приведен пример).
    <ImageView image={require('@site/docs/General/get-id/img/fasta_id.webp').default} size="80%" align="left" />
    * Если после ввода команды ID не появляется, но устройство видно через `lsusb`, попробуйте заменить верхний уровень или систему верхнего уровня.

:::warning Внимание
Если появляются несколько ID, выберите соответствующий вашей модели материнской платы MCU.
:::

### Заполнение USB ID в конфигурации

    * Откройте веб-интерфейс принтера, найдите раздел конфигурации в левом боковом меню. Для fluidd это `{...}`, для mainsail — «Машина», затем откройте `printer.cfg`.

        <ImageView image={require('@site/docs/General/get-id/img/fluidd-cfg.webp').default} size="50%" align="left" />

    * Скопируйте синий ID и вставьте его в параметр `serial:` узла `[mcu]` в файле конфигурации `printer.cfg`. После сохранения и перезагрузки устройство подключится к материнской плате. Если Klipper выводит сообщение об ошибке `ADC out of range`, это нормально. Подключите нагревательную платформу и термистор, настройте пины термисторов и выходов для сопла и платформы, затем сохраните и перезагрузите снова.

        :::danger

        * Ниже приведена ссылка на конфигурацию материнской платы MCU ID для примера.
        * Если речь идет о ID материнской платы, обязательно добавьте `serial:` к `[mcu]` и вставьте найденный ID.
        * Обратите внимание, что `/dev/serial/by-id/` нельзя опустить.
        :::
            <ImageView image={require('@site/docs/General/get-id/img/cfg-usb-id.webp').default} size="100%" align="left" />

:::danger
Важно: Все ID, указанные в документации, являются примерами. На самом деле каждый ID материнской платы различен. Пожалуйста, заполните актуальные полученные ID.
:::
