## Заполнение CAN ID

    1. Закройте раздел `DEVICES` и заполните ID
    :::danger Важно

    * Перед # в первом красном прямоугольнике необходимо удалить пробелы, а также изменить `serial:` на `canbus_uuid:`
    * В стандартной конфигурации FLY может присутствовать `[MCU]`, но если ID ниже является `serial: /tmp/klipper_host_mcu`, нужно переименовать `[MCU]` в `[mcu HOST]` или удалить эти две строки
    * Для копирования можно использовать следующий образец `canbus_uuid:`
    :::

    ```cfg
    [mcu]
    canbus_uuid:
    ```

      <tr>

          - В fluidd найдите [MCU]

          - В mainsail найдите [MCU]

    2. Впишите ID в конфигурацию
    :::tip

    * После ввода ID нажмите `SAVE & RESTART` в правом верхнем углу
    :::

    <table>
      <tr>
        <td>
          <ImageView image={require('@site/docs/General/get-id/img/wed/fluidd_can_save.webp').default} size="100%" align="center" />

        <td>

        </td>
      </tr>
    </table>
    3. Если Klipper выдает сообщение `ADC out of range`, это нормально. Подключите нагревательный стол и термодатчик к материнской плате и настройте выводные пины для сопла и нагревательного стола, затем сохраните и перезагрузите систему.

:::danger
Внимание: Все ID, указанные в документе, являются примерами. У каждого материнского компонента ID уникален. Пожалуйста, используйте фактические полученные ID.
:::
