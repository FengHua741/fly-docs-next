# Часто задаваемые вопросы о FlyOS-Fast

## Вопрос: При выполнении команды появляется сообщение `command not found`
- **Причина**: Система не содержит эту команду.
- **Решение**: В системе уже предустановлены многие распространенные пакеты. Если вам нужен другой пакет, отправьте его нам. Если это необходимо, он будет включен в следующую версию.

## Вопрос: Почему нельзя выполнить команду `apt update`?
- **Причина**: FlyOS-Fast — это облегченная и быстрая система, которая не имеет менеджера пакетов apt, поэтому команда `apt update` недоступна.
- **Решение**: В системе уже предустановлены многие распространенные пакеты. Если вам нужен другой пакет, отправьте его нам. Если это необходимо, он будет включен в следующую версию.

## Вопрос: При выполнении команды записи появляется сообщение `-bash: ~/klippy-env/bin/python: No such file or directory`
- **Причина**: FlyOS-Fast не использует виртуальную среду venv.
- **Решение**: Замените `~/klippy-env/bin/python` на `python` для выполнения.

## Вопрос: Почему нельзя войти как пользователь `fly`?
- **Причина**: FlyOS-Fast имеет только пользователя root, все операции выполняются под этим пользователем.
- **Решение**: В системе нет пользователя `fly`. Для входа просто используйте пользователя `root`.

## Вопрос: Почему не открывается fly-tools на порту 9999?
- **Причина**: FlyOS-Fast не устанавливает fly-tools.
- **Решение**: Fluidd и mainsail уже поддерживают запрос ID, а все конфигурации экранов находятся в [системной конфигурации](/docs/DebugDoc/flyos-fast/config).

## Вопрос: Почему на экране показывается только логотип, но не отображается интерфейс KlipperScreen?
- **Причина**: KlipperScreen не включен.
- **Решение**: Включите KlipperScreen в [системной конфигурации](/docs/DebugDoc/flyos-fast/config), установив значение `klipper_screen` равным `1`.

## Вопрос: Почему система не предоставляет веб-интерфейс для обновления компонентов klipper и других?
- **Причина**: Компоненты klipper, moonraker и другие встроенные в FlyOS-Fast являются версиями с дополнительной разработкой.
- **Решение**: Если нужно, вы можете самостоятельно обновить их через командную строку, но не рекомендуется этого делать. Дождитесь обновления системы.

## Вопрос: Почему скрипты установки сторонних компонентов, таких как Happy Hare, не работают?
- **Причина**: По умолчанию пользователь в FlyOS-Fast — это `root`, а эти скрипты проверяют наличие root и не выполняют установку.
- **Решение**: Просто измените скрипт так, чтобы он обходил проверку root. Подробнее можно посмотреть [здесь](https://github.com/kluoyun/Happy-Hare/tree/flyos-dev). Остальные скрипты имеют аналогичные изменения.
