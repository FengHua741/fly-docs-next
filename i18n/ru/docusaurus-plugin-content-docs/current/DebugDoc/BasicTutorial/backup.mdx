# Метод резервного копирования конфигурации

:::danger Важно

* Данная инструкция применима для FLY-Pi, Raspberry Pi, Orange Pi и других хост-компьютеров.
* Если используется EMMC, необходимо включить режим записи, иначе данная инструкция не будет применима.
* Данная инструкция основана на операциях под Windows. Пожалуйста, не форматируйте неизвестные диски и т.д.
* Не форматируйте!!! Не форматируйте!!! Не форматируйте!!!

:::

## Скачивание необходимого программного обеспечения

    * Щелкните, чтобы перейти на страницу загрузки, на которой вам предложат подходящую версию для вашего компьютера.
    * Программа является портативной версией, не требует установки. После скачивания просто распакуйте её для использования.
    * <Button variant="contained" disableElevation component="a" rel="noopener" href="https://www.diskgenius.cn/download.php">Инструмент управления жёсткими дисками DG</Button>

## Начало резервного копирования

    :::info
    * Не форматируйте!!!
    * После форматирования восстановление невозможно!!!
    * Если это TF-карта, просто подключите её к компьютеру.
    * Если это EMMC, необходимо вручную включить режим записи!!!
    :::

### Подключение
    * Подключите систему, которую нужно скопировать, к компьютеру.
    * Откройте `DiskGenius.exe`.
    * Если всё сделано правильно, отобразится нужный вам диск для резервного копирования.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/boot.webp').default} size="80%" align="center" />

### Поиск
    * В левой части щелкните по основному разделу, затем нажмите "Просмотр файлов" сверху, после чего появятся файлы снизу.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/home.webp').default} size="80%" align="center" />
    * Найдите папку `home` и дважды щелкните по ней.
    * После этого обычно будет одна папка, имя которой — ваше имя пользователя системы.
    * Если их две или более, вам нужно самостоятельно определить, какая содержит ваши системные настройки.
    * Затем дважды щелкните по нужной папке.
    * После этого найдите папку `printer_data`.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/printer-data.webp').default} size="80%" align="center" />
    * Папка `printer_data` содержит ваши печатные файлы, конфигурацию klipper, журналы klipper и другие важные данные. Вы можете экспортировать всё содержимое целиком.

### Экспорт конфигурации
    * Щелкните правой кнопкой мыши по папке `printer_data`, и появится нижеследующее уведомление.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/export.webp').default} size="40%" align="center" />
    * Выберите папку для сохранения или экспортируйте напрямую.
    * Подтвердите экспорт.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/export1.webp').default} size="80%" align="center" />
    * Откройте экспортированный файл и найдите папку `config`. Здесь находятся ваши системные настройки.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/export2.webp').default} size="80%" align="center" />
    * Экспорт конфигурации завершён.
    * Обратите внимание, что файл `moonraker` не следует загружать в другую систему, так как из-за различий в системах klipper может не подключиться.
