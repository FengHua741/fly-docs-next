* 上位機が正常にネットワークに接続でき、下位機と接続されているデータ線がデータ転送機能を持っていることを確認してください。
* SSHで上位機に接続した後、`lsusb`と入力してEnterを押します。もし以下の画像で囲まれた情報が出力されたら、次のステップに進んでください。
    * 何も情報が返ってこない場合、これは上位機のシステム問題であり、こちらでは対応できません。正常なシステムに変更するか、上位機を交換してください。
    * `lsusb`コマンドがないと表示される場合、このコマンドでインストールできます。
        ```bash
        sudo apt-get install usbutils
        ```

* `1d50:614e`は今回使用するデバイスに該当します。`OpenMoko, Inc. stm32f407xx`のサフィックスは参考価値がありますが、判断基準ではありません。
    * 一部の上位機はシステムの問題により、完全に表示されない場合があります。
    * `lsusb`コマンドを入力するとデバイスが表示されるが、`1d50:614e`が表示されない場合は、データ線を交換し、マザーボードを上位機の他のUSBポートに接続してみてください。
* 上記の手順を以前に実行し、成功してKlipperファームウェアを書き込み、マザーボードが正常に動作しているが、Klipperファームウェアを更新する必要がある場合は、「ファームウェア更新」セクションを確認してください。
