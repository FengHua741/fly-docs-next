## USB IDを検索する

1. ブラウザを開き、URLバーに上位機のIPアドレスを入力します。例えば、私の上位機のIPアドレスが`192.168.101.179`の場合、それを直接入力してEnterキーを押すだけです。
  <ImageView image={require('@site/docs/General/get-id/img/wed/ip.webp').default} size="80%" align="center" /> 

2. 上位機のWEBインターフェースを開き、左側のサイドバーにある設定オプションで以下の内容を探します：

## USB IDの入力

    1. `DEVICES`を閉じてIDを入力します

    :::danger 注意
      * 設定ファイルで以下を変更してください:  
      ```
      [mcu] 
      serial: /tmp/klipper_host_mcu
      ```  
      に変更して:
      ```
      [mcu host] 
      serial: /tmp/klipper_host_mcu
      ```
      * 追加してください:
      ```
      [mcu]
      serial: <ここに先ほど取得したIDを入力>
      ```
      :::
    <table>
      <tr>
        <td>
          - fluiddで[MCU]を見つけます
          <ImageView image={require('@site/docs/General/get-id/img/wed/fluidd_cfg.webp').default} size="100%" align="center" />

        <td>

        </td>
      </tr>

    2. IDを設定に記入します
    :::tip

    * IDを入力したら右上の`SAVE & RESTART`をクリックしてください
    :::
    <table>
      <tr>
        <td>
          <ImageView image={require('@site/docs/General/get-id/img/wed/fluidd_save.webp').default} size="100%" align="center" />
        </td>

      </tr>

    3. Klipperが`ADC out of range`と表示された場合、これは正常な現象です。ヒートベッドやセンサーをメインボードに接続し、ヒーターとヒートベッドのセンサーピンを設定してから保存して再起動してください。

:::danger
注意: ドキュメント中に登場するすべてのIDは例示です。実際の各メインボードのIDは異なりますので、実際に取得したIDを入力してください。
:::
