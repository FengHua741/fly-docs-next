## Z—オフセットの微調整方法
* 第一步：Pursa/OrcaSlicer/Cura/Bambu Studioを使用して、200x200、高さ0.2の方形スライスを印刷し、Z—オフセットを微調整する
* 第二步：プリンタのIPで`fluidd` `mainsail`にアクセスし、ダッシュボードの`移動制御`でZ軸オフセットをさらに微調整し、その値を記憶する
<ImageView image={require('@site/docs/DebugDoc/faq/img/Zoff.webp').default} size="80%" align="left" />
* 第三步：以下のコマンドでKlipperを再起動する
```bash
FIRMWARE_RESTART
```
* 第四步：以下のコマンドでZ—オフセットのUI画面を調整し、印刷中に調整した値を設定する
```bash
G28
PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
```
<ImageView image={require('@site/docs/DebugDoc/faq/img/UI.webp').default} size="50%" align="left" />

:::info
* エラー`Eddy current sensor error`が出た場合は、Eddyの設定内のz_offset値を大きくしてください。図を参照

<ImageView image={require('@site/docs/DebugDoc/faq/img/EDDYZOFF.webp').default} size="50%" align="left" />
:::

## 操作説明

    :::warning 注意事項

    * 電流校正を先に行い、その後にコイル周波数校正を行う必要があります
    * 複数のZ軸を持つプリンターは、手動で一度レベル調整し、全ての校正が完了した後に再度レベル調整を行い、再び校正します
    * 下位機のファームウェアはKliiperバージョンと完全に一致させる必要があります

    :::

    * 電流校正を先に行い、その後にコイル周波数校正を行う必要があります

    * `FUIDD`ウェブページに下記の設定を追加してから、後続の操作を行ってください
        ```cfg
        [force_move]
        enable_force_move: true
        ```

    ### 第一步：EDDY 電流の校正

        * 1: コンソールに下記のコマンドを入力します
            ```bash
            SET_KINEMATIC_POSITION z=80
            ```
        * 2: XとYを原点に戻し、ツールヘッドをベッドの中央に移動します
            ```bash
            G28 X Y
            ```
        * 上記の手順が完了した後、Eddy電流を校正する必要があります。まずEDDYをベッドの中央に移動させ、ノズルがベッドに触れる直前まで下げます。その後、コンソールで下記のコマンドを実行し、自動的に電流を取得します。最後に`SAVE_CONFIG`でパラメータを保存します。フィードバック値が約15なら正常です。0または30が返ってくる場合は、接続不良を確認してください。
            ```bash
            LDC_CALIBRATE_DRIVE_CURRENT CHIP=fly_eddy_probe
            ```
        * 正常な戻り値は`15`前後であるべきです
        * 校正が完了したら`SAVE_CONFIG`でパラメータを保存します

    ### 第二步：Eddy 周波数の校正

        :::info

        * Eddyを設置する際、ノズルの位置から少なくとも1-2 mm上部に配置する必要があります
        * Eddy周波数を校正する際、コイルPCBはベッドから少なくとも5mm離しておく必要があります

        :::

        * その後、ノズルをプラットフォームの中央に移動します（注意：このステップでは、プリンタにheightmapがロードされていないことを確認してください）
        * 手動Zオフセット校正を開始します [Paper test](https://www.klipper3d.org/Bed_Level.html#the-paper-test)
            ```bash
            PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
            ```
        * 校正が完了したら`SAVE_CONFIG`でパラメータを保存します
        :::info
        * `z_tilt`または`quad_gantry_level（QGL）`機能を持つプリンタは、まず平行度を調整してください。i3はX軸の平行度を、三叉戟はベッドの平行度を、Voron 2.4はガントリーの平行度を調整してから、もう一度Eddy周波数を校正してください。
        :::
    ### 第三步：温度補償

        * 全ての軸を原点に戻します
       ```bash
       G28
       ```

        * `quad_gantry_level`または`z_tilt`がある場合、まず`Z_TILT_ADJUST`または`QUAD_GANTRY_LEVEL`を実行してから、もう一度原点に戻します
        * コンソールでG0 Z5を入力するか、ウェブページなどでZ軸をベッド上方5mmに移動します
        * 以下のコードを実行して、プリンタの`idle timeout`を長く設定し、温度上昇中にタイムアウトしないようにします
            ```bash
            SET_IDLE_TIMEOUT TIMEOUT=65000
            ```
        * 以下のコマンドで校正を開始します
            ```bash
            TEMPERATURE_PROBE_CALIBRATE PROBE=fly_eddy_probe TARGET=56 STEP=4
            ```
        :::warning 注意事項

        * ヒント：上記のコマンドでは、EDDyの目標温度を`56℃`に設定しており、多くのデバイスに適しています。
        * あなたの3Dプリンタの筐体の実際の温度が高い場合、目標温度を適宜上げることができます。
        * 目標温度が高いほど、校正に必要な時間が長くなります。なぜなら、Eddyが温度平衡に達するまでに時間がかかるからです。

        :::
    
        1. この操作を実行すると、UIにZ軸調整ボックスが表示されます。紙をノズルとベッドの間に挟んで、手動 [Z軸オフセット校正（Paper Test）方法](https://www.klipper3d.org/Bed_Level.html#the-paper-test)を使用し、この値を確認してください。
        2. 値を受け入れた後、ベッド温度を最高に、ノズル温度を220℃に設定します。
        3. エアコンや窓が開いている部屋で作業する場合は、Eddyの温度が上昇するのを確実にするために、エアコンや窓を閉めることをお勧めします。風が温度上昇に影響を与えるからです。
        4. Eddyの温度が上昇するにつれて、システムは自動的に4℃ごとに手動Zオフセット校正を促します。ベッド温度が非常に高いため、火傷に注意してください!!!
        5. 手動Zオフセット校正（Paper Test）を繰り返して、校正が完了するまで続けます。Eddyの温度が上昇しなくなった場合は、下記のコマンドを使用して校正を早期に終了できます。
        
        * ドリフト校正中に使用可能な追加のgcodeコマンドには、TEMPERATURE_PROBE_NEXTがあります。
            * `TEMPERATURE_PROBE_NEXT`    EDDyが設定された目標温度`56℃`に達する前に強制的にEDDyが新しいデータをサンプリングするために使用されます。
            * `TEMPERATURE_PROBE_COMPLETE`    EDDyが設定温度`56℃`に達しない場合に校正を完了するために使用されます。
            * `ABORT`  校正を中止し、結果を無視するために使用されます。
        * これらの手順を完了すると、Eddyは幅広い温度範囲で優れた最初の層のプリント結果を提供できるようになります！
## 各種最適化マクロ

    ### `[quad_gantry_level]`のスキャンベッド最適化
        * `quad_gantry_level`が設定されている場合は、下記のマクロを設定に追加してください。設定されていない場合は追加する必要はありません。
        * このマクロは、ガントリーの高低差に応じて探査高さを下げます
            ```cfg
            [gcode_macro QUAD_GANTRY_LEVEL]
            rename_existing: _QUAD_GANTRY_LEVEL
            gcode:
                SAVE_GCODE_STATE NAME=STATE_QGL
                BED_MESH_CLEAR
                {% if not printer.quad_gantry_level.applied %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_QGL
            ```

    ### `[z_tilt]`のスキャンベッド最適化

        * `z_tilt`が設定されている場合は、下記のマクロを設定に追加してください。設定されていない場合は追加する必要はありません。
        * このマクロは、ガントリーの高低差に応じて探査高さを下げます
            ```cfg
            [gcode_macro Z_TILT_ADJUST]
            rename_existing: _Z_TILT_ADJUST
            gcode:
                SAVE_GCODE_STATE NAME=STATE_Z_TILT
                BED_MESH_CLEAR
                {% if not printer.z_tilt.applied %}
                _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _Z_TILT_ADJUST horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_Z_TILT
            ```

    ### スキャンベッド最適化

        * 大出力ACベッド（500W以上）を使用する場合
        * 下記のマクロは、スキャンベッド時にベッドをオフにして、完了後に再加熱します
            ```cfg
            [gcode_macro BED_MESH_CALIBRATE]
            rename_existing: _BED_MESH_CALIBRATE
            gcode:
                {% set TARGET_TEMP = printer.heater_bed.target %}
                M140 S0
                _BED_MESH_CALIBRATE {rawparams}
                M140 S{TARGET_TEMP}
            ```
    ## ネットベッドコマンド
    
        * 全体のネットベッドをスキャンするコマンド
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan
            ```
        * 局所ネットベッドコマンド。Klipperがオブジェクト除外機能を使用できることを確認してください。開始マクロに追加することをお勧めします。
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan adaptive=1 
            ```
