## Z—オフセットの微調整方法
* 第一步：Pursa/OrcaSlicer/Cura/Bambu Studioを使用して、200x200高さ0.2の四角いスライスを切ってZ—オフセットの微調整のために印刷します
* 第二步：プリンターのIPを入力して`fluidd` `mainsail`にアクセスし、ダッシュボードの`移動制御`でZ軸オフセットをさらに微調整し、微調整した数値を記憶します
<ImageView image={require('@site/docs/DebugDoc/faq/img/Zoff.webp').default} size="80%" align="left" />
* 第三步：以下のコマンドでKlipperを再起動します
```bash
FIRMWARE_RESTART
```
* 第四步：以下のコマンドでZ—オフセットの調整UIを表示し、印刷中に調整した数値を入力します
```bash
G28
PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
```
<ImageView image={require('@site/docs/DebugDoc/faq/img/UI.webp').default} size="50%" align="left" />

:::info
* エラー`Eddy current sensor error`が発生した場合は、eddy設定のz_offsetの数値を大きくしてください。図を参照してください

<ImageView image={require('@site/docs/DebugDoc/faq/img/EDDYZOFF.webp').default} size="50%" align="left" />
:::

:::info
 * 上記の操作を行ってもベッドスキャンが失敗する場合は、Zリミットを仮想リミットに変更してください
:::


## 操作説明

    :::warning 注意事項

    * 電流校正を先に行い、次にコイル周波数校正を行ってください
    * 複数のZ軸を持つプリンターは、手動で一度レベルを調整し、すべての校正を行った後、再度レベルを調整してから再校正してください
    * 下位機器のファームウェアはKlipperのバージョンと完全に一致している必要があります

    :::

    :::info

    * 電流校正を先に行い、次にコイル周波数校正を行ってください
    * 複数のZ軸を持つプリンターは、手動で一度レベルを調整し、すべての校正を行った後、再度レベルを調整してから再校正してください
    * 下位機器のファームウェアはKlipperのバージョンと完全に一致している必要があります

    :::


    * `FUIDD`ウェブページで下記の設定を追加してから、以下の操作を行ってください
        ```cfg
        [force_move]
        enable_force_move: true
        ```

    ### 第一步：EDDY電流の校正

        * 1:コンソールに下記のコマンドを入力します
            ```bash
            SET_KINEMATIC_POSITION z=80
            ```
        * 2:XとYの帰位を行い、ツールヘッドをベッドの中央に移動します
            ```bash
            G28 X Y
            ```
        * 上記の手順が完了したら、Eddy電流を校正する必要があります。まず、EDDYをベッドの中央に移動し、ノズルがベッドに触れるまで下げます。次に、コンソールで以下のコマンドを実行して自動的に電流を取得し、最後に`SAVE_CONFIG`でパラメータを保存します。フィードバック値が約15であれば正常です。0または30の場合、接触不良を確認してください（接点不良）
            ```bash
            LDC_CALIBRATE_DRIVE_CURRENT CHIP=fly_eddy_probe
            ```
        * 正常な戻り値は`15`前後です
        * 校正が完了したら`SAVE_CONFIG`でパラメータを保存します

    ### 第二步：EDDY周波数の校正


        :::info

        * Eddyをインストールする際、底部がノズルに対して少なくとも1-2 mm上にある必要があります
        * Eddy周波数を校正する際、コイルPCBはベッドから少なくとも5mm離れている必要があります

        :::

        * 次に、ノズルをプラットフォームの中央に移動します（注意：このステップではheightmapがロードされていないことを確認してください）
        * 手動Zオフセット校正を開始します [Paper test](https://www.klipper3d.org/Bed_Level.html#the-paper-test)
            ```bash
            PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
            ```
        * 校正が完了したら`SAVE_CONFIG`でパラメータを保存します
        :::info
        * `z_tilt`または`quad_gantry_level（QGL）`機能を持つプリンターは、まず平行度を調整してください。i3はX軸の平行度を、三叉戟はベッドの平行度を、Voron 2.4は龍門架の平行度を調整してから、Eddy周波数を再校正してください
        :::
    ### 第三步：温度補正

        * すべての軸を帰位します
       ```bash
       G28
       ```

        * `quad_gantry_level`または`z_tilt`がある場合は、まず`Z_TILT_ADJUST`または`QUAD_GANTRY_LEVEL`を一度実行し、次に帰位します
        * コンソールにG0 Z5と入力するか、ウェブページなどを通じてZ軸をベッドの上方`5mm`に移動します
        * 以下のコードを実行して、プリンターの`idle timeout`を長く設定し、温度上昇中にタイムアウトしないようにします
            ```bash
            SET_IDLE_TIMEOUT TIMEOUT=65000
            ```
        * 以下のコマンドを実行して校正を開始します
            ```bash
            TEMPERATURE_PROBE_CALIBRATE PROBE=fly_eddy_probe TARGET=56 STEP=4
            ```
        :::warning 注意事項

        * 注意：上記のコマンドで、EDDYの目標温度は`56℃`に設定されています。これは多くのデバイスに適しています。
        * 3Dプリンターの実際の温度が高い場合は、目標温度を適宜上げてください。
        * 目標温度が高いほど、校正にかかる時間が長くなります。EDDYが温度平衡に達するまで時間がかかるためです。

        :::
    
        1. この操作を実行すると、UIにZ軸の調整ボックスが表示されます。上記で述べた手動 [Z軸オフセット校正（Paper Test）方法](https://www.klipper3d.org/Bed_Level.html#the-paper-test)を使用して、ノズルとベッドの間に紙を挟んでください。
        2. 値を確認したら、ベッド温度を最高に、ノズル温度を220℃に設定します。
        3. 空調や窓が開いている部屋で作業する場合は、Eddyの温度が上昇するために空調や窓を閉めることをお勧めします。風が温度上昇に影響を与える可能性があるためです。
        4. Eddyの温度が上昇するにつれて、システムは自動的に4℃ごとに手動Zオフセット校正を実行するように促します。ベッド温度が非常に高いため、火傷に注意してください!!!
        5. 手動Zオフセット校正（Paper Test）を繰り返して校正が完了するまで続けます。Eddyの温度が上昇しなくなった場合、下記の関連コマンドを使用して校正を早期に終了できます。
        
        * ドリフト校正中で使用可能な追加のgcodeコマンドには、TEMPERATURE_PROBE_NEXTが含まれます。
            * `TEMPERATURE_PROBE_NEXT`    EDDYが設定した目標温度`56℃`に達する前に強制的に新しいデータをサンプリングするために使用されます。
            * `TEMPERATURE_PROBE_COMPLETE`    EDDYが設定温度`56℃`に達しない場合に校正を完了するために使用されます。
            * `ABORT`  校正を中止し、結果を無視するために使用されます。
        * これらのステップを完了すると、Eddyは広範な温度範囲で優れた最初の層の印刷効果を提供できるようになります！
## 各種最適化マクロ

    ### `[quad_gantry_level]`スキャン最適化
        * `quad_gantry_level`が設定されている場合は、下記のマクロを設定に追加してください。設定されていない場合は追加する必要はありません
        * このマクロは、龍門の高低差に応じて探針の高さを下げます
            ```cfg
            [gcode_macro QUAD_GANTRY_LEVEL]
            rename_existing: _QUAD_GANTRY_LEVEL
            gcode:
                SAVE_GCODE_STATE NAME=STATE_QGL
                BED_MESH_CLEAR
                {% if not printer.quad_gantry_level.applied %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_QGL
            ```

    ### `[z_tilt]`スキャン最適化

        * `z_tilt`が設定されている場合は、下記のマクロを設定に追加してください。設定されていない場合は追加する必要はありません
        * このマクロは、龍門の高低差に応じて探針の高さを下げます
            ```cfg
            [gcode_macro Z_TILT_ADJUST]
            rename_existing: _Z_TILT_ADJUST
            gcode:
                SAVE_GCODE_STATE NAME=STATE_Z_TILT
                BED_MESH_CLEAR
                {% if not printer.z_tilt.applied %}
                _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _Z_TILT_ADJUST horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_Z_TILT
            ```

    ### スキャン最適化

        * 大出力のACベッド（500W以上）を使用している場合
        * 下記のマクロは、スキャン中にベッドをオフにして、スキャンが完了した後に再加熱します
            ```cfg
            [gcode_macro BED_MESH_CALIBRATE]
            rename_existing: _BED_MESH_CALIBRATE
            gcode:
                {% set TARGET_TEMP = printer.heater_bed.target %}
                M140 S0
                _BED_MESH_CALIBRATE {rawparams}
                M140 S{TARGET_TEMP}
            ```
    ## ネットベッドコマンド
    
        * `printer.cfg`の設定に下記のマクロを追加してください
            ```bash
            [gcode_macro BED_MESH_CALIBRATE]
             rename_existing: _BED_MESH_CALIBRATE
             gcode: 
                   _BED_MESH_CALIBRATE METHOD=rapid_scan {rawparams}
            ```
        * ローカルネットベッドコマンドでは、Klipperがオブジェクト排除機能を使用できることを確認してください。開始マクロに追加することをお勧めします
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan adaptive=1 
            ```
