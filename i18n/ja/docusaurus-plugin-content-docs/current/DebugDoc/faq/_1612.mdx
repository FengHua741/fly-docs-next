### 第二ステップ：温度補正
:::info
* 温度補正を行う前に、ヒートベッドとノズルは加熱しないでください。
:::

* 下の画像に示されているマクロコマンド`TEMP_COMPENSATION`をクリックして実行します。
<ImageView image={require('@site/docs/DebugDoc/faq/img/temp.webp').default} size="50%" align="left" />
<ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/eddy6.webp').default} size="50%" align="left" />

1. この操作を行った後、UIはZ軸調整ボックスを表示します。上述した手動[Z軸オフセットキャリブレーション（Paper Test）方法](https://www.klipper3d.org/Bed_Level.html#the-paper-test)を使用して、紙をノズルとベッドの間に挟み、その値を確認します。
2. 値を受け入れた後、ヒートベッドの温度を最大にし、ノズルの温度を250℃に設定します。
3. 部屋にエアコンや窓がある場合、Eddyの温度上昇を確保するためにエアコンを切るか窓を閉めてください。風は温度の上昇を妨げます。
4. Eddyの温度が上がると、システムは手動Zオフセットキャリブレーションを3℃ごとに自動的に指示します。ヒートベッドの温度は非常に高いため、注意してください！！
5. 手動Zオフセットキャリブレーション（Paper Test）を繰り返し、キャリブレーションが完了するまで行います。Eddyの温度が上がらなくなった場合は、以下の関連コマンドを使用してキャリブレーションを事前に終了できます。

* ドリフトキャリブレーション中に使用できる追加のgcodeコマンドには、`TEMPERATURE_PROBE_NEXT`があります。
* `TEMPERATURE_PROBE_NEXT`は、EDDyが目標温度`80℃`に達する前に新しいデータを強制的に取得するために使用されます。
* `TEMPERATURE_PROBE_COMPLETE`は、EDDyが設定温度`80℃`に達する前にキャリブレーションを完了するために使用されます。
* `ABORT`は、結果を無視してキャリブレーションを終了するために使用されます。
* キャリブレーションが完了したら、温度オフセット設定を保存するために`SAVE_CONFIG`を使用してください！
* 上記のように、他の多くのプログラムと比較して、このキャリブレーションプロセスはより挑戦的で時間もかかります。広い温度範囲で優れた最初の層のプリント結果を得るために、練習と何度も試行が必要になるかもしれません！

## zオフセットの微調整方法
* 第一ステップ：pursa/OrcaSlicer/cura/Bambu Studioを使用して、高さ0.2mmの200x200mmの四角形を切って、zオフセットの微調整を行います。
* 第二ステップ：プリンターのIPアドレスを使って`fluidd`または`mainsail`に入り、ダッシュボードの`移動コントロール`でz軸オフセットを微調整し、z_OFFsetを微調整します。微調整した数値を覚えておいてください。
<ImageView image={require('@site/docs/DebugDoc/faq/img/Zoff.webp').default} size="80%" align="left" />
* 第三ステップ：以下のコマンドを使用してKlippperを再起動します。
```bash
FIRMWARE_RESTART
```
* 第四ステップ：以下のコマンドを使用してZオフセットを調整します。ポップアップされるUIインターフェースで`zオフセット`を調整する数値は、印刷プロセス中に調整された数値です。[Paper test](https://www.klipper3d.org/Bed_Level.html#the-paper-test)
```bash
G28
PROBE_CALIBRATE METHOD=AUTO  
```
<ImageView image={require('@site/docs/DebugDoc/faq/img/UI.webp').default} size="50%" align="left" />
* キャリブレーションが完了したら`SAVE_CONFIG`をクリックしてパラメータを保存します。

:::info
* 上記の操作が完了しましたが、掃床時に失敗した場合は、zリミットを仮想リミットに変更し、元のendstop_pin`endstop_pin: probe:z_virtual_endstop`を上書きしてください。
:::
