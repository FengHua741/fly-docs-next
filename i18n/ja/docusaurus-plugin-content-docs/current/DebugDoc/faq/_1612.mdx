## Zオフセットの微調整方法
* 第一步：Pursa/OrcaSlicer/cura/Bambu Studioを使用して、200x200の高さ0.2の四角いピースを切り出して、Zオフセットを微調整するためのプリントを行います。
* 第二步：プリンタのIPを入力して`fluidd` `mainsail`に入り、ダッシュボードの`移動制御`でZ軸オフセットをさらに微調整し、調整した数値を覚えておきます。
<ImageView image={require('@site/docs/DebugDoc/faq/img/Zoff.webp').default} size="80%" align="left" />
* 第三步：以下のコマンドでKlipperを再起動します。
```bash
FIRMWARE_RESTART
```
* 第四步：以下のコマンドでZオフセットの調整を行うUI画面を表示し、プリント中に調整した数値を入力します。
```bash
G28
PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
```
<ImageView image={require('@site/docs/DebugDoc/faq/img/UI.webp').default} size="50%" align="left" />

:::info
* エラー`Eddy current sensor error`が発生した場合は、eddy設定内のz_offsetの数値を大きくしてください。図を参照してください。

<ImageView image={require('@site/docs/DebugDoc/faq/img/EDDYZOFF.webp').default} size="50%" align="left" />
:::

:::info
 * 上記の操作を完了した後もスキャンに失敗する場合は、Zリミットを仮想リミットに変更してください。
:::


## 操作説明

    :::warning 注意事項

    * 電流校正を先に行い、その後にコイル周波数校正を行ってください。
    * 複数のZ軸を持つマシンは、手動で一度調整し、全ての校正を完了させた後に再度調整し、再び校正を行ってください。
    * 下位機器のファームウェアはKliiperバージョンと完全に一致する必要があります。

    :::

    :::info

    * 電流校正を先に行い、その後にコイル周波数校正を行ってください。
    * 複数のZ軸を持つマシンは、手動で一度調整し、全ての校正を完了させた後に再度調整し、再び校正を行ってください。
    * 下位機器のファームウェアはKliiperバージョンと完全に一致する必要があります。

    :::


    * `FUIDD`ウェブページで下記の設定を追加した後、以下の操作を行ってください
        ```cfg
        [force_move]
        enable_force_move: true
        ```

    ### 第一步：EDDY電流の校正

        * 1: コンソールに下記のコマンドを入力します
            ```bash
            SET_KINEMATIC_POSITION z=80
            ```
        * 2: XとYを原点に戻し、ツールヘッドをベッドの中央に移動させます
            ```bash
            G28 X Y
            ```
        * 上記の手順を完了した後、EDDY電流の校正が必要です。まずEDDYをベッドの中央に移動させ、ノズルがベッドにわずかに接触するように調整します。次に、コンソールで下記のコマンドを実行して自動的に電流を取得し、最後に`SAVE_CONFIG`でパラメータを保存します。フィードバック値が約15であれば正常です。フィードバックが0または30の場合は、接続を確認してください（接触不良）。
            ```bash
            LDC_CALIBRATE_DRIVE_CURRENT CHIP=fly_eddy_probe
            ```
        * 正常な戻り値は`15`前後です。
        * 校正が完了したら`SAVE_CONFIG`でパラメータを保存します。

    ### 第二步：EDDY周波数の校正


        :::info

        * EDDYのインストール時、底部はノズルの上に少なくとも1-2mmの位置にする必要があります。
        * EDDY周波数校正時、コイルPCBはベッドから少なくとも5mm離れるようにしてください。

        :::

        * その後、ノズルをプラットフォームの中央に移動させます（注意：このステップではマシンがheightmapをロードしていないことを確認してください）。
        * 手動でZオフセット校正を開始します [紙テスト](https://www.klipper3d.org/Bed_Level.html#the-paper-test)
            ```bash
            PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
            ```
        * 校正が完了したら`SAVE_CONFIG`でパラメータを保存します。
        :::info
        * `z_tilt`または`quad_gantry_level（QGL）`機能を持つプリンタは、まず平行度を調整してください。i3はX軸の平行度を、三叉戟はベッドの平行度を、Voron 2.4はガントリーの平行度を調整し、再度EDDY周波数を校正してください。
        :::
    ### 第三步：温度補正

        * すべての軸を原点に戻します。
       ```bash
       G28
       ```

        * `quad_gantry_level`または`z_tilt`がある場合、まず`Z_TILT_ADJUST`または`QUAD_GANTRY_LEVEL`を一度実行し、次に再度原点に戻します。
        * コンソールにG0 Z5と入力するか、ウェブページなどでZ軸をベッド上方5mmに移動させます。
        * 以下のコードを実行して、機器の`idle timeout`を長く設定し、温度上昇中にタイムアウトしないようにします。
            ```bash
            SET_IDLE_TIMEOUT TIMEOUT=65000
            ```
        * 以下の指示で校正を開始します。
            ```bash
            TEMPERATURE_PROBE_CALIBRATE PROBE=fly_eddy_probe TARGET=56 STEP=4
            ```
        :::warning 注意事項

        * ヒント：上記のコマンドで、EDDYの目標温度は`56℃`に設定されています。これは多くの機器に適しています。
        * あなたの3Dプリンタの実際のキャビン温度が高い場合は、目標温度を適切に上げることができます。
        * 目標温度が高いほど、校正にかかる時間が長くなることに注意してください。EDDYが温度平衡に達するまでに時間がかかるためです。

        :::
    
        1. この操作を実行すると、UIにZ軸調整ボックスが表示されます。上述の手動[Zオフセット校正（Paper Test）方法](https://www.klipper3d.org/Bed_Level.html#the-paper-test)を使用して、紙をノズルとベッドの間に挟み、値を確認してください。
        2. 値を確認した後、ベッド温度を最高温度に、ノズル温度を220℃に設定します。
        3. エアコンや窓を開けている部屋で作業する場合、EDDYの温度上昇を確実にするため、エアコンや窓を閉めることをお勧めします。風は温度上昇に影響を与えます。
        4. EDDYの温度が上昇するにつれて、システムは4℃ごとに手動Zオフセット校正を実行するように指示します。ベッド温度が高いので、火傷に注意してください！
        5. 手動Zオフセット校正（Paper Test）を繰り返し、校正が完了するまで続けます。EDDYの温度が上昇しなくなった場合は、以下の関連コマンドを使用して校正を早期に終了することができます。
        
        * ドリフト校正中に使用可能な追加のGコードコマンドには、TEMPERATURE_PROBE_NEXTが含まれます。
            * `TEMPERATURE_PROBE_NEXT`    EDDYが設定された目標温度`56℃`に達する前に、EDDYが新しいデータをサンプリングするように強制します。
            * `TEMPERATURE_PROBE_COMPLETE`    EDDYが設定温度`56℃`に達しない場合に校正を完了します。
            * `ABORT`  校正を中止し、結果を無視するために使用します。
        * これらの手順を完了すると、EDDYは幅広い温度範囲で優れた最初の層プリントを提供できるようになります！
## 各種最適化マクロ

    ### `[quad_gantry_level]`スキャン最適化
        * `quad_gantry_level`が設定されている場合、以下のマクロを設定に追加できます。設定されていない場合は追加する必要はありません。
        * このマクロはガントリーの高低差に応じて探査高さを下げます。
            ```cfg
            [gcode_macro QUAD_GANTRY_LEVEL]
            rename_existing: _QUAD_GANTRY_LEVEL
            gcode:
                SAVE_GCODE_STATE NAME=STATE_QGL
                BED_MESH_CLEAR
                {% if not printer.quad_gantry_level.applied %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_QGL
            ```

    ### `[z_tilt]`スキャン最適化

        * `z_tilt`が設定されている場合、以下のマクロを設定に追加できます。設定されていない場合は追加する必要はありません。
        * このマクロはガントリーの高低差に応じて探査高さを下げます。
            ```cfg
            [gcode_macro Z_TILT_ADJUST]
            rename_existing: _Z_TILT_ADJUST
            gcode:
                SAVE_GCODE_STATE NAME=STATE_Z_TILT
                BED_MESH_CLEAR
                {% if not printer.z_tilt.applied %}
                _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _Z_TILT_ADJUST horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_Z_TILT
            ```

    ### スキャン最適化

        * 大出力ACベッド（500W以上）を使用する場合
        * 以下のマクロはスキャン中にベッドをオフにして、スキャンが完了した後に再び加熱します。
            ```cfg
            [gcode_macro BED_MESH_CALIBRATE]
            rename_existing: _BED_MESH_CALIBRATE
            gcode:
                {% set TARGET_TEMP = printer.heater_bed.target %}
                M140 S0
                _BED_MESH_CALIBRATE {rawparams}
                M140 S{TARGET_TEMP}
            ```
    ## ネットベッドコマンド
    
        * `printer.cfg`設定ファイルに以下のマクロを追加します。
            ```bash
            [gcode_macro BED_MESH_CALIBRATE]
             rename_existing: _BED_MESH_CALIBRATE
             gcode: 
                   _BED_MESH_CALIBRATE METHOD=rapid_scan {rawparams}
            ```
        * ローカルネットベッドコマンドは、Klipperがオブジェクト除外機能を使用できることを確認してください。開始マクロに追加することをお勧めします。
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan adaptive=1 
            ```
