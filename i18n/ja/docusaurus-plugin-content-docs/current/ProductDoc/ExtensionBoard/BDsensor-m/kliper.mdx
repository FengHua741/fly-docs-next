# **BDsensor-m**のインストール

## センサーケーブルをメインボードのEXP1ポートに接続
    * センサーのケーブル長が足りない場合は、パッケージ内の延長ケーブルを使用してください
    * BDsensor-mのCKLとSDA線は、基板上の任意のGPIOピンに接続可能です。また、BDセンサーケーブルを直接Bltouchポートに接続することもできます。例えば：

    ```bash
    BLtouch    |    BDsensor-m
    5V       -->     5V
    GND      -->     GND
    S        -->     CLK/SCL    (Input)
    GND      -->     GND
    Zmin     -->     SDA    (Input/Output) 
    ```

    * 一部のピンがMCUのgpiosに直接接続されていない場合（例えば、フィルターコンデンサが付いているか、MOSFET、ダイオードまたは光カプラで隔離されているが、抵抗や抵抗によるプルアップ/ダウンで隔離されている場合は可能）、`BDsensor-m`と一緒に使用することはできません。そして、ファームウェアは接続エラーを報告します。例えば
    * ファンやヒーターのコネクタはMOSFETで隔離されています、
    * 一部の基板では、温度サーミスタやエンドストップ/プローブのコネクタが通常GNDにフィルターコンデンサで接続されています、

## Klipperファームウェアにパッチをインストール

    * 以前に変更したklipperファイルを破棄し、klipperを更新します

        ```bash
        cd
        cd ~/klipper
        git checkout .
        git pull
        ```

    * BDセンサーの最新コードをクローン

        ```bash
        cd && git clone https://github.com/markniu/Bed_Distance_sensor.git
        ```

    * インストール

        ```bash
        cd  ~/Bed_Distance_sensor/klipper/
        ./install_BDsensor.sh
        ```

    * ファームウェアをコンパイル

        ```bash
        cd ~/klipper/
        make menuconfig
        make clean
        make
        ```

    * BDセンサーが接続されているメインボードにファームウェアを書き込みます

## Moonrakerを実行している場合、以下の部分をmoonraker.confに追加してください。そうすることで、ウェブページまたはklipperscreenからクリック一つでBDsensorを更新できます。

    ```bash
    [update_manager BDsensor]
    type: git_repo
    primary_branch: new
    channel: dev
    path: ~/Bed_Distance_sensor
    origin: https://github.com/markniu/Bed_Distance_sensor.git
    install_script: ./klipper/install_BDsensor.sh
    is_system_service: False
    managed_services: klipper
    info_tags:
    desc=Bed Distance Sensor
    ```

## printer.cfgの編集

    * この部分を**printer.cfg**にコピーし、`[BDsensor]` `sda_pin`  `scl_pin`を編集し、他のプローブ部分（例えば **BLtouch**）を無効に忘れないでください。BDセンサーは、メインボードまたはツールヘッドのCANモジュールに接続できます。
    * `speed`を0.8に変更します。これはz軸の傾斜とPROBE_ACCURACYコマンドにのみ適用されます。値が小さいほど、MCUがBDセンサーを読み取る際の精度が高くなります。 `[BDsensor]`
    * BDセンサーをZ軸のホーム位置として使用する場合、`[stepper_z]`の`endstop_pin`を`endstop_pin: probe:z_virtual_endstop`に変更してください
    * **printer.cfg**に`[safe_z_home]`があることを確認してください
    * `bed_mesh`と`z_tilt`または`quad_gantry_level`の値を1（0.7-1.0mmが推奨）に変更してください、klipperのデフォルト値は5mmで、センサーの範囲を超えやすいです
    * **ノズルの高さは`z_adjust:`で設定するのが適しています。正の値はベッドに近づけ、負の値はベッドから遠ざけます。その他のノズルの高さを調整する設定にはバグがあります**
    * 快速ベッドスキャンを有効にするには、`no_stop_probe:true`の前に#を削除してください
    * 以下は設定例です。

        ```cfg
        [BDsensor] 
        scl_pin:PC6  # サーボ信号ポート
        sda_pin:PC3  # リミット信号ポート
        delay: 20 # 20us per pulse, this value should be >=20 but must be below 50
        z_offset:0 # this `z_offset` must be set to 0. 
        z_adjust:0.0 # z axis adjustment, replace the z_offset function. within -0.3 to 0.3mm
        x_offset: -34
        y_offset: 0
        #no_stop_probe:true # enable this for fast probe, the toolhead will not stop at the probe point.
        position_endstop: 0.8 # the Z axis will stop at this position (mm) while homing z, recommend value is 0.4~1.0
        #speed:0.8 # this speed only works for the z tilt and PROBE_ACCURACY command.

        [stepper_z]
        endstop_pin: probe:z_virtual_endstop 
        #position_endstop: 0.5
        homing_speed: 5
        second_homing_speed: 0.8

        [bed_mesh]
        speed: 200
        horizontal_move_z:1
        algorithm: bicubic

        [quad_gantry_level]
        horizontal_move_z:1

        ```

## インストール後、以下のGコードコマンドを送信して確認してください

    ```bash
    M102   S-1     # センサー情報の読み取り
    M102   S-2     # 一つの距離値の読み取り
    ```

## 接続の確認

    * **コンソール**から`M102 S-1`を送信すると、以下のようなメッセージが返ってきます。空白または他の文字列が返ってきた場合は、接続と配線順序を確認してください。

        ```bash
        Send: M102 S-1
        Recv: V1.0 pandapi3d.com
        ```

## キャリブレーション

    * ノズルを清掃し、Z軸を移動してノズルがベッドに触れる位置まで調整します（BDsensor-mはこの位置をゼロ位置として使用するため、`z_offset`は必要ありません。これが[BDsensor-m]部分で値が0の理由です）
    * **コンソール**からGコードコマンド`M102 S-6`を送信すると、プリンターはZ軸を0.1mmずつゆっくりと上昇させ、4mmに達するまで繰り返します。センサーインストール前にM102 S-6を実行しないでください。また、キャリブレーション中はプリンターの電源を切らないでください。そうしないと古いキャリブレーションデータが削除されます。この場合、再度キャリブレーションしてください。
    * その後、`M102 S-5`を送信してBDセンサーが正しくキャリブレーションされたかどうかを確認できます。これはBDセンサー内に保存されている元のキャリブレーションデータを返します。

**注意事項**:

* Z軸のホーム速度は5が最適です

* M102 S-5が返す最初のキャリブレーション元データが400以上の場合、センサーがベッドに近すぎることを意味し、より近くに再インストールする必要があります。最初のデータの推奨値は100です。また、二番目のデータの値が最初のデータより10以上大きいことを確認してください。

  * FAQ: キャリブレーションが1で始まり、二番目の値が9、第三番目の値が24の場合は何を意味しますか？

  * これは、0-0.1ミリメートルの解像度が9で、0.1-0.2ミリメートルの解像度が15であることを意味します。したがって、最初の解像度0-0.1ミリメートルを10以上にするためにもう一度キャリブレーションをお勧めします。

* G28を実行した後やZ_tilt、quad_gantry_levelコマンドでZ軸の高さを調整することを忘れないでください

* 正確に大文字と小文字の部分名を指定しないと、klipperは`Unknown pin chip name 'probe'`と報告します
