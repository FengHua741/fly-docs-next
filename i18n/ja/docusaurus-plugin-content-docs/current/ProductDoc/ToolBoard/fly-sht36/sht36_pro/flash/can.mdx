# CANファームウェアのコンパイル

## コンパイルの開始

    <SSH />

    <MakeOperate/>

    <CAN />

    * マイクロコントローラー起動時に設定するGPIOピンを選択します。入力：!gpio13 请注意!は大文字です
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/can.webp').default} size="100%" align="center" />
    * `Q` キーを押して、**Save configuration** が表示されたら、その後 `Y` キーを押します
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * 現在は設定が保存され、コマンドラインインターフェイスに戻っているはずです
    * コンパイルを開始するために `make -j4` を入力します。時間がかかります
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * 最後に以下の内容が出力されたらコンパイル成功です

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## ファームウェアのフラッシュ

    * ファームウェアをフラッシュする前に、XT30データ線を使用してツールボードをUTOCまたはブリッジファームウェアがインストールされたメインボードに接続します。参照配線図は以下にあります
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/sht36pro_utoc.webp').default} size="80%" align="left" />
    * CAN HとCAN Lの抵抗値が電源オフ状態で約60Ωであることを確認してください。120Ωであれば120Ωのジャンパーを接続し、40Ωであればジャンパーを外す必要があります
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/120.webp').default} size="80%" align="center" />
