# CANファームウェアのコンパイル

## コンパイルの開始

    <SSH />

    <MakeOperate/>

    <CAN />

    * micro-controllerの起動時に設定するGPIO pinsを選択します。入力：!gpio13 （!は大文字です）
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/can.webp').default} size="100%" align="center" />
    * `Q` キーを押すと **Save configuration** が表示され、その後 `Y` キーを押します
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * 現在、設定が保存されコマンドラインに戻っているはずです
    * コンパイルを開始するために `make -j4` を入力します（時間がかかります）
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * 最後に以下のような出力が出ればコンパイル成功です

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```
    <LS />


## ファームウェアの焼付け

    * ファームウェアを焼付ける前に、XT30データラインを使用してツールボードをUTOCまたはブリッジファームウェアがインストールされたメインボードに接続してください。参考配線図は以下の通りです
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/sht36pro_utoc.webp').default} size="80%" align="left" />
    * CAN HとCAN Lが電源オフ状態で60Ω程度になるようにマルチメーターで抵抗値を測定してください。120Ωの場合には120Ωジャンパーを接続し、40Ω程度の場合にはジャンパーを外してください
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/120.webp').default} size="80%" align="center" />

    <CANFLASH />
