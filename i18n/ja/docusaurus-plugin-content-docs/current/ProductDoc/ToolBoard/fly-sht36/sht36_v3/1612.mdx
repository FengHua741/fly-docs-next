# Eddyの使用

    :::info
    * Eddyをインストールする際、ノズルの下に少なくとも1-2mmのスペースが必要です
    * Eddyの周波数を校正する際、コイルPCBはヒートベッドから最低5mm離す必要があります
    * ヒートベッドは常温でテストし、テスト中はヒートベッドを開けないでください

    :::

## 参照設定と使用説明

##### `printer.cfg`設定ファイルに以下の設定を追加し、eddyをデバッグしてください

    ### ベッドスキャン設定

        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 43
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

        * 現在、外部1612モジュールがリリースされていないため、この設定は一時的に使用できません
        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 42
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

    ### 温度補正設定

        ```cfg
        [temperature_probe fly_eddy_probe]
        sensor_type:Generic 3950
        sensor_pin:SHT36:gpio28
        ```

    ## ネットベッドコマンド

        * `printer.cfg`設定ファイルに下記のマクロを追加してください
            ```bash
            [gcode_macro BED_MESH_CALIBRATE]
             rename_existing: _BED_MESH_CALIBRATE
             gcode: 
                   _BED_MESH_CALIBRATE METHOD=rapid_scan {rawparams}
            ```
        * 局域ネットベッドコマンドは、Klipperがオブジェクト排除機能を使用できることを確認する必要があります。開始マクロに追加することをお勧めします。
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan adaptive=1 
            ```

### bed_mesh設定

    * [bed_mesh]の`horizontal_move_z`パラメータは、2に設定することをお勧めします。これにより、プラットフォームスキャン時にEddyが可能な限りプラットフォームに近づきます。

    ```cfg
    [bed_mesh]
    horizontal_move_z: 2
    ```

## 衝突防止マクロ

    ### `[quad_gantry_level]`スキャン最適化
        * `[quad_gantry_level]`が設定されている場合、以下のマクロを設定に追加してください。設定されていない場合は追加する必要はありません。
        * このマクロは、ゲートの高さの差に応じて探査高さを下げます
            ```cfg
            [gcode_macro QUAD_GANTRY_LEVEL]
            rename_existing: _QUAD_GANTRY_LEVEL
            gcode:
                SAVE_GCODE_STATE NAME=STATE_QGL
                BED_MESH_CLEAR
                {% if not printer.quad_gantry_level.applied %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_QGL
            ```

    ### `[z_tilt]`スキャン最適化

        * `[z_tilt]`が設定されている場合、以下のマクロを設定に追加してください。設定されていない場合は追加する必要はありません。
        * このマクロは、ゲートの高さの差に応じて探査高さを下げます
            ```cfg
            [gcode_macro Z_TILT_ADJUST]
            rename_existing: _Z_TILT_ADJUST
            gcode:
                SAVE_GCODE_STATE NAME=STATE_Z_TILT
                BED_MESH_CLEAR
                {% if not printer.z_tilt.applied %}
                _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _Z_TILT_ADJUST horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_Z_TILT
            ```

    ### eddyが高い確率でヒートベッドに干渉するスキャン最適化マクロ

        * 大出力ACヒートベッド（500W以上）を使用する場合
        * 以下のマクロは、ベッドスキャン時にヒートベッドをオフにし、スキャンが完了した後で再度加熱します
            ```cfg
            [gcode_macro BED_MESH_CALIBRATE]
            rename_existing: _BED_MESH_CALIBRATE
            gcode:
                {% set TARGET_TEMP = printer.heater_bed.target %}
                M140 S0
                _BED_MESH_CALIBRATE {rawparams}
                M140 S{TARGET_TEMP}
            ```

   # デバッグ手順：
   ## 第一歩：周波数変更

    ### ldc1612のデフォルト周波数変更
    この方法はsht36-v3/SB2040-v3またはSB2040-Pro-v3にのみ使用されます
    Mobaxtermツールを使用してコマンドを実行できます

    <Button variant="contained" disableElevation href="/docs/ResDownload/auxiliary_software/">Mobaxtermツールのダウンロード</Button>が必要です

    [詳細使用説明はこちら](/docs/ProductDoc/SBC/fly-pi-v2/ssh)

    最初のコマンドを実行し、Enterキーを押します

    ```cfg
sed -i 's/LDC1612_FREQ = 12000000/LDC1612_FREQ = 40000000/g' ~/klipper/klippy/extras/ldc1612.py
    ```

    次のコマンドを実行し、Enterキーを押します
  ```cfg
grep "LDC1612_FREQ = 40000000" ~/klipper/klippy/extras/ldc1612.py
  ```
    置換が成功すると、`LDC1612_FREQ = 40000000`を含む行が返されます。返事がない場合は置換が失敗しています。
    または、このコマンドで置換が成功したかどうか確認できます
    <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/1612.webp').default} size="110%" align="left" />
```cfg
sed -n '10,15p' ~/klipper/klippy/extras/ldc1612.py
```
   <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/ldc161240mhz.webp').default} size="30%" align="left" />

   温かいヒント：変更後、以下のコマンドでklipperサービスを再起動し、eddyの周波数を再度校正してください
   ```cfg
   systemctl restart klipper
   ```
   eddyの周波数校正コマンド
   ```cfg
   PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
   ```
