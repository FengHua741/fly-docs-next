# ファームウェア書き込み説明
    
    * ファームウェアのコンパイルとファームウェアの書き込みにはPCソフトウェアを使用する必要があります
    * SHT36 V3はRS232およびCANの2種類の接続方法でPCソフトウェアと接続できます
    * RS232およびCAN接続には`XT30(2+2)-F`を使用します
    * 以下の図は`XT30(2+2)-F`ケーブルのスタイルです
        <ImageView image={require('@site/docs/General/get-id/img/xt30.webp').default} size="80%" align="left" />

:::danger 注意点
* メインボードのファームウェア書き込みにはKatapultからKlipperを焼く必要があります
* メインボードに既にKatapultファームウェアが焼かれていたら、最後のKatapultファームウェアの書き込みは不要です。これはメインボードのKatapultファームウェアが正常に焼けなかった場合の予防措置です
* メインボードをリセットボタンを2回連続で押して、メインボード上のLEDが点滅しているか確認してください。もし点滅している場合は、メインボードがKatapultファームウェアに入っていることを意味します

:::

## どのファームウェアが必要かを判断する方法

    * SHT36 V3は以下の方法でPCソフトウェアと接続できます
    * <Button variant="contained" disableElevation component="a" rel="noopener" href="./can">CANファームウェア</Button>
        * 好み: `Katapult`ファームウェアを焼いた後は、マザーボードに触れることなくファームウェアを更新でき、バス接続もサポートされます
        * 不利: `KlipperScreen`と一緒に使用すると、ホーム位置タイムアウトなどのエラーが発生する可能性があります
        * ヒント: CANを使用する場合、PCソフトウェアがCANに対応しているか確認し、UTOCまたはブリッジファームウェアが焼かれたマザーボードとの組み合わせが必要です
    * <Button variant="contained" disableElevation component="a" rel="noopener" href="./rs232">RS232ファームウェア</Button>
        * 好み: 特殊なソフトウェアなしで使用でき、通信が信頼性があり安定しています
        * 不利: ファームウェアを再び焼くには、手動DFUモードに移行するか、RSTをダブルクリックして`Katapult`に入らないといけません
        * ヒント: G2TまたはUTORと組み合わせて使用する必要があります
