# 固件焼込み説明

    * MMUはRS232、CAN、USBの3つの接続方法で上位機器と接続できます
    * RS232とCANの場合、`XT30(2+2)-F`を使用して接続する必要があります。USBの場合、データ転送機能付きの`TYPE-C`ケーブルを使用する必要があります
    * 以下の`XT30(2+2)-F`ケーブルのスタイル
        <ImageView image={require('@site/docs/General/get-id/img/xt30.webp').default} size="80%" align="left" />

:::danger 注意点
* メインボードのファームウェア焼込みにはKatapultからKlipperを焼込みます
* メインボードにKatapultがすでに焼込まれていることが確認できれば、最後のKatapultファームウェアの焼込みは必要ありません。これはメインボードのKatapultファームウェア焼込みが失敗した場合の予防措置です
* メインボード上のLEDが2回連続で点滅したら、メインボードがKatapultファームウェアに入っていることを示します
:::

## どの種類のファームウェアが必要かをどのように判断するか

    * MMUは以下のような方法で上位機器と接続できます
    * <Button variant="contained" disableElevation component="a" rel="noopener" href="./can">CANファームウェア焼込み</Button>
        * 利点：`Katapult`ファームウェアを焼込み済みであれば、メインボードに触れずにファームウェアを更新でき、バス接続もサポートしています
        * 欠点：`KlipperScreen`と併用すると、ホームポジションタイムアウトなどのエラーが発生する可能性があります
        * ヒント：CANを使用する場合は、上位機器がCANをサポートしていることを確認し、UTOCまたはブリッジファームウェアを焼込んだメインボードと組み合わせる必要があります
    * <Button variant="contained" disableElevation component="a" rel="noopener" href="./rs232">RS232ファームウェア焼込み</Button>
        * 利点：特別なソフトウェアなしで使用でき、通信が信頼性があり安定しています
        * 欠点：ファームウェアを再焼込みするには、手動DFUモードに移行するか、RSTボタンを2回押して`Katapult`に入らなければなりません
        * ヒント: G2TまたはUTORと組み合わせて使用する必要があります
    * <Button variant="contained" disableElevation component="a" rel="noopener" href="./usb">USBファームウェア焼込み</Button>
        * 利点：`Katapult`ファームウェアを焼込み済みであれば、メインボードに触れずにファームウェアを更新でき、メモリカードやDFU焼込みを使用する場合は従来の方法でファームウェアを更新する必要があります。通信は信頼性があり安定しています
        * 欠点：複数のUSBデバイスがある場合、データ転送に影響を与える可能性があります
