# ファームウェア焼成説明

    * SB2040 V3はRS232とCANの2つの接続方法を使用してホストPCと接続できます
    * RS232とCANは`XT30(2+2)-F`を使用して接続する必要があります
    * 以下は`XT30(2+2)-F`データ線のスタイルです
        <ImageView image={require('@site/docs/General/get-id/img/xt30.webp').default} size="80%" align="left" />

:::danger 注意点
* メインボードのファームウェア焼成にはKatapultを介してKlipperを焼成する必要があります
* メインボードにKatapultファームウェアが既に焼成されている場合は、最後のKatapultファームウェア焼成を行う必要はありません。これは、メインボードのKatapultファームウェアが正常に焼成されなかった場合の予防策です
* メインボードのリセットボタンを2回素早く押すと、メインボード上のLEDがフラッシュするかどうかを確認できます。これがフラッシュすれば、メインボードはKatapultファームウェアに入っています
:::

## どの種類のファームウェアが必要かを見分ける方法

    * SB2040 V3は以下の方法でホストPCと接続できます
    * <Button variant="contained" disableElevation href="./can">CANファームウェア焼成</Button>
        * 利点：`Katapult`ファームウェアを焼成した場合、マザーボードを触れずにファームウェアを更新でき、バス接続もサポートします
        * 欠点：`KlipperScreen`と一緒に使用すると、ホーム位置決定のタイムアウトエラーなどが発生する可能性があります
        * ヒント：CANを使用する場合は、ホストPCがCANをサポートしていることを確認し、UTOCまたはブリッジファームウェアを焼成したマザーボードが必要です
    * <Button variant="contained" disableElevation href="./rs232">RS232ファームウェア焼成</Button>
        * 利点：特別なソフトウェアなしで使用でき、通信は信頼性があり安定しています
        * 欠点：ファームウェアを再焼成するには、手動DFUモードに移行するか、RSTボタンをダブルクリックして`Katapult`に入れなければなりません
        * ヒント: G2TまたはUTORを使用する必要があります
