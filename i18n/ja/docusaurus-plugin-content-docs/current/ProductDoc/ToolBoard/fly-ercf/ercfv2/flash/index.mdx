# 固件焼込み説明

    * ERCF V2はRS232とCANの2つの接続方法でホストコンピュータと接続することができます
    * RS232とCANは`XT30(2+2)-F`を使用して接続する必要があります
    * 下記は`XT30(2+2)-F`データ線のスタイルです
        <ImageView image={require('@site/docs/General/get-id/img/xt30.webp').default} size="80%" align="left" />

:::danger 注意
* メインボードのファームウェア焼込みはKatapultを通じてKlipperを焼込みます
* メインボードがすでにKatapultが焼込まれていることが確認された場合、最後のKatapultファームウェアの焼込みは不要です。それはメインボードのKatapultファームウェアが焼込まれるのを防ぐための予防措置です
* リセットボタンを素早く2回押して、メインボード上のLEDが点滅しているかどうかを確認できます。点滅している場合、メインボードはKatapultファームウェアに入っています
:::

## どの種類のファームウェアが必要かを見つける方法

    * ERCF V2は以下の方法でホストコンピュータと接続することができます
    * <Button variant="contained" disableElevation href="./can">CANファームウェア焼込み</Button>
        * 利点：`Katapult`ファームウェアを焼込んだ場合、メインボードに触れずにファームウェアを更新することができます。そしてバス接続をサポートします
        * 欠点：`KlipperScreen`と一緒に使用すると、ホームポジションタイムアウトなどのエラーが発生する可能性があります
        * ヒント：CANを使用する場合は、ホストコンピュータがCANをサポートしていることを確認し、UTOCまたはブリッジファームウェアが焼込まれたメインボードが必要です
    * <Button variant="contained" disableElevation href="./rs232">RS232ファームウェア焼込み</Button>
        * 利点：特別なソフトウェアなしで使用でき、通信は信頼性があり安定しています
        * 欠点：ファームウェアを再焼込みするには、手動DFUに移動するか、RSTボタンをダブルクリックして`Katapult`に入ることができます
        * ヒント: G2TまたはUTORを使用する必要があります
    * <Button variant="contained" disableElevation href="./usb">USBファームウェア焼込み</Button>
        * 利点：`Katapult`ファームウェアを焼込んだ場合、メインボードに触れずにファームウェアを更新することができます。メモリカードやDFU焼込みを使用する場合は、従来の方法でファームウェアを更新する必要がありますが、通信は信頼性があり安定しています
        * 欠点：複数のUSBデバイスがある場合、データ転送に影響を与える可能性があります
