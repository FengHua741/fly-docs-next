# USB ファームウェアのコンパイル


## コンパイル開始

    <SSH />

    <MakeOperate/>

    <USB />

    * `Q` キーを押して **Save configuration** が表示されたら、その後に `Y` キーを押す
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * 現在は設定が保存され、コマンドラインに戻ったはずです
    * `make -j4` を入力してコンパイルを開始します。時間がかかります
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * 最後に以下の内容が出力されたらコンパイル成功です

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```
## 焼付けモードへの確認

    * 上位機が正常にネットワーク接続できていることを確認し、下位機と接続されているデータ線がデータ転送機能を持っていることを確認してください。
    * 上位機にSSH接続後、`lsusb` と入力してエンターを押します。下図で赤枠で囲まれた情報が表示されたら次に進んでください。
        * 何も情報が返ってこない場合、これは上位機のシステムの問題です。こちらでは対応できません。正常なシステムに切り替えるか、上位機を交換してください。
        * `lsusb` コマンドがないという場合は、次のコマンドを実行してインストールできます
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" /> 
    * 赤枠内のIDは参考として使用できますが、**固定ではなく可変**です。`OpenMoko, Inc. stm32f407xx` の後ろの `XX` は可変です。
        * 部分的な上位機はシステムの問題により、完全に表示されないか表示されない場合があります。
