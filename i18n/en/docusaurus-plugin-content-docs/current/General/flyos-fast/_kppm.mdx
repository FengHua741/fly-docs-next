## Configuration

### Disable Power-off Shutdown
    :::danger Note
    - Power-off shutdown must be disabled, otherwise the power-off resume function cannot be used.
    - The power-off resume function has an automatic shutdown after saving progress feature.
    :::

- Enter the device's IP address in the browser's address bar, for example: `http://192.168.6.179`
- Access the configuration page
    - Open the device IP address in the browser, for example: `http://192.168.1.2/`
    - In Fluidd, uncheck as shown in the left image below -> "Filter hidden files and folders".
    - In Mainsail, check as shown in the right image below -> "Show hidden files".

### plr.cfg Configuration File
    :::danger Note
    * Please replace the following content in the configuration file: (PINS)  
    * with <code>{props.power_pine}</code>

    :::

    - In the printer configuration page, find the `plr.cfg` file.
    - Empty the contents inside, then paste the configuration below into it.
    - The content of the configuration file is as follows:

    ```cfg
    [mcu host]   
    serial: /tmp/klipper_host_mcu
    
    [power_loss_resume]
    power_pin: (PINS)
    is_shutdown: True # Whether to execute shutdown operation, enabled by default
    paused_recover_z: -2.0 # If paused during printing when power off, the Z-axis movement distance when resuming print, default is no movement
    start_gcode:
        # G-code to execute before resuming print
        # All parameters saved before power-off can be accessed via {PLR}
        # Use M118 {PLR} to output all available parameters
        M118 Starting resume: {PLR.print_stats.filename}
        M118 Interruption position: X:[{PLR.POS_X}] Y:[{PLR.POS_Y}] Z:[{PLR.POS_Z}] E:[{PLR.POS_E}]
        M140 S{PLR.bed.target}      ; Set bed temperature
        M104 S{PLR.extruder.target-10}  ; Set extruder temperature
        M109 S{PLR.extruder.target-10}  ; Wait for extruder to heat up to set temperature
        G91                             ; Relative coordinates
        G1 Z2 F100                      ; Lift Z, prepare for X,Y reset
        G90                             ; Absolute coordinates
        G28 X Y                         ; Reset XY
        M140 S{PLR.bed.target}          ; Set bed temperature
        M104 S{PLR.extruder.target}     ; Set extruder temperature
        M190 S{PLR.bed.target}          ; Wait for bed to heat up to set temperature
        M109 S{PLR.extruder.target}     ; Wait for extruder to heat up to set temperature
        M83                             ; Relative extrusion
        # G1 E0.5 F400                  ; Extrude a little bit
    layer_count: 2 # Resume printing after specified number of layers, execute layer_change_gcode
    layer_change_gcode:                
        # G-code to execute after resuming {layer_count} layers
        M118 Resume print speed
        M106 S{PLR.fan_speed}             ; Turn on cooling fan
        M220 S{PLR.move_speed_percent}    ; Set requested speed percentage
        M221 S{PLR.extrude_speed_percent} ; Set requested extrude speed percentage
    shutdown_gcode:
        # G-code to execute before shutdown
        M118 Low voltage, shutting down
        # M112 ; Emergency stop

    ```

    :::warning Notice

    - The `start_gcode` macro in the above configuration file may need to be modified based on the actual machine conditions.
    - Note that if `[homing_override]` is used, do not arbitrarily set homing positions within `[homing_override]`. Any failure due to this will not be responsible.

    :::

- After saving the above configuration file,
- Open the `printer.cfg` file and add the following content at the beginning of the file:

    ```cfg

    [include plr.cfg]

    ```

- Click the save button in the top-right corner and restart. 
- The Klipper power-off resume function has now been configured.

## Testing

- Print any file and click the `Emergency Stop` button during printing to simulate a power-off.
- Click `Firmware Restart` again, wait for Klipper to connect normally.
- If a pop-up prompt appears on the webpage or KlipperScreen, it indicates that the power-off resume function is working properly.
- You can continue testing real power-off situations later.
