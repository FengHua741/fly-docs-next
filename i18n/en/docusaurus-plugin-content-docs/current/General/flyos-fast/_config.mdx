## Important Notice
    :::tip Must Read

    - The settings in the `part` section of the configuration file will only take effect when the system is first started after burning.
    - Due to the complexity of the configuration process, it is strongly recommended to configure through a browser after connecting to the machine via network.
    - This tutorial only provides modifiable options.

    :::

    :::warning Attention

    - Try to use the `mainsail` or `fluidd` web pages for configuration modification.
    - Do not open web page translation, otherwise the system may fail to start after saving.
    - Chinese or special characters are not recommended in the configuration file.
    - After modifying the configuration file, the system needs to be restarted to take effect.

    :::

    :::danger

    - The content below `######### PLEASE DO NOT EDIT THE FOLLOWING #########` in the configuration file must not be modified.

    :::

## How to Modify the Configuration File

### Modifying the Configuration File After Burning

- The configuration items that only take effect on the `first startup` must be modified by using a professional text editor to open the configuration file `config.txt` in the removable disk `FlyOS-Conf` after burning.

### Online Configuration Modification While System is Running

- If it's a regular configuration item, modifications made in the system web configuration page will take effect after restarting the system.
    - Open the device IP address in the browser, for example: `192.168.1.2/`
    - For fluidd, uncheck as shown on the left in the figure below -> "Filter and hide files and folders".
    - For mainsail, check as shown on the right in the figure below -> "Show hidden files".

    - At this point, you can see the `.flyos-config` folder. Enter this folder and there will be a `sys-config.conf` file.
    - The `sys-config.conf` file is a symbolic link to the configuration file `config.txt` in the removable disk `FlyOS-Conf`.

    - Open the `sys-config.conf` file, modify the required configuration items, save and close it, then restart the system.

    - Note: The format of the configuration items in the `sys-config.conf` file is `key=value`, do not modify it to `key: value`.
    - After saving the modifications, restarting the system will make them effective.

## Configuration Items

### Wireless Network Configuration

- The following configuration items will take effect every time the system starts.

| Configuration Item | Default Value | Usable Values | Description |
| :---: | :---: | :--- | :--- |
| `WIFI_SSID` | `Empty` | Any WIFI name | WIFI name. If filled in, the system will automatically prioritize connecting to this WIFI on startup. |
| `WIFI_PASS` | `Empty` | Leave blank if no password | WIFI password. If filled in, the system will connect to WIFI using the password. |

### System Configuration

- The following configuration items will take effect every time the system starts.
- Below are the modifiable options.

| Configuration Item | Default Value | Usable Values | Description |
| :---: | :-------: | :--- | :--- |
| `board` |  <code>{props.board}</code> |   <code>{props.board}</code> | Mainboard model. Please do not modify configurations outside the usable values. |
| `localtime` | `Asia/Shanghai` | [World Time Zone Table](/docs/DebugDoc/flyos-fast/timezone) | Set the system's default time zone. The system will automatically adjust the time after connecting to the network. |
| `canbus_bitrate` | `1000000` | `1000000`<br/> `500000`<br/> `250000` | Set the default rate for all CAN devices in the FAST system. Modifications can only be made here, with the default being 1M. |

----

### Screen Display Configuration

- The following configuration items will take effect every time the system starts.

| Configuration Item | Default Value | Usable Values | Description |
| :---: | :-------: | :--- | :--- |
| `screen` | `hdmi` | <code>nano</code>,<code>{props.screen}</code> <code>{props.screen3}</code> <br/> <code>{props.screen1}</code><code>{props.screen2}</code>| Set the screen. `none` will turn off all displays. |
| `rotate` | `90` | `0`, `90`, `180`, `270` | Set the rotation angle for FLY-TFT. This does not apply to HDMI. |
| `klipper_screen` | `0` | `0`, `1` | Set whether to enable KlipperScreen.<br/>1: Enable<br/>0: Disable<br/>Effective when `printer=klipper`. |
| `moon2uart` | `0` | `0`, `1` | Set whether to enable moon2uart.<br/>1: Enable<br/>0: Disable<br/>Effective when `printer=klipper`. |

### Klipper Related Configuration

- The following configuration items will take effect every time the system starts.

| Configuration Item | Default Value | Usable Values | Description |
| :---: | :-------: | :--- | :--- |
| `klipper_webui` | `fluidd` | `fluidd`, `mainsail` | Set the default Klipper front-end UI.<br/>Does not affect the online one-click switching function.<br/>Effective when `printer=klipper`. |
| `canbus_bitrate` | `1000000` | `1000000`, `500000`, `250000` | Set the default rate for all CAN devices in the FAST system. Modifications can only be made here, with the default being 1M. |

### Printer Type Configuration

### Power-off Shutdown

- The following configuration items will take effect every time the system starts.
- To use power-off resume printing, the power-off shutdown function must be disabled.

| Configuration Item | Default Value | Usable Values | Description |
| :---: | :-------: | :--- | :--- |
| `shutdown_pin_state` |<code>{props.state}</code>| `0`, `1` | Set the voltage level state of the power-off button.<br/> 1: Low voltage repeat<br/> 0: High voltage trigger |
| `shutdown_pin` | <code>{props.shutdown_pin}</code>| <code>{props.shutdown_pin}</code> | Set the trigger pin of the power-off button. Set to `none` to disable. |

----
