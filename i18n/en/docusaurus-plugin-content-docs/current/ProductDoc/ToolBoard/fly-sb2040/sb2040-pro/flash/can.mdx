# CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select GPIO pins to set at micro-controller startup, input: gpio24
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro/flash/img/can.webp').default} size="100%" align="center" />
    * Press `Q` key, a **Save configuration** prompt will appear, then press `Y` key
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and you are back to the command line interface
    * Input `make -j4` to start compilation, it takes some time
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Finally, if the output is as follows, the compilation is successful

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flash Firmware

    * Before flashing the firmware, connect the tool board to UTOC or a mainboard flashed with bridge firmware using an XT30 data cable. Below is a reference wiring diagram.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/sb2040_utoc.webp').default} size="80%" align="left" />
    * Please ensure that the resistance between CAN H and CAN L is around `60立` when measured with a multimeter in a power-off state. If it is `120立`, you need to connect a `120立` jumper wire. If it is around `40立`, you need to remove the jumper wire
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro/flash/img/120.webp').default} size="50%" align="center" />
