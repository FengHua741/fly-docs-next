# CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select **GPIO pins to set at micro-controller startup**, input: **!gpio5,!gpio19,!gpio24**
    :::warning
        Please enter in English input mode: `!gpio5,!gpio19,!gpio24`
    :::
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/can.webp').default} size="100%" align="center" />
    * Press `Q`, then a **Save configuration** prompt will appear, press `Y` at this time
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and you have returned to the command line interface
    * Enter `make -j4` to start compiling, it takes some time
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * If the output below appears, compilation is successful

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

    <LS />


## Burn Firmware

    * Before burning the firmware, connect the tool board to UTOC or the mainboard with bridge firmware using an XT30 data cable, refer to the wiring diagram below
    * Pay attention to whether the DIP switch is set correctly
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/sb2040v3_utoc.webp').default} size="80%" align="center" />
    * Ensure that the resistance between CAN H and CAN L is around `60立` when measured with a multimeter under power-off conditions; if it is `120立`, the `120立` DIP switch needs to be turned on; if it is around `40立`, the DIP switch needs to be turned off
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/120.webp').default} size="100%" align="center" />
