# RS232 Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <RS232 />

    * Select **GPIO pins to set at micro-controller startup**, input: **!gpio5,!gpio19,!gpio24**
    :::warning
        Please enter in English input mode `!gpio5,!gpio19,!gpio24`
    :::
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/rs232.webp').default} size="100%" align="center" />
    * Press `Q`, then when **Save configuration** appears, press `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and you should have exited to the command line interface
    * Enter the following command to start compiling, it takes a bit of time
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * If the output below is displayed, compilation is successful

        ```bash
        Linking out/klipper.elf
        Creating uf2 file out/klipper.uf2
        ```


## Enter Burning Mode

    :::tip

    The left one is the `RESET button`, the right one is the `BOOT button`

    :::

    * There are two methods to enter DFU
    * The first method: With the main board completely powered off, hold the BOOT button and connect the data transfer cable to the main board and the host machine
    * The second method: Connect the data transfer cable to the main board and your host machine, hold the BOOT button and press the RST button, then release the RST button and finally release the BOOT button
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/boot.webp').default} size="100%" align="center" />
