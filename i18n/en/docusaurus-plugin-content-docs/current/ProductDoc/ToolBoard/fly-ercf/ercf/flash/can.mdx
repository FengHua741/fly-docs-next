# CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select GPIO pins to set at micro-controller startup, input: gpio17
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercf/flash/img/can.webp').default} size="100%" align="center" />
    * Press `Q` key, a **Save configuration** prompt will appear, then press `Y` key
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and exited to the command line interface
    * Enter the following command to start compiling, it takes some time
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * The final output indicating success is as follows:

        ```bash
        Linking out/klipper.elf
        Creating bin file out/klipper.bin
        ```

## Burn Firmware

    * Before burning the firmware, connect the tool board to UTOC or the main board with bridge firmware using an XT30 data cable, refer to the wiring diagram below
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * Please ensure that the resistance between CAN H and CAN L is around `60立` when measuring with a multimeter in a power-off state; if it is `120立`, connect a `120立` jumper; if it is around `40立`, remove the jumper
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercf/flash/img/120.webp').default} size="50%" align="center" />
