# CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select GPIO pins to set at micro-controller startup, input: gpio17
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/can.webp').default} size="100%" align="center" />
    * Press `Q` key, appear **Save configuration**, then press `Y` key
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and exited to the command line interface
    * Enter the following command to start compilation, it takes some time
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * If the output below is displayed, the compilation is successful

        ```bash
        Linking out/klipper.elf
        Creating bin file out/klipper.bin
        ```

## Flash Firmware

    * Before flashing the firmware, connect the tool board to UTOC or a mainboard with bridge firmware installed using an XT30 data cable, refer to the wiring diagram below
    * Please ensure that the DIP switches are set correctly
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * Please make sure that the resistance between CAN H and CAN L is around `60立` when measuring with a multimeter in a power-off state, if it is `120立`, you need to connect a `120立` jumper, if it is around `40立`, you need to remove the jumper
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/120.webp').default} size="100%" align="center" />
