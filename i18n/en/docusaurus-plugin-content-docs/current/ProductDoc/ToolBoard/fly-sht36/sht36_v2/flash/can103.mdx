# 103 Version CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select **GPIO pins to set at micro-controller startup**, input: !PC13 (Note: it should be uppercase)
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/103.webp').default} size="100%" align="center" />
    * Press `Q` key, then you will see **Save configuration**, press `Y` key again at this time
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and exit to the command line interface
    * Input `make -j4` to start compilation, it takes a bit of time
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * If the following content is output at last, the compilation is successful

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

    <LS />

## Burn Firmware

    * Use an XT30 data cable to connect the tool board to UTOC or the main board with bridge firmware before burning the firmware, below is the reference wiring diagram
    * Please pay attention to whether the DIP switch is set to the corresponding position
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Please ensure that the resistance value between CAN H and CAN L is about `60立` when measured with a multimeter in a power-off condition, if it is `120立`, a `120立` jumper needs to be connected, if it is about `40立`, the jumper needs to be removed
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/120.webp').default} size="50%" align="center" />
