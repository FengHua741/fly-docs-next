# 072 Version CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select **GPIO pins to set at micro-controller startup**, enter: !PC13 (please note it is uppercase)
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/072.webp').default} size="100%" align="center" />
    * Press `Q`, when **Save configuration** appears, press `Y` again
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and exited to the command line interface
    * Enter `make -j4` to start compilation, it takes some time
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * If the following content is output, the compilation is successful

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

    <LS />


## Burn Firmware

    * Before burning the firmware, connect the tool board to UTOC or a mainboard with bridge firmware flashed using an XT30 data cable, and refer to the wiring diagram below
    * Please ensure that the DIP switch is set to the corresponding position
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Ensure that the resistance between CAN H and CAN L is around `60立` in a power-off state; if it is `120立`, connect the `120立` jumper; if it is around `40立`, remove the jumper
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/120.webp').default} size="50%" align="center" />
