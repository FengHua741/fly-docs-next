# Using Eddy

    :::info
    * When installing Eddy, ensure the bottom is at least 1-2 mm above the nozzle
    * When calibrating Eddy frequency, keep the coil PCB at least 5mm away from the heat bed
    * Please ensure the heat bed is at room temperature during testing and do not turn on the heat bed during the test

    :::

## Reference Configuration and Usage Instructions

##### Add the following configuration to the `printer.cfg` file for debugging Eddy

    ### Bed Leveling Configuration

        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 43
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

        * Please note that currently there is no external 1612 module available, so this configuration is not yet usable
        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 1.0
        i2c_address: 42
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

    ### Temperature Compensation Configuration

        ```cfg
        [temperature_probe fly_eddy_probe]
        sensor_type:Generic 3950
        sensor_pin:SHT36:gpio28
        ```

    ## Bed Mesh Commands
    
        * Add the following macro to the `printer.cfg` configuration
            ```bash
            [gcode_macro BED_MESH_CALIBRATE]
             rename_existing: _BED_MESH_CALIBRATE
             gcode: 
                   _BED_MESH_CALIBRATE METHOD=rapid_scan {rawparams}
            ```
        * For local bed mesh commands, ensure Klipper can use object exclusion feature, it's recommended to add to the startup macro
            ```bash
            BED_MESH_CALIBRATE METHOD=rapid_scan adaptive=1 
            ```

### bed_mesh Configuration

    * In `[bed_mesh]`, set the `horizontal_move_z` parameter to 2 to allow Eddy to be as close to the platform as possible during scanning

    ```cfg
    [bed_mesh]
    horizontal_move_z: 2
    ```
## Collision Prevention Macros

    ### `[quad_gantry_level]` Optimization for Bed Leveling
        * If `[quad_gantry_level]` is configured, add the macro below to the configuration; otherwise, it's not necessary
        * This macro will reduce the probe height as the gantry height difference decreases
            ```cfg
            [gcode_macro QUAD_GANTRY_LEVEL]
            rename_existing: _QUAD_GANTRY_LEVEL
            gcode:
                SAVE_GCODE_STATE NAME=STATE_QGL
                BED_MESH_CLEAR
                {% if not printer.quad_gantry_level.applied %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _QUAD_GANTRY_LEVEL horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_QGL
            ```

    ### `[z_tilt]` Optimization for Bed Leveling

        * If `[z_tilt]` is configured, add the macro below to the configuration; otherwise, it's not necessary
        * This macro will reduce the probe height as the gantry height difference decreases
            ```cfg
            [gcode_macro Z_TILT_ADJUST]
            rename_existing: _Z_TILT_ADJUST
            gcode:
                SAVE_GCODE_STATE NAME=STATE_Z_TILT
                BED_MESH_CLEAR
                {% if not printer.z_tilt.applied %}
                _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
                {% endif %}
                _Z_TILT_ADJUST horizontal_move_z=2 METHOD=rapid_scan ADAPTIVE=1
                # G28 Z
                RESTORE_GCODE_STATE NAME=STATE_Z_TILT
            ```

    ### Eddy Likely Heat Bed Interference Optimization Macro

        * If using a high-power AC heat bed (over 500W)
        * The macro below turns off the heat bed during leveling and reheats it after leveling is complete
            ```cfg
            [gcode_macro BED_MESH_CALIBRATE]
            rename_existing: _BED_MESH_CALIBRATE
            gcode:
                {% set TARGET_TEMP = printer.heater_bed.target %}
                M140 S0
                _BED_MESH_CALIBRATE {rawparams}
                M140 S{TARGET_TEMP}
            ```

   # Debugging Steps:
   ## Step 1: Modify Frequency

    ### Default Frequency Modification for ldc1612
    This method is only for sht36-v3/SB2040-v3 or SB2040-Pro-v3
    Use Mobaxterm tool to execute commands
    
    Download the required <Button variant="contained" disableElevation href="/docs/ResDownload/auxiliary_software/">Mobaxterm Tool</Button>
    
    [For detailed usage, please refer to](/docs/ProductDoc/SBC/fly-pi-v2/ssh)


    Execute the first command and press Enter

    ```cfg
sed -i 's/LDC1612_FREQ = 12000000/LDC1612_FREQ = 40000000/g' ~/klipper/klippy/extras/ldc1612.py
    ```

    Execute the second command and press Enter
  ```cfg
grep "LDC1612_FREQ = 40000000" ~/klipper/klippy/extras/ldc1612.py
  ```
    If the replacement is successful, a line containing `LDC1612_FREQ = 40000000` will be returned. If there is no return, the replacement has not succeeded.
    Or use this command to check if it was successful
    <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/1612.webp').default} size="110%" align="left" />
```cfg
sed -n '10,15p' ~/klipper/klippy/extras/ldc1612.py
```
   <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/ldc161240mhz.webp').default} size="30%" align="left" />

   Reminder: After modification, use this command to restart the Klipper service and recalibrate Eddy frequency
   ```cfg
   systemctl restart klipper
   ```
   Eddy frequency calibration command
   ```cfg
   PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
   ```
