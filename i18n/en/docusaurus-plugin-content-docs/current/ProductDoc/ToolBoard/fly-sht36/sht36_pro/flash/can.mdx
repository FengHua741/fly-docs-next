# CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select GPIO pins to set at micro-controller startup, input: `!gpio13` Please note that `!` is uppercase
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/can.webp').default} size="100%" align="center" />
    * Press the `Q` key, a **Save configuration** prompt will appear, then press the `Y` key
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and you should exit to the command line interface
    * Enter the following command to start compiling, it may take some time
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * If the output below appears, the compilation is successful

        ```bash
        Linking out/klipper.elf
        Creating bin file out/klipper.bin
        ```
    <LS />


## Burn Firmware

    * Before burning the firmware, use an XT30 data cable to connect the tool board to UTOC or the main board with bridge firmware installed. Below is a reference wiring diagram
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/sht36pro_utoc.webp').default} size="80%" align="left" />
    * Ensure that the resistance between CAN H and CAN L is around `60立` when measured with a multimeter in a power-off state. If it is `120立`, you need to add a `120立` jumper. If it is around `40立`, you need to remove the jumper
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/120.webp').default} size="80%" align="center" />

    <CANFLASH />
