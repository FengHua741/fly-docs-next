# CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select GPIO pins to set at micro-controller startup, enter: !gpio13 Please note that! is uppercase
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/can.webp').default} size="100%" align="center" />
    * Press `Q` key, and when **Save configuration** appears, press `Y` key
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and you should exit to the command line interface
    * Enter `make -j4` to start compilation, it takes some time
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * If the following content is output, the compilation is successful

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Burn Firmware

    * Before burning the firmware, use an XT30 data cable to connect the tool board to the UTOC or the motherboard that has been flashed with the bridge firmware, the reference wiring diagram is below
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/sht36pro_utoc.webp').default} size="80%" align="left" />
    * Please ensure that the resistance measured with a multimeter between CAN H and CAN L under power-off condition is about `60立`, if it is `120立` then a `120立` jumper needs to be connected, if it is about `40立` the jumper needs to be removed
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/120.webp').default} size="80%" align="center" />
