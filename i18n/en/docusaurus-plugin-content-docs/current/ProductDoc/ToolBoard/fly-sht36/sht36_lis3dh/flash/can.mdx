# CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select GPIO pins to set at micro-controller startup, input: !gpio20 (note that ! is uppercase)
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/can.webp').default} size="100%" align="center" />
    * Press `Q`, then **Save configuration** will appear, press `Y` again at this time
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and you should exit to the command line interface
    * Input `make -j4` to start compilation, it takes a while
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * If the output below appears, the compilation is successful

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

    <LS />


## Burn Firmware

    * Before burning the firmware, connect the tool board to UTOC or the main board flashed with bridge firmware using an XT30 data cable. Below is the reference wiring diagram
    * Please check if the DIP switches are set correctly
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Please ensure that the resistance between CAN H and CAN L measured with a multimeter in power-off condition is around `60立`. If it is `120立`, add a `120立` jumper; if it is around `40立`, remove the jumper
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/120.webp').default} size="50%" align="center" />
