# CAN Firmware Compilation

## Start Compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Select GPIO pins to set at micro-controller startup, input: `!gpio20` Please note that `!` is uppercase
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/can.webp').default} size="100%" align="center" />
    * Press `Q` key, and when **Save configuration** appears, press `Y` key again
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and you should have exited to the command line interface
    * Enter the following command to start compiling, it takes a bit of time
    ```bash
    make -j4
    ```
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Finally, if the output below appears, the compilation is successful

        ```bash
        Linking out/klipper.elf
        Creating bin file out/klipper.bin
        ```

    <LS />


## Burn Firmware

    * Before burning the firmware, use an XT30 data cable to connect the tool board to UTOC or the mainboard with bridge firmware installed. Below is the reference wiring diagram
    * Please check whether the DIP switch is set correctly
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Please ensure that the resistance between CAN H and CAN L is around `60立` in a power-off state. If it is `120立`, you need to connect a `120立` jumper wire. If it is around `40立`, you need to remove the jumper wire
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/120.webp').default} size="50%" align="center" />
