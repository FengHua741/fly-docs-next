    * Press `Q` key, when **Save configuration** appears, press `Y` again
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * Now the configuration should be saved and you should have exited to the command line interface
    * Enter `make -j4` to start compiling, it takes a bit of time
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * If the following content is output at the end, the compilation is successful

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Confirm if in Burning Mode

    * Ensure that the host machine can connect to the internet normally and ensure that the data cable connecting the host machine and the device has data transfer functionality.
    * After SSH connection to the host machine, enter `lsusb` and press Enter. If the information circled in the picture below appears, you can proceed to the next step.
        * If there is no feedback at all, this is a system problem with the host machine, and we are unable to help. You need to replace a confirmed normal system or replace the host machine.
        * If there is a prompt that the `lsusb` command is not found, you can execute the following command to install it:
            ```bash
            sudo apt-get install usbutils
            ```
            <ImageView image={require('@site/docs/General/get-id/img/6177.webp').default} size="80%" align="left" /> 
    * The ID displayed in the red box can be used as a reference but is not fixed (`XX` is variable).
        * Some host machines may display incomplete or no information due to system problems.
