# FlyOS-Fast 常见问题

## 问：执行命令提示`command not found`
- **原因**：系统没有这个命令。
- **解决方法**：系统内已经内置了众多常用软件包，如需其它包可提交给我们，如有必要会在下一版本预装。

## 问：为什么无法执行`apt update`命令？
- **原因**：FlyOS-Fast为轻量化快速系统，没有apt包管理工具，无法执行`apt update`命令。
- **解决方法**：系统内已经内置了众多常用软件包，如需其它包可提交给我们，如有必要会在下一版本预装。

## 问：执行烧录命令提示`-bash: ~/klippy-env/bin/python: No such file or directory`
- **原因**：FlyOS-Fast没有采用venv虚拟环境。
- **解决方法**：将`~/klippy-env/bin/python`替换为`python`执行即可。

## 问：为什么不能登录`fly`用户？
- **原因**：FlyOS-Fast只有一个root用户，所有操作均在root用户。
- **解决方法**：系统没有`fly`用户，直接使用`root`登录即可。

## 问：为什么打不开9999端口的fly-tools？
- **原因**：FlyOS-Fast没有安装fly-tools。
- **解决方法**：fluidd与mainsail均已支持查询ID，屏幕等配置均在[系统配置](/docs/DebugDoc/flyos-fast/config)。

## 问：为什么屏幕只显示LOGO，没有KlipperScreen界面？
- **原因**：KlipperScreen未启用。
- **解决方法**：在[系统配置](/docs/DebugDoc/flyos-fast/config)中启用KlipperScreen，将`klipper_screen`设置为`1`。

## 问：为什么系统不提供网页更新klipper等组件？
- **原因**：FlyOS-Fast内置的klipper，moonraker等组件均为二次开发版本。
- **解决方法**：如有需要可自行在命令行中更新，但是不建议这么做，等待系统更新即可。

## 问：快乐兔等第三方组件安装脚本无法使用？
- **原因**：FlyOS-Fast默认用户为`root`, 这些第三方安装脚本都判断为root而不进行安装。
- **解决方法**：只需简单修改绕过安装脚本的root检测即可，可参考[这里](https://github.com/kluoyun/Happy-Hare/tree/flyos-dev), 其它的安装脚本都大同小异。
